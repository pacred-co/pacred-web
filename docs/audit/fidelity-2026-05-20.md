Screen 7 (`/admin/hr/humanresource`) and screen 6 (`/admin/accounting/*`) ship as Pacred-original menubar hubs while a legacy with concrete tables/forms exists — but the legacy uses left-rail nested menus, not top menubars, and the actual destination pages are mostly TODO (will 404). **Faithful work:** Screens 1, 4 are genuinely faithful 1:1 transcriptions of the legacy table HTML (correct columns, correct labels, correct role gating, correct SQL → tb_* model). Screen 2 covers all 9 action queues but skips the heavy customer-row composite cell.

## Top 5 paradigm gaps for ภูม to close FIRST

1. **🔴 SCREEN 5 — `/admin/forwarders` reads rebuilt `forwarders` table, not legacy `tb_forwarder`.** Status enum is `pending_payment/shipped_china/in_transit/arrived_thailand/out_for_delivery/delivered/cancelled` instead of legacy `fStatus 1..7,99`. Transport_type is `truck/ship/air` strings instead of legacy `fTransportType 1/2/3` ints. Customer name comes from `profiles.first_name+last_name`, not legacy `tb_users.userID`. **No customer/staff retraining is possible — this is the rebuilt app under new sidebar paint.** Re-port the entire page to read `tb_forwarder` with the legacy fStatus tabs (รอเข้าโกดังจีน · ถึงโกดังจีนแล้ว · กำลังส่งมาไทย · ถึงไทยแล้ว · รอชำระเงิน · เตรียมส่ง · กำลังจัดส่ง · ส่งแล้ว · เครติดสินค้า · สถานะพิเศษ — 10 tabs, not 7).
2. **🔴 SCREEN 3 — `/admin/barcode/cargo/all` is non-functional.** `@ericblade/quagga2` is imported but is NOT in `package.json` — the CameraScanner client component prints "Quagga library not installed" instead of opening the camera. Fix: `pnpm add @ericblade/quagga2` (the file already has a TODO marker at L29-31 and L99). Camera unavailability blocks the warehouse barcode-scan workflow which is the entire reason the page exists.
3. **🔴 SCREEN 7 — `/admin/hr/humanresource` cards link to live pages that DON'T match legacy.** "พนักงาน · ทั้งหมด" → `/admin/admins` (rebuilt admin table, not `admin-table.php`-style with the legacy 6 deep Partner sub-filters incl. `position=driver&truck-type=4Wheels`). "เวลาเข้างาน" → `/admin/hr/attendance` not legacy `time-attendance-system.php`. The TOP-menubar pattern itself diverges from the legacy LEFT-rail nested menu (4 sections → 4 levels of nesting incl. Partner > shipping-clearance > Import/Export/I&E).
4. **🔴 SCREEN 6a — `/admin/accounting/cargo` ships as a hub of CARDS, not the actual ledger page.** Legacy `acc-system-cargo.php` lands directly on `home.php` (a single ระบบบัญชี Cargo H2 banner) THEN navigates via the dropdown. Our version lands on a 6-card "เข้าใช้งานเร็ว" grid which has no legacy equivalent — staff will not recognise this landing.
5. **🔴 SCREEN 2 — `/admin/forwarder-action` is missing the action='Note' default for the bulk-edit notes flow.** The legacy supports inline `?action=Note` AS THE LAND-HERE default with the long full-customer row (60+ inline fields incl. SVIP badge, address concatenation, payment method, print statuses 1/2/3, fStatusCarOn/Off badges, due-date arithmetic). Our port shows 9 thin columns. Operators searching for "the note about cabinet number X" will not find the cabinet, ZIP, ShipBy, payment-method context that lives in the legacy composite cell.

- Screen 6b: `/admin/accounting/freight` has NO legacy equivalent — explicitly Phase-C ground.
