# เทียบใบวางบิลของซัพพลายเออร์กับระบบเรา — บทเรียน

> เก็บจาก 2026-07-23 (บิลต้นทุน MOMO · owner สั่ง "ต้องขึ้น คิวในระบบ คิวที่ momo เรียกเก็บมา
> ดิฟกัน + - เท่าไร ต้นทุน MOMO เก็บเราเท่าไร ระบบเรา ขายเขาไปเท่าไร มีช่องแสดงผล diff กำไร").
> ใช้ได้กับทุกเคสที่เอาเอกสารของ "เจ้าอื่น" มาเทียบกับข้อมูลของเรา — ไม่ใช่แค่ MOMO.

## L-1 · ทั้ง 2 ฝั่งมี "หน่วยซ่อน" — ต้องปรับฐานก่อนเทียบ เสมอ

เทียบเลขข้ามระบบ จุดตายไม่ใช่การลบเลข แต่คือ **เลขสองตัวไม่ได้อยู่หน่วยเดียวกัน** แล้ว
ดิฟที่ได้ก็ผิดแบบเงียบๆ (ไม่ error ไม่ crash — แค่ตัวเลขโกหก).

รอบนี้เจอพร้อมกัน **2 ชั้น**:

| ฝั่ง | กับดัก | ถ้าลืม |
|---|---|---|
| ใบซัพพลายเออร์ | คอลัมน์คิว **ไม่ใช่ยอดทั้งบรรทัดเสมอ** — บาง template เป็น "ต่อกล่อง" ต้อง × จำนวนกล่อง | คิวที่เขาเรียกเก็บอ่านต่ำ → ขึ้นดิฟ **+ ปลอม** ทุกบรรทัดที่มีหลายกล่อง |
| ระบบเรา | `fvolume` ก็ไม่ใช่ยอดทั้งแถวเสมอ — กฎ `famountcount` (per-box ต้อง × famount) | คิวเราอ่านต่ำ ×famount บนแถวคีย์มือ/legacy/TTW |

**วิธีจับว่าปรับฐานถูกหรือยัง (ใช้ได้จริง):** ดู "ขนาด" ของดิฟ ไม่ใช่แค่ว่ามีดิฟไหม —
ปรับฐานถูก ดิฟจะเหลือแค่ **เศษทศนิยม** (เขาพิมพ์ 4 ตำแหน่ง เราเก็บ 6 → 0.0001-0.0010)
ปรับฐานผิด ดิฟจะโผล่มาเป็น **เท่าตัว**. รันกับใบจริง 4 ใบแล้วเห็นเศษ 4-vs-6 ตำแหน่งล้วน =
หลักฐานว่าถูก แข็งกว่าอ่านโค้ดซ้ำ 10 รอบ.

## L-2 · Σ เฉพาะที่ "เทียบได้" — ที่เทียบไม่ได้ต้องรายงานแยก ห้ามกลืน

บรรทัดที่หาแถวในระบบไม่เจอ **ไม่มีคิว/ขาย/ต้นทุนของเรา**ให้เทียบ. ถ้าเอาไปรวมใน Σ:
ยอดบิลจะบวม กำไรจะเพี้ยน และคนอ่านจะไม่รู้เลยว่ามีของที่ยังไม่ landed.

กติกา: `Σ เทียบ` (เฉพาะ matched) + `Σ ทั้งใบ` + `จำนวน/ยอดที่ยังเทียบไม่ได้` **โชว์ 3 ตัวคู่กัน**
และ invariant ต้องปิด: `matched + unmatched = ทั้งใบ` (ทั้งจำนวนบรรทัดและยอดเงิน) — ผูกไว้ในเทส

เพิ่มอีกชั้น: แถวที่ **ยังไม่ตั้งราคาขาย** (sell = 0) ต้องนับแยกด้วย ไม่งั้นกำไรจะติดลบหลอกตา
ทั้งที่แค่ยังไม่ได้ตั้งราคา.

## L-3 · เลือก "ขา" ที่จะเทียบให้แคบเท่าที่ซัพพลายเออร์เก็บจริง

อยากรู้กำไร → ต้องจับคู่ **ขาย** กับ **ทุน** ที่เป็นขาเดียวกัน.

MOMO เก็บเราเฉพาะ **ค่าเฟรทจีน→ไทย** → ต้องเทียบกับ `ftotalprice` (ค่านำเข้า) เท่านั้น
**ไม่ใช่** ยอดบิลรวมของลูกค้า (`calcForwarderOutstanding` ที่มีค่าส่งไทย/ตีลัง/จีน+ ปนอยู่).
ถ้าเอายอดรวมมาเทียบ กำไรจะพองเทียบกับต้นทุนที่ไม่เคยรวมขาพวกนั้น.

**ทางลัดที่ควรใช้เสมอ:** ไปดูว่าหน้ารายงานที่ทีมบัญชีใช้อยู่แล้วเขาจับคู่ยังไง แล้วใช้ตัวเดียวกัน
(รอบนี้ = `/admin/report-cnt` cnt-list-table: `costSum`/`priceSum`/`profitSum`) → 2 จอพูดตรงกัน
บัญชีไม่ต้องแปลเลขข้ามหน้า และไม่ต้องเถียงกันว่าเลขไหนถูก.

## L-4 · กำไรรายกล่อง "ติดลบ" ไม่ใช่ขาดทุน — เป็นผลของฐานคนละตัว

อาการที่ถูกทักซ้ำๆ ("ขายต่ำกว่าทุน ผิดปกติหรือเปล่า") มีสูตรอธิบายชัด:

```
ขาย คิดตาม น้ำหนัก (฿/kg) · ทุน คิดตาม คิว (฿/CBM)
จุดคุ้มทุน = (ทุนต่อคิว) ÷ (ขายต่อกิโล)      เช่น 2,500 ÷ 11 = 227.3 kg/คิว
กล่องที่ความหนาแน่น < จุดคุ้มทุน → ติดลบ · > → บวก · ทั้งชิปเม้นรวมแล้วบวก
```

พิสูจน์กับ prod (1782459481): 2 กล่องติดลบ (216.7 · 196.1 kg/คิว) แต่ทั้งชิปเม้น 311.3 kg/คิว
= **+฿789.75**.

**สิ่งที่ต้องทำบนจอ** ไม่ใช่ซ่อนเลขลบ แต่คือ **โชว์ยอดทั้งชิปเม้นข้างๆ** — และ Σ นั้นต้องดึงจาก
**พี่น้องทุกแถวใน DB** ไม่ใช่แค่บรรทัดที่อยู่บนใบรอบนี้ (ซัพพลายเออร์บิลไม่ครบครอบครัวได้)
ไม่งั้นคำว่า "ทั้งชิปเม้น" กลายเป็นคำโกหก. ถ้าทั้งชิปเม้นติดลบจริง → ค่อยเตือนแดง อันนั้นผิดจริง.

## L-5 · เอกสารประวัติต้อง "แช่เลข" ไม่ใช่คิดสด

ยอดสรุปในหน้าประวัติ (กล่อง/คิว/กิโล/ขาย/กำไร) ถ้า derive สดจากตารางหลักทุกครั้งที่เปิด
→ ตัวเลขจะ **เปลี่ยนย้อนหลัง** เมื่อมีคนแก้ราคา/น้ำหนักทีหลัง → ตรวจย้อนไม่ได้ว่าตอนกดจ่ายเห็นอะไร

snapshot ตอนสร้างเอกสาร (แนวเดียวกับใบเสร็จที่ PIN ยอด frozen · G1 2026-07-08). และ
**อย่าเก็บค่าที่คำนวณได้** — กำไร = ขาย − ทุน ให้ derive ตอนอ่าน มีที่มาที่เดียว เลข 2 ตัวขัดกันเองไม่ได้.
จังหวะที่ดีที่สุดคือทำตอนตารางยังว่าง (รอบนี้ prod 0 แถว → ไม่มี backfill เลย).

## L-6 · เอกสารมี 2 เลข: เลขของตัวเอง กับ เลขที่มันอ้างถึง

ใบเสร็จ MOMO มีทั้ง `NO: REC-…` (ตัวตนของมัน) และ `อ้างอิง: INV-…` (ใบแจ้งหนี้ที่มันอ้างถึง).
regex ที่จับ "เลขแรกที่เจอ" จะได้ผิดตัวแบบสุ่มตามลำดับที่ pdf.js คายข้อความออกมา.

→ ยึด **เลขที่พิมพ์หลัง "NO:"** เป็นตัวตนเสมอ · fallback ค่อยว่ากัน · แล้วล็อกด้วยเทสที่ใช้
ข้อความจริงจากใบที่มีทั้ง 2 เลข.

**เรื่องตั้งชื่อไฟล์:** ไฟล์ที่โหลดจากเว็บซัพพลายเออร์มักชื่อซ้ำ (`…(15).pdf`) → ตั้งชื่อภายในด้วย
**เลขในเอกสารเอง** แล้วให้ storage เติม timestamp กันชน = คนเปิดดูรู้ทันทีว่าไฟล์ไหนคือใบไหน.
