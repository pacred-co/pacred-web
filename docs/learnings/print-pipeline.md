# พิมพ์เอกสาร (print) — บทเรียนที่แพงที่สุดของ 2026-07-23/24

> เก็บจากรอบที่ owner ต้องส่ง **PDF จริงกลับมา 2 รอบ** ถึงจะจบ — ทั้งที่ผมรายงานว่า
> "แก้แล้ว · gate เขียว" ไปก่อนหน้านั้น. ทุกข้อในนี้คือของที่ **tsc/lint/build บอกว่าผ่านหมด**

## 🔴 L-0 · กฎเหล็ก: "พิมพ์" พิสูจน์ได้ทางเดียวคือ **ดู PDF ที่ออกมาจริง**

`build ผ่าน` · `grep เจอ class` · `หน้า render ถูก` — **ไม่มีอันไหนพิสูจน์เรื่องพิมพ์ได้เลย**
เพราะ `@media print` เป็นอีกโลกที่ไม่ถูกแตะตอน render ปกติ:

| สิ่งที่เช็ค | พิสูจน์อะไรได้ | พิสูจน์ไม่ได้ |
|---|---|---|
| build/tsc/lint | โค้ดคอมไพล์ได้ | ทุกอย่างด้านล่าง |
| grep เจอ `.no-print` | มี class | **มีกฎที่ซ่อนมันจริงไหม** |
| screenshot หน้าจอ | จอสวย | กระดาษออกมาหน้าตาไหน |
| อ่าน `<title>` ในไฟล์ | มี tag | **tag ไหนที่เบราว์เซอร์ใช้จริง** |

**ขั้นต่ำที่ต้องทำก่อนพูดว่า "พิมพ์ได้แล้ว":** ยิง JS บนหน้า render จริง ตรวจ 4 อย่าง —
(1) กฎ print โหลดจริงไหม (ไล่ `document.styleSheets` หา MEDIA_RULE)
(2) มี element `position:fixed/sticky` ตัวไหน **ไม่มี** `print:hidden` เหลืออยู่
(3) `document.title` ตัวแรกใน HTML คืออะไร
(4) วัดความสูงเนื้อหาเป็น mm แล้วหารด้วยพื้นที่หน้ากระดาษ → ได้กี่หน้า
ถ้าเป็นเอกสารสำคัญ → **ให้คนพิมพ์จริงแล้วส่ง PDF กลับมาดู**

---

## L-1 · `<title>` ในหน้า **ไม่ชนะ** metadata — และมันเงียบสนิท

Next แปะ `<title>` จาก metadata ของ layout ให้ **ทุกหน้า** อยู่แล้ว. `<title>` ที่เขียนใน body
จึงกลายเป็นตัวที่ 2 ใน HTML และ **เบราว์เซอร์ใช้ตัวแรกเสมอ** → PDF ได้ชื่อ generic
ไม่มี error ไม่มี warning ไม่มีอะไรเตือน.

```
❌ <title>{docNo}</title>            ใน body — ไม่มีผล ถ้า layout ตั้ง title.default/template
✅ export const metadata = { title: { absolute: "…" } }
✅ export async function generateMetadata({ params }) { return { title: { absolute: docNo } } }
```
`absolute` = ไม่ต่อท้าย `| ชื่อเว็บ` (ชื่อไฟล์จะได้สะอาด)

**วิธีเช็คที่เชื่อได้:**
```bash
curl -s <url> | grep -oE '<title[^>]*>[^<]*</title>' | head -1   # ← ตัวแรกเท่านั้นที่นับ
```
ผมเคลมว่า "ชื่อไฟล์ = เลขเอกสารแล้ว" โดยดูแค่ว่ามี tag อยู่ในไฟล์ → **ผิดทั้ง 13 หน้า**

## L-2 · `min-h-screen` = ตัวสร้าง "หน้าขาวเบิ้ล"

`min-h-screen` (100vh) บนจอ = สูงเต็มจอ · **ตอนพิมพ์ = สูงเต็มหน้ากระดาษ**
→ กล่องครอบถูกดันให้เต็มหน้าเสมอ แล้ว padding/margin ที่เกินมาแค่ 1px ก็ล้นไป
สร้างหน้าถัดไปที่ว่างเปล่า → **เอกสาร 2 แผ่น พิมพ์ออกมา 4 แผ่น**

```css
@media print {
  .min-h-screen, .h-screen { min-height: 0 !important; height: auto !important; }
}
```
⚠️ อย่าเหมารวม `min-height` ที่ตั้งเป็น **mm** (เช่น `.receipt-page { min-height: 287mm }`) —
พวกนั้นตั้งใจให้เต็มหน้ากระดาษจริงๆ

ตัวอื่นในตระกูลเดียวกันที่ต้องระวัง: `break-after/page-break-after` บน element **ตัวสุดท้าย**
(ต้องมี `:last-child { break-after: auto }` เสมอ) · `height: 100vh` · `min-h-dvh`

## L-3 · `@page` ที่ไม่ประกาศ = ขนาดกระดาษตามใจเครื่องแต่ละคน

หน้าที่ไม่มี `@page` เลย → ใช้ค่าเริ่มต้นของเบราว์เซอร์/เครื่อง (บางเครื่อง Letter) → **ตกขอบ**
audit เจอ **4 หน้าเอกสารเงิน**ไม่ประกาศเลย.

ตั้ง **ค่าตั้งต้นกลาง** ที่ globals.css:
```css
@page { size: A4 portrait; }
```
หน้าที่มีขนาดเฉพาะ (สติกเกอร์ · A5 · แนวนอน) ประกาศเองใน `<style>` ของหน้า ซึ่งอยู่
**ท้ายเอกสาร = มาทีหลัง = ชนะ** → ไม่ต้องแก้อะไร

---

## L-4 · `.no-print` ที่ไม่มีกฎซ่อน = ป้ายเปล่า

`.no-print` ถูกใช้ **31 ไฟล์** แต่กฎที่ซ่อนมันอยู่ใน stylesheet ที่ scope เฉพาะ legacy wrapper
+ โหลดแค่ ~5 หน้า → หน้าแอดมิน 9 หน้าติดป้ายไว้แต่ไม่มีใครบังคับ → **ปุ่มติดไปในกระดาษ**

> class ที่ต้องพึ่ง "กฎที่อยู่ที่อื่น" คือ **สัญญาที่ไม่มีใครตรวจ** — ประกาศกฎกลางเสมอ

**และอย่าไล่ทีละตัว** — รอบแรกผมแก้แถบขวา desktop แล้วบอกว่าจบ แต่ owner พิมพ์จาก
จอแคบ → โดนแถบล่างมือถือ (คนละ element). วิธีที่ถูก = **ยิง JS ไล่ทุก element ที่ลอย
ทั้งจอกว้างและจอแคบ** เจอครบ 7 จุดในรอบเดียว

## L-5 · เอกสารต้องไม่ถูกตัดคาหน้า

```css
@media print { .doc-unit { break-inside: avoid; page-break-inside: avoid; } }
```
ถ้าใส่ไม่พอในหน้านี้ มันจะย้ายไปหน้าถัดไป **ทั้งใบ** แทนที่จะโดนผ่าครึ่ง

**อยากให้ลง 1 แผ่น → วัดก่อน อย่าเดา:**
```js
const mm = px => px / (96/25.4);
mm(el.getBoundingClientRect().height)     // เทียบกับ 297 − (margin บน+ล่าง)
```
เคสฟอร์ม 50 ทวิ: 160.4mm × 2 = 320.8 > 281 → บีบ **line-height + padding เท่านั้น**
(ห้ามลดขนาดฟอนต์ — เอกสารต้องอ่านออก/เซ็นได้) → 132.7mm × 2 = 265.4 **ลงแผ่นเดียว**

## L-6 · gate ที่ "บล็อก" ต้องดูว่าอยู่ฝั่งไหนของเหตุการณ์เงิน

(เจอในรอบเดียวกัน · คนละเรื่องกับพิมพ์ แต่รากเดียวกัน = guard วางผิดที่)

guard ตัวเดียวถูกเรียก 2 จังหวะ: **ก่อนลูกค้าจ่าย** (quote) และ **หลังเงินเข้าแล้ว** (บัญชีอนุมัติ)
· บล็อกตอน quote = ป้องกันได้จริง ✅
· บล็อกตอนอนุมัติ = **กันอะไรไม่ได้เลย** เงินอยู่ในบัญชีแล้ว มีแต่ทำงานค้าง ❌

และต้องแยก **ช่องที่คิดเงิน** ออกจาก **ช่องที่แค่พิมพ์บนเอกสาร** —
ชื่อ+เลขภาษี = บล็อก (ออกใบเสร็จนามนิติไม่ได้จริง) · ที่อยู่ = ผ่าน+เตือน (ยอดเท่ากันทุกบาท)

ถ้า gate ทำให้ลูกค้าตัน (ไก่-กับ-ไข่) → **แก้ด้วยการให้เครื่องมือ ไม่ใช่ปลด gate**
(เคสนี้: ฟอร์ม 50 ทวิ ที่กรอกข้อมูลให้แล้ว เหลือแค่เซ็น)

## เช็คลิสต์ก่อนบอกว่า "พิมพ์ได้แล้ว"

- [ ] กฎ `@media print` โหลดจริงบนหน้า (ตรวจจาก `document.styleSheets`)
- [ ] `position:fixed/sticky` ที่ไม่มี `print:hidden` = **0** ทั้งจอกว้างและจอแคบ
- [ ] `<title>` **ตัวแรก** ใน HTML = เลข/ชื่อเอกสาร
- [ ] มี `@page` (ของตัวเอง หรือค่าตั้งต้นกลาง)
- [ ] วัดความสูง → จำนวนหน้าตรงกับที่ควรเป็น (ไม่มีหน้าขาวแทรก)
- [ ] เอกสารมีเงื่อนไข → บล็อกถูกฝั่ง + มีทางออกให้ผู้ใช้ + อธิบายบนจอ
- [ ] **ให้คนพิมพ์จริง 1 ใบ แล้วดู PDF**
