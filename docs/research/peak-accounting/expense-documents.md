## 1️⃣ ใบสั่งซื้อ (PO) + ใบจ่ายเงินมัดจำ (Deposit)

### 1) ใบสั่งซื้อ (Purchase Order · PO) — art 5775125
- **ใช้ทำอะไร:** เอกสารแสดงความประสงค์สั่งซื้อจากซัพพลายเออร์ = สัญญาการสั่งซื้อ · ทำ **ก่อน** รับของ/วางบิลจริง.
- **สร้าง 3 วิธี:** เมนู รายจ่าย > ใบสั่งซื้อ > + สร้าง · นำเข้า (Import) · คลังเอกสาร (+ สร้างจากใบเสนอราคาได้).
- **ฟิลด์:** อ้างอิง · เลขที่เอกสาร (auto `PO-YYYYMM00NN` · ตั้งรูปแบบได้) · ข้อมูลผู้ขาย (ชื่อ*/ที่อยู่/เบอร์ · ดึง auto) · วันที่ออก* · กลุ่มจัดประเภท (โครงการ/แผนก/สาขา · สูงสุด 4) · ประเภทราคา (แยก/รวมภาษี) · สกุลเงิน (default THB · FX อ้างอิง ธปท. ย้อน 1 วัน) · รายการ (บัญชี/คำอธิบาย ≤1,000 ตัว/จำนวน/ราคา/ส่วนลด/ภาษี/WHT) · หมายเหตุ · แนบไฟล์ · แท็ก.
- **สถานะ:** บันทึกร่าง · **อนุมัติใบสั่งซื้อ**. **Convert chain:** PO → **บันทึกซื้อสินค้า (EXP)**.
- **🔴 การลงบัญชี: NON-ACCOUNTING** — "ยังไม่บันทึกบัญชี/ไม่ตัดสต็อก เป็นเพียงเอกสารยืนยัน" → **ไม่มี Dr/Cr ตอนสร้าง PO** · ลงบัญชีจริงตอน convert เท่านั้น.

### 2) ใบจ่ายเงินมัดจำ (Deposit Payment · DP) — art 9438787
- **ใช้ทำอะไร:** จ่ายมัดจำล่วงหน้าบางส่วนให้ผู้ขาย ก่อนรับของ/วางบิลจริง.
- **สร้าง 2 กรณี:** ไม่อ้างอิง PO · อ้างอิง PO ที่อนุมัติแล้ว.
- **convert chain:** PO อนุมัติ → ใบจ่ายมัดจำ → จ่ายชำระ → สถานะ **"รอหักมัดจำ"** → บันทึกซื้อ (EXP) กด **"+ เลือกเงินมัดจำ"** ดึงมาตัด → ยอดหลังหักมัดจำ = ยอดชำระส่วนที่เหลือ.
- **🔴 การลงบัญชี** (บทความไม่ระบุเลข GL เป็นข้อความ · โชว์แค่รูป journal · treatment มาตรฐาน):
  - **ตอนจ่ายมัดจำ:** Dr **115104 เงินจ่ายล่วงหน้า-เงินมัดจำ** [+ Dr ภาษีซื้อ] / Cr ธนาคาร [+ Cr WHT ค้างจ่าย ถ้าหัก].
  - **ตอนบันทึกซื้อ+ตัดมัดจำ:** Dr ค่าใช้จ่าย/สินค้า [+ Dr ภาษีซื้อ] / **Cr 115104 (ล้าง asset)** + Cr เจ้าหนี้/เงินสด (ส่วนที่เหลือ).
- **Gotcha:** ต้องสถานะ "รอหักมัดจำ" (จ่ายแล้ว) ก่อนตัด · วันที่ DP ≤ วันซื้อ · ตัดได้สูงสุด 10 ใบมัดจำ/1 บันทึกซื้อ · WHT ระบบคิดบนยอด**หลัง**หักมัดจำ · ยังไม่รองรับสกุลต่างประเทศ + Old PEAK.

## 2️⃣ บันทึกซื้อสินค้า (Purchase Record — ซื้อเข้าสต็อก)

> ต่างจาก "บันทึกค่าใช้จ่าย" ตรงที่ **ซื้อสินค้า = เข้าสต็อก + นับสต็อก** (ลง 114102) · ค่าใช้จ่ายทั่วไปลง 5xxxxx ตรงๆ ไม่เข้าสต็อก.

### 1) บันทึกซื้อ (base) — art 5988274
- **ฟอร์ม:** ชื่อคู่ค้า · รหัสสินค้า/จำนวน · ราคา/หน่วย (ใส่สูตร `=5000/10` ได้) · วันที่ชำระ · ช่องทาง.
- **สถานะ:** "ยังไม่ชำระ (ตั้งหนี้)" หรือ "ชำระเงิน" → **อนุมัติบันทึกซื้อสินค้า** → เข้าสต็อกทันที.
- **🔴 การลงบัญชี** (ภาพ PV จริง · ซื้อ 50 หน่วย @1,000 = 50,000 + VAT 7%):
  - Dr **114102 สินค้าสำเร็จรูป** 50,000 · Dr **115402 ภาษีซื้อยังไม่ถึงกำหนด** 3,500 · Cr **111301 ธนาคาร** 53,500 *(ตั้งหนี้ → Cr เจ้าหนี้แทน)*.
- **สต็อกติดลบ (ขายก่อนของเข้า):** ระบบตัดต้นทุนย้อนหลังในเอกสารเดียว: Dr **510103 ต้นทุนขาย** / Cr **114102**.

### 2) จ่ายมี fee + กำไร/ขาดทุน FX — art 13723210 ⭐ สำคัญกับเรา (ซื้อจีน ¥)
- **ใช้เมื่อ:** จ่ายเจ้าหนี้สกุลต่างประเทศ + เรทวันจ่าย ≠ เรทวันบันทึก + มีค่าธรรมเนียม.
- **ขั้นตอน:** บันทึกซื้อ "ตั้งหนี้ก่อน" → ชำระเงินแบบ **"ขั้นสูง"** → ติ๊ก "ค่าธรรมเนียม/รายการปรับปรุง" · แต่ละบรรทัดเลือกทิศทาง (เพิ่ม/หักเงินที่ต้องจ่าย) + บัญชี:
  - fee → "เพิ่มเงินที่ต้องจ่าย" · **530501 ค่าธรรมเนียมธนาคาร** (Dr)
  - FX gain → "หักเงินที่ต้องจ่าย" · **420103 กำไร FX** (Cr · จ่ายจริง < หนี้)
  - FX loss → "เพิ่มเงินที่ต้องจ่าย" · บัญชีขาดทุน FX (Dr · จ่ายจริง > หนี้)
- **🔴 สูตร gain/loss (ตัวอย่าง USD):** หนี้ตั้งต้น 1,000×32.4736 = 32,473.60 · จ่ายจริง 1,000×32.4616 = 32,461.60 → ผลต่าง **+12.00 = กำไร FX**.
- **การลงบัญชี:** Dr เจ้าหนี้ (หนี้เดิม) · Dr 530501 fee · Cr ธนาคาร (เงินจริงไหลออก) · Cr 420103 FX gain *(หรือ Dr ขาดทุน FX)*.

### 3) ซื้อแบบจ่ายมัดจำ / ตัดมัดจำ — art 5781961
- **Flow 3 สเต็ป:** (1) PO อนุมัติ (2) จ่ายมัดจำ → Dr **115104** / Cr ธนาคาร (**ลบรายการสินค้าออก** เพราะของยังไม่มา) (3) รับของ+จ่ายส่วนที่เหลือ **ตัดมัดจำด้วยบรรทัดค่าติดลบ**:
  - บรรทัดสินค้า: 114102 · 2×500 = 1,000 · บรรทัดตัดมัดจำ: 115104 · ราคา/หน่วย **−300** → ฐานภาษี 700 · VAT 49 · **จ่ายสุทธิ 749**.
- **Gotcha:** สเต็ปมัดจำห้ามใส่สินค้า · ตัดมัดจำใช้จำนวนติดลบ · ออกใบกำกับภาษีแยก (มัดจำ 1 / รับของ 1).

## 3️⃣ บันทึกค่าใช้จ่าย + รับใบเสร็จ/ภาษีซื้อ + จ่าย ภพ.30

### 1) บันทึกค่าใช้จ่ายค่าบริการทั่วไป (EXP · form NE003) — art 5784400
- **ฟิลด์:** ผู้ขาย · วันที่ออก · บัญชีค่าใช้จ่าย · VAT · เลข+วันที่ใบกำกับภาษี · วันบันทึกภาษีซื้อ · งวดภาษี · "ชำระโดย"(สด/ธนาคาร) หรือ **"ยังไม่ชำระ (ตั้งหนี้)"** · เช็คบ็อกซ์ WHT.
- **สถานะ:** ร่าง → รออนุมัติ → **อนุมัติ** → รอชำระ/ชำระแล้ว · เลข EXP → journal PV · เชื่อ = 2 เล่ม (UV วันตั้งหนี้ + PV วันจ่าย).
- **🔴 การลงบัญชี (จ่ายสด · PV จริง · ค่าบริการ 1,000 + VAT 7% + WHT 3%):**

  | GL | บัญชี | Dr | Cr |
  |---|---|---|---|
  | 510104 | ต้นทุนการให้บริการ | 1,000 | |
  | **115402** | **ภาษีซื้อยังไม่ถึงกำหนด** | 70 | |
  | 111301 | ธนาคาร (SCB) | | 1,040 |
  | 215204 | ภ.ง.ด.53 ค้างจ่าย | | 30 |
  → เงินจ่ายจริง = 1,000 + VAT 70 − WHT 30 = **1,040**.
- **จ่ายเชื่อ:** UV → Dr ค่าใช้จ่าย + Dr 115402 / Cr เจ้าหนี้ · PV → Dr เจ้าหนี้ + Dr WHT / Cr ธนาคาร.
- 🔴 **จุดสำคัญ:** VAT ไป **115402 (ยังไม่ถึงกำหนด) เสมอ** แม้จ่ายสด — ยังไม่เข้ารายงานภาษีซื้อ **จนกว่าจะ "รับใบเสร็จ"** (ข้อ 2 · mirror ฝั่งขาย 215102→215101).

### 2) 🔴 กดรับใบเสร็จ + ลงทะเบียนใบกำกับภาษี — art 6171668 (CRITICAL)
- **เมื่อไหร่:** ได้รับใบเสร็จ+ใบกำกับภาษีซื้อจริงจากคู่ค้า → ยืนยันรับ + ลงทะเบียน (ย้ายภาษีซื้อจากพัก→ถึงกำหนด → เข้ารายงานภาษีซื้อ).
- **ขั้นตอน:** รายจ่าย > บันทึกค่าใช้จ่าย > ดูทั้งหมด → แท็บ **"รอรับใบเสร็จ"** → dropdown **"รับใบเสร็จ" → "รับใบเสร็จและใบกำกับภาษี"** → pop-up กรอก เลข+วันที่ใบกำกับ+วันบันทึกภาษีซื้อ → บันทึก → ย้ายไปแท็บ **"รับใบเสร็จแล้ว"**.
- **🔴 การลงบัญชี:** (บทความไม่โชว์ journal ตรงๆ · แต่กลไก PEAK ยืนยัน) **Dr 115401 ภาษีซื้อ / Cr 115402 ภาษีซื้อยังไม่ถึงกำหนด** (ย้ายพัก→ขอคืนได้ · mirror 100% ของ 215102→215101 ฝั่งขาย). 🔴 ยังไม่มี screenshot ยืนยันเลข Dr/Cr.
- **Gotcha:** ทำได้ทีละเอกสาร (ยังไม่รองรับหลายใบพร้อมกัน).

### 3) จ่ายชำระ ภ.พ.30 (ไม่ใช้ PEAK Tax) — art 5796439
- **ฟิลด์:** ผู้ขาย = **กรมสรรพากร** · บรรทัด = **"เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ"** · **ภาษี = "ไม่มี"** (สำคัญ) · ช่องทางจ่าย → อนุมัติ.
- **🔴 การลงบัญชี:** **Dr เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ / Cr 111301 ธนาคาร** — เอกสารนี้แค่**ล้างยอดเจ้าหนี้ที่ตั้งไว้** ไม่แตะ 215101/115401 โดยตรง.
- ⚠️ การ offset **215101 ภาษีขาย vs 115401 ภาษีซื้อ → เจ้าหนี้สรรพากร ภพ.30** = ทำที่ **ขั้นปิดงวด VAT แยก** (ไม่อยู่ในบทความนี้ · 🔴 gap).
- **input > output (ขอคืน/เครดิตยกไป):** บทความไม่ระบุ · 🔴 gap ต้องหาเพิ่ม.

## 4️⃣ หัก ณ ที่จ่าย (WHT/ภงด) + เงินเดือน

### 6.1 WHT หลายอัตรา (base) — art 5775178
- **ใช้:** ค่าใช้จ่าย 1 ใบมี WHT หลายอัตรา (บริการ 3% + ค่าเช่า 5%). ตารางรายการมีคอลัมน์ **"หัก ณ ที่จ่าย"** ท้ายสุด (เลือกอัตรา/แบบ ภงด ต่อบรรทัด).
- ตัวอย่าง: 510104 บริการ 2,000 · VAT 7% · WHT 60 (3%) | 530201 ค่าเช่า 10,000 · ไม่มี VAT · WHT 500 (5%).
- **การลงบัญชี:** Dr ค่าใช้จ่าย + Dr input VAT · **Cr ภงด.X ค้างจ่าย (215201/203/204)** ตามอัตรา · Cr ธนาคาร (สุทธิหลังหัก).

### 6.2 WHT ออกแทนคู่ค้า (จ่ายแทนไปก่อน) — art 6046050 ⭐ journal เต็ม
- **ใช้:** คู่ค้าขอให้เราออก WHT ไปก่อน (จ่ายคู่ค้า **เต็ม ไม่หัก**) แล้วคู่ค้าโอนคืน → ตั้งเป็น **ลูกหนี้** ไม่ใช่หักจากยอดจ่าย.
- **ขั้นตอน:** ชำระเงินแบบ "ขั้นสูง" → ติ๊ก "ค่าธรรมเนียม/ปรับปรุง" → "เพิ่มเงินที่ต้องจ่าย" · บัญชี **113301 ลูกหนี้อื่น** · จำนวน = ยอด WHT (600) → คู่ค้าโอนคืน → ออกใบเสร็จรับเงินผัง 113301 ล้างลูกหนี้.
- **🔴 การลงบัญชี (PV จริง · ค่าบริการ 20,000 + VAT 7% + WHT 3%):**

  | GL | บัญชี | Dr | Cr |
  |---|---|---|---|
  | 510103 | ต้นทุนขายสินค้า | 20,000 | |
  | 115402 | ภาษีซื้อยังไม่ถึงกำหนด | 1,400 | |
  | 113301 | **ลูกหนี้อื่น** | **600** | |
  | 111301 | ธนาคาร (SCB) | | 21,400 |
  | 215204 | ภ.ง.ด.53 ค้างจ่าย | | 600 |
  → ต่างจากปกติ: Cr ธนาคาร **21,400 เต็ม** (ปกติ 20,800) + Dr 113301 600 รอคู่ค้าคืน · แต่ยัง Cr 215204 600 (เรายังต้องนำส่งสรรพากร).

### 6.3 นำส่งภาษี ภ.ง.ด.1/3/53 ให้สรรพากร — art 5981683 ⭐ journal เต็ม
- **ใช้:** ล้างยอด WHT ค้างจ่ายที่สะสม ตอนนำส่งกรมสรรพากร. ผู้ขาย = **กรมสรรพากร** · แต่ละบรรทัด = บัญชีค้างจ่าย (ไม่มี VAT/WHT).
- **🔴 การลงบัญชี (PV จริง):** Dr **215201** 750 · Dr **215203** 2,000 · Dr **215204** 2,500 / **Cr 111301 ธนาคาร** 5,250.
- **หลักการ:** Dr ค้างจ่าย (ล้างหนี้) / Cr ธนาคาร → ยอดค้างจ่ายกลับเป็น 0 (dashboard "ภงด.X ค้างจ่าย" = ยอดคงเหลือ GL 215xxx).

### 6.4 บันทึกจ่ายเงินเดือน (ไม่ใช้ PEAK Payroll) — art 5668442
- **ใช้:** เงินเดือน + ประกันสังคม + ภงด.1 ใน 1 ใบ · ผู้ขาย = **"พนักงาน"** (ห้ามใส่ชื่อจริง กันคนเห็นเงินเดือนรายคน) · รายการหัก = ค่าติดลบ.
- **🔴 การลงบัญชี (ตัวอย่าง เงินเดือน 50,000):**

  | GL | บัญชี | Dr | Cr |
  |---|---|---|---|
  | 530101 | เงินเดือน ค่าจ้าง | 50,000 | |
  | 530104 | เงินสมทบ ปกส.+กองทุนทดแทน (นายจ้าง) | 750 | |
  | 215501 | ประกันสังคมค้างจ่าย (ลูกจ้าง 750 + นายจ้าง 750) | | 1,500 |
  | 215201 | ภ.ง.ด.1 ค้างจ่าย | | 800 |
  | 111301 | ธนาคาร | | 48,450 |
  → ต้องชำระจริง 48,450 (= 50,000 + 750 − 1,500 − 800... สุทธิที่จ่ายพนักงาน 48,450).
- **Gotcha:** ขาด/ลา/สาย = บรรทัดค่าติดลบ · เดือนถัดไปใช้ "คัดลอก" · ปกส.+ภงด.1 ที่ Cr ค่อยนำส่งด้วยวิธี 6.3.

## 5️⃣ ใบกำกับภาษีซื้อ + ใบรวมจ่าย + รับใบลดหนี้/เพิ่มหนี้

> 🔑 แกนหลัก = สลับ **115402 (พัก) ↔ 115401 (ขอคืนได้)** เมื่อได้ใบกำกับภาษีตัวจริง.

### 1) ใบกำกับภาษีซื้อ (Purchase Tax Invoice) — ลงทะเบียน input VAT · art 5848208
- **ใช้:** ตอนบันทึกซื้อ VAT ตั้งเป็น 115402 ไว้ก่อน · ได้ใบกำกับจริง → "ลงทะเบียน" ย้ายเป็น 115401 (ขอคืนได้) + เข้ารายงานภาษีซื้องวดนั้น.
- **เมนู:** รายจ่าย > ใบกำกับภาษีซื้อ · แท็บ "รอรับใบกำกับภาษี" (มีตัวเลขค้าง) / "รับใบกำกับภาษีแล้ว".
- **2 วิธี:** แท็บรอรับ → "+ ใบกำกับภาษี" ท้ายแถว · หรือเปิดเอกสาร EXP → "เพิ่มใบกำกับภาษี".
- **ฟอร์ม 4 ช่อง:** เลขที่ใบกำกับ · วันที่ใบกำกับ · **วันที่บันทึกภาษีซื้อ** (เลือกได้ = วันที่ VAT เข้ารายงาน) · งวดภาษี.
- **🔴 การลงบัญชี (สมุด "รายวันทั่วไป" · JVTX · VAT 70):** **Dr 115401 ภาษีซื้อ 70 / Cr 115402 ภาษีซื้อยังไม่ถึงกำหนด 70** (ยอดสินค้า/ค่าใช้จ่ายไม่ขยับ).
- **Gotcha:** PEAK กันลงทะเบียนซ้ำอัตโนมัติ (คู่ค้า+ชนิด VAT).

### 2) ใบรวมจ่าย (Combined Payment · CPN) — art 5841630  = **mirror ใบวางบิลของเรา**
- **ใช้:** เอาเอกสารซื้อ/EXP ที่ยังไม่จ่าย หลายใบของคู่ค้าเดียวกัน มารวมจ่ายทีเดียว. รวมได้: EXP · บันทึกซื้อ · รับใบลดหนี้(อ้างอิง) · รับใบเพิ่มหนี้(อ้างอิง) · ใบจ่ายมัดจำ.
- **ขั้นตอน:** เลือกผู้ติดต่อ → ติ๊กเอกสารที่รวม → ระบุวันออก → วันที่+วิธีชำระ → อนุมัติ.
- **🔴 เงื่อนไข:** ต้อง **สกุลเงินเดียวกัน + คู่ค้าเดียวกัน** เท่านั้น.
- **การลงบัญชี:** (ไม่โชว์ journal · โมเดล) Dr **212101 เจ้าหนี้การค้า (เคลียร์รวมทุกบิล)** / Cr **111301 ธนาคาร** [+ Cr WHT ถ้ามี]. ตัวใบรวมจ่าย = workflow จับกลุ่ม · journal ตัดหนี้เกิดตอน "อนุมัติ+ชำระ".

### 3) รับใบลดหนี้ (Credit Note · CNX · อ้างอิงเอกสารเดิม) — art 6324600
- **ใช้:** ผู้ขายออกใบลดหนี้ (ชำรุด/คืนของ/ลดราคา) → ลดยอดหนี้/ต้นทุน + ปรับภาษีซื้อลง. อ้างอิงได้ 3 ชนิด (มัดจำ/ซื้อ/EXP · **ไม่รวม PO**).
- **ฟิลด์:** สาเหตุลดหนี้ (บังคับ) · ลดทุกรายการ/เฉพาะรายการ · **มีคืนสินค้าหรือไม่** (คืน = สต็อกลด auto).
- **🔴 การลงบัญชี (สมุด "รายวันรับ" · RV · ตัวอย่าง 1,400 + VAT 98 = 1,498):**
  - **กรณี A บิลเดิมยังไม่จ่าย:** Dr **212101 เจ้าหนี้** 1,498 / Cr **114102 สินค้า** 1,400 · Cr **115402 ภาษีซื้อ** 98.
  - **กรณี B บิลเดิมจ่ายแล้ว (ผู้ขายคืนเงิน):** Dr **111301 ธนาคาร** 1,498 / Cr 114102 1,400 · Cr 115402 98.
  → ทั้ง 2 กรณี Cr สินค้า+ภาษีซื้อ (กลับสต็อก/ลดต้นทุน) · ต่างแค่ Dr (หนี้ vs เงินคืน).
- **Gotcha:** 🔴 **ระบบไม่ปรับ WHT (50 ทวิ) ให้ — ต้องแก้เอง** · เอกสารที่หักมัดจำแล้ว ออกลดหนี้ไม่ได้.

### 4) รับใบเพิ่มหนี้ (Debit Note · DBNX · อ้างอิงเอกสารเดิม) — art 6324698
- **ใช้:** ผู้ขายเรียกเก็บเพิ่ม/รับของเข้าสต็อกเพิ่ม → เพิ่มยอดหนี้/ต้นทุน + เพิ่มภาษีซื้อ (กระจกของใบลดหนี้).
- **ฟิลด์:** สาเหตุเพิ่มหนี้ · **มีรับสินค้าเพิ่มหรือไม่** (รับเพิ่ม = สต็อกเพิ่ม auto) · ช่องทางชำระ.
- **🔴 การลงบัญชี (สมุด "รายวันจ่าย" · PV · ตัวอย่างสินค้า 20,000 + VAT 1,400, จ่ายทันที):** Dr **114102 สินค้า** 20,000 · Dr **115402 ภาษีซื้อ** 1,400 / Cr **111301 ธนาคาร** 21,400 *(หรือ Cr 212101 เจ้าหนี้ ถ้า "ยังไม่ชำระ")*.
- **Gotcha:** WHT ไม่ปรับให้เอง (เหมือนใบลดหนี้).

## 💎 สรุปพิมพ์เขียวฝั่งจ่าย (สำหรับสร้าง accounting engine Pacred)

1. **PO = doc-only** ไม่ลงบัญชี → เก็บเป็น status/link เฉยๆ.
2. **ทุกเอกสารซื้อตั้ง VAT พักที่ 115402 (ยังไม่ถึงกำหนด) ก่อนเสมอ** → **"รับใบเสร็จ / ลงทะเบียนใบกำกับซื้อ" = trigger ย้าย 115402 → 115401 (ขอคืนได้)** — pattern deferred-then-recognize เหมือนฝั่งขาย (215102→215101). ต้องมี event `received_receipt` + เก็บเลข/วันที่ใบกำกับ + วันบันทึกภาษีซื้อ.
3. **WHT = 2 จังหวะ:** (1) บันทึกค่าใช้จ่าย → Cr `ภงด.X ค้างจ่าย` (215201/203/204 ตามบุคคล/นิติ) (2) นำส่ง → Dr ค้างจ่าย / Cr ธนาคาร. ต้องมี GL ค้างจ่ายแยกตามแบบ + dashboard ยอดค้าง.
4. **มัดจำจ่าย** = เกิด asset 115104 ตอนจ่าย → **ล้าง** ตอน match บันทึกซื้อ (offset ด้วยบรรทัดค่าติดลบ) — ต้องมี mechanism "เลือกมัดจำมาตัด" + carry ยอดคงเหลือ.
5. **FX (ซื้อ ¥/$):** ตั้งหนี้เรทวันบันทึก → จ่ายเรทวันจ่าย → ผลต่าง = 420103 กำไร FX / บัญชีขาดทุน FX + fee 530501 (ผ่านโหมดชำระ "ขั้นสูง").
6. **ออกแทนคู่ค้า** = ตั้ง 113301 ลูกหนี้อื่น แทนการหักจ่าย + Cr ธนาคารเต็ม → รับคืนล้างลูกหนี้.
7. **เงินเดือน** = 1 เอกสารรวม: Dr 530101 เงินเดือน + Dr 530104 ปกส.นายจ้าง / Cr 215501 ปกส.ค้างจ่าย + Cr 215201 ภงด.1 + Cr ธนาคาร(สุทธิ).
8. **ใบรวมจ่าย (CPN)** = mirror ใบวางบิล (BN) ของเรา · Dr เจ้าหนี้รวม / Cr ธนาคาร.
9. **ใบลดหนี้/เพิ่มหนี้** = ปรับ (เจ้าหนี้ 212101 หรือ ธนาคาร) คู่กับ (สินค้า 114102 + VAT 115402) กลับทิศกัน · แยก 2 กรณี ยังไม่จ่าย(กระทบเจ้าหนี้) vs จ่ายแล้ว(กระทบเงินสด). **เราไม่มี → ต้องเพิ่ม.**
10. 🔴 **WHT/50-ทวิ ทุกเอกสารปรับ-ลด-เพิ่มหนี้ ไม่ auto — ต้องออกแบบให้ปรับเอง.**
