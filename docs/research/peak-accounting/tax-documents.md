## 🏛️ สถาปัตยกรรม 2 LAYER (ค้นพบหลัก — ต้องเข้าใจก่อนอ่านต่อ)

PEAK แยกเป็น 2 เลเยอร์ชัดเจน:

| Layer | ที่อยู่ | หน้าที่ | แตะ GL ไหม |
|---|---|---|---|
| **A · Bookkeeping / GL** | **main PEAK** (peakaccount.com) | ออกเอกสาร revenue/expense → post journal (215101 ภาษีขาย · 115401 ภาษีซื้อ · 215201/203/204 ภงด.ค้างจ่าย) | ✅ ใช่ — ที่เรา capture ไว้แล้ว |
| **B · Tax-filing / form-prep** | **PEAK Tax** (tax.peakaccount.com · แอปแยก) | ดึง tax line-item ที่ post ไว้แล้ว → ประกอบเป็นแบบ ภ.พ.30 / ภ.ง.ด. / ใบ 50-ทวิ → ยื่นสรรพากร | ❌ **ส่วนใหญ่ไม่แตะ** (form-only) |

**🔑 ข้อยกเว้นสำคัญ (ปิด gap):** Layer B มี **2 action ที่เขียน closing-JV กลับเข้า Layer A** —
1. **"บันทึกปิดบัญชี ภ.พ.30"** → JVTX ปิดงวด VAT (§1)
2. **สร้างแบบ ภ.ง.ด. (อนุมัติ + ชำระ)** → PV/EXP ปิดงวด WHT (§3)

ทุกอย่างอื่นใน PEAK Tax (ดึงข้อมูล · คีย์ใบกำกับเอง · สร้างใบ 50-ทวิ · ส่งอีเมล) = **form-only ไม่ post GL** (verbatim ยืนยัน 4 จุด). GL ของภาษีเกิดตอนออกเอกสารต้นทางใน main PEAK แล้ว.

**เข้า PEAK Tax:** (ก) ตรง `tax.peakaccount.com` (ข) จาก PEAK → ไอคอน grid (ศูนย์รวมแอปฯ) มุมขวาบน → PEAK TAX · **สิทธิ์:** "ดูแลการเงินและบัญชี" · **เชื่อมต่อ (ครั้งเดียว · super-admin):** ตั้งค่า → เชื่อมต่อกับ PEAK → auto-pull 3 สาย (ภาษีขาย output VAT · ภาษีซื้อ input VAT · หัก ณ ที่จ่าย WHT).

**นัยต่อระบบเรา:** filing layer ต้อง **"ดึง/อ้างอิง" doc ที่ลงบัญชีแล้ว (SOT) ไม่ใช่คำนวณใหม่** · post GL ปิดงวดแยกจากการ post ตอนออกเอกสาร (ไม่ post ซ้ำ).

# 1) ภ.พ.30 (VAT return) — 🔴 GAP-CLOSER

> **แก่น:** ภ.พ.30 ทำ **2 จังหวะ** — (A) ดึงข้อมูล/กรอกแบบ = form-only · (B) **"บันทึกปิดบัญชี" = สร้าง GL journal ปิดงวดจริง (JVTX)** หักกลบภาษีขาย/ซื้อ ตั้งเจ้าหนี้สรรพากรสุทธิ. **ยืนยันจากภาพ JVTX จริง #JVTX-202602001.**

**เมนู:** ภาษีมูลค่าเพิ่ม > แบบ ภ.พ.30 · **รอบ:** รายเดือน.

### ขั้นตอน 5 step
1. **ดึงข้อมูล** รายการภาษีขาย + ภาษีซื้อ (ข้ามได้ถ้าเปิด Auto Fetch) — *form-only ไม่มี GL*
2. **สร้างแบบ ภ.พ.30 > ถัดไป** → ระบบ validate เตือน "รายการที่ควรแก้ไข" / "แนะนำให้ตรวจสอบ" (ผู้ติดต่อใหม่ · เอกสารหาย · ภาษีซื้อค้าง 6 เดือน = ⏳) → **อนุมัติ**
3. 🔴 **บันทึกปิดบัญชี** → เลือกวันที่ปิดบัญชี ภ.พ.30 > **ยืนยัน** → **สร้าง GL journal (JVTX)** (>10,000 แถว ~30 นาที · ระหว่างรอ พิมพ์/ยื่น/ชำระได้)
4. **ยื่นแบบ + ชำระเงิน:** ปุ่ม "ยื่นแบบฟอร์มแล้ว" + "ชำระเงินแล้ว" (วันที่ · ช่องทาง เงินสด/ธนาคาร/e-Wallet/สำรองจ่าย · ปรับปรุงเศษสตางค์ · อัปโหลดใบเสร็จ) → **สร้าง journal EXP** (จ่ายเงิน ล้างเจ้าหนี้)
5. **ดูการบันทึกบัญชี:** คลิกเลข **JVTX** (ปิดงวด) หรือ **EXP** (จ่าย)

### ฟิลด์ในแบบ ภ.พ.30 (ตามแบบราชการ)
ข้อ1 ยอดขายเดือนนี้ · ข้อ4 ยอดขายที่ต้องเสียภาษี · ข้อ5 ภาษีขายเดือนนี้ · ข้อ6 ยอดซื้อที่มีสิทธิ · ข้อ7 ภาษีซื้อเดือนนี้ · ข้อ8 ต้องชำระ (ถ้า5>7) · ข้อ9 ชำระเกิน (ถ้า5<7) · **ข้อ10 ภาษีที่ชำระเกินยกมา** · ข้อ11 ต้องชำระ · ข้อ12 ชำระเกิน · การขอคืนภาษี.

### 🔴 GL JOURNAL ปิดงวด (ยืนยันจากภาพ #JVTX-202602001)
| เลขบัญชี | ชื่อบัญชี | เดบิต | เครดิต |
|---|---|---:|---:|
| **215101** | ภาษีขาย ภ.พ.30 | 700.00 | — |
| **115401** | ภาษีซื้อ | — | 350.00 |
| **215351** | เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ | — | 350.00 |

- คำอธิบาย: *"บันทึกปิดภาษีซื้อภาษีขาย เข้า เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ แบบภ.พ.30 #PP30-202602"* · สมุด=รายวันทั่วไป · ผู้ติดต่อ=กรมสรรพากร.
- **ตอน "ชำระเงินแล้ว" (step 4) = journal EXP ที่ 2:** `Dr 215351 เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ / Cr เงินสด-ธนาคาร` → ล้างเจ้าหนี้เป็น 0.

### 🔴 กรณี input VAT > output VAT (ภาษีซื้อ > ขาย)
- **ไม่มีปุ่ม "ชำระเงิน"** (ระบบซ่อน เพราะไม่มียอดต้องจ่าย) · ยอดเกินลง **ข้อ9 / ข้อ12 ชำระเกิน**
- 2 ทางเลือก: (1) **เครดิตยกไปเดือนหน้า** ผ่าน **ข้อ10 "ภาษีที่ชำระเกินยกมา"** (ครั้งแรก: แก้ไข > กรอกยอด > ยืนยัน) หรือ (2) **ขอคืน** (การขอคืนภาษี ในแบบ)
- closing journal กรณีนี้: ผลต่างเป็น **ลูกหนี้/เครดิตสรรพากร** แทนเจ้าหนี้.

### เคลมภาษีซื้อข้ามเดือน (ใบกำกับมาช้า)
- ใบกำกับซื้อลงวันที่เดือนก่อน มาเคลมในแบบเดือนหลัง — **ได้ถึง 6 เดือน** นับจากเดือนถัดจากวันในใบกำกับ (เช่นใบกำกับ 03/12/2024 เคลมได้ถึง 06/2025)
- flow: สร้างแบบ > เลือกงวด > ตรวจภาษีขาย > ถัดไป > **เลือกรายการภาษีซื้อเพื่อเคลม** (ฟิลเตอร์ เดือนนี้/เดือนก่อน · ปุ่ม "เพิ่มใบกำกับ") > บันทึกปิดภาษี
- 🔴 **เขียน JVTX กลับเข้า PEAK** เช่นกัน (verbatim: *"ระบบก็จะมีการบันทึกบัญชีกลับไปยังระบบ PEAK ให้เรียบร้อยด้วยเอกสาร JVTX"*).

### Gotcha
- แก้เอกสารหลังสร้าง ภ.พ.30 = ต้อง **ยกเลิกแบบก่อน** (ไล่จากเดือนล่าสุดย้อนขึ้น · Dropdown > ยกเลิก = reverse JV) แล้วสร้างใหม่ · เอกสารถูก **ล็อกแก้ไม่ได้** เมื่อมีแบบ ภ.พ.30 คลุม · รวมหลายแบบจ่ายทีเดียวไม่ได้ · เช็คจ่ายไม่รองรับตรงๆ (เลือกธนาคารแทน).
- **e-filing:** PEAK เตรียมตัวเลข → ผู้ใช้เอาไปยื่นเว็บกรมสรรพากรเอง (ไม่ยิงตรง) · PEAK mark "ยื่นแล้ว" + บันทึกบัญชี.

# 2) ข้อมูลใบกำกับภาษี ขาย/ซื้อ ใน PEAK Tax — form-only (data layer)

> PEAK Tax มี 2 ทางได้ข้อมูลภาษี: **(หลัก) ดึง auto จาก main PEAK** · **(เสริม) คีย์เอง** สำหรับเอกสารที่ไม่ได้ออกเป็น IV/EXP ในบัญชีหลัก (เช่นบันทึกผ่าน JV). **ทั้ง 2 ทาง form-only ไม่ post GL.**

### สร้างข้อมูลใบกำกับภาษี**ขาย** เอง (5907708)
- เมนู: ภาษีมูลค่าเพิ่ม > รายการภาษีขาย > สร้างรายการภาษีขาย
- ฟอร์ม: toggle พื้นฐาน/ขั้นสูง · ประเภท **ใบกำกับภาษี / ใบเพิ่มหนี้ / ใบลดหนี้ / เอกสารขาย(ไม่ใช่ใบกำกับ)** · ผู้ติดต่อ (ดึงจากฐาน PEAK) · วันที่ · เลขที่ใบกำกับ · งวดภาษี · **มูลค่ายกเว้น / 0% / 7% / VAT (auto แก้ได้) → รวมภาษี** · อ้างอิง · แนบไฟล์ · ยืนยัน
- 🔴 **GL: form-only** — verbatim *"สร้าง…ด้วยตนเองที่ PEAK Tax จะไม่มีการลงบันทึกบัญชี"*
- สถานะ: รอยื่นภาษี → ยื่นแล้ว MM/YYYY · **พิมพ์เอกสารส่งลูกค้าไม่ได้** (ต้องทำใน main PEAK)

### สร้างข้อมูลใบกำกับภาษี**ซื้อ** เอง (13437404)
- เมนู: ภาษีมูลค่าเพิ่ม > รายการภาษีซื้อ > สร้างรายการภาษีซื้อ
- ฟอร์ม: ประเภท ใบกำกับ/ใบเพิ่มหนี้/ใบลดหนี้ · ผู้ติดต่อ · วันที่ · เลขที่ · งวดภาษี · **มูลค่า 7% / VAT (auto แก้ได้)** · อ้างอิง · แนบไฟล์ · ยืนยัน
- ⚠️ **ภาษีซื้อที่บันทึกผ่าน JV (ไม่ใช่ EXP) ระบบไม่ดึงมา** → ต้องคีย์เองกรณีนี้
- 🔴 **GL: form-only** — verbatim *"จะไม่มีการบันทึกกลับรายการบัญชีภาษีซื้อ หรือการบันทึกบัญชีใดๆ บน PEAK เลย"*

### "นำข้อมูลออก" ในเมนูสร้างแบบ ภ.พ.30 (6103438)
- ⚠️ **ชื่อชวนเข้าใจผิด — "นำออก" = ลบรายการออกจากร่าง ภ.พ.30 ไม่ใช่ export ไฟล์** (ปุ่ม 🗑 นำออกทุกรายการ)
- ใช้ตอนอยากตรวจ/เลือกเอกสารทีละรายการ: ลบทั้งหมด → เพิ่มกลับด้วย +เพิ่มรายการ
- หน้าจอ ภ.พ.30: คอลัมน์ เลขที่ใบกำกับ · วันที่ · ผู้ติดต่อ · ยกเว้น · 0% · 7% · ภาษี · ใบลดหนี้(CN)=ยอดติดลบในวงเล็บ · footer #PP30-YYYYMM

**Gotcha รวม:** (1) PEAK Tax พิมพ์ใบกำกับจริงให้ลูกค้าไม่ได้ (2) ใบที่คีย์เองไม่กระทบบัญชี → ระวังยอดภาษีในงบ (main PEAK) กับยอดที่ยื่น ภ.พ.30 (PEAK Tax) ไม่ตรงถ้ามีคีย์มือ (3) รายการที่บันทึกผ่าน JV ในบัญชีหลักไม่ไหลมา PEAK Tax เอง.

# 3) แบบ ภ.ง.ด. (WHT filing) — 🔴 มี GL 2 จังหวะ

> **ต่างจากใบ 50-ทวิ:** การ **"สร้างแบบ ภ.ง.ด."** (1/3/53) = มี GL journal (ไม่ใช่ form-only) — ยืนยันจากภาพ JV จริง PV-202510019 (ภงด53) + PV-202511001 (ภงด1).

### 2 จังหวะ GL (ทุกแบบ ภ.ง.ด.)
1. **ตอนปิดบัญชี/อนุมัติแบบ** — reclass liability: `ภ.ง.ด.X ค้างจ่าย` → `เจ้าหนี้สรรพากร ภ.ง.ด.X รอชำระ` (สร้างเป็นเอกสาร EXP · ผู้ขาย=กรมสรรพากร) · verbatim *"สมุดรายวันจะปิดปรับปรุงรายการ ภ.ง.ด. X ค้างจ่าย เป็นเจ้าหนี้สรรพากร ภ.ง.ด. X รอชำระ"*
2. **ตอน "ชำระเงินแล้ว"** — JV จ่าย: `Dr เจ้าหนี้สรรพากร ภ.ง.ด.X รอชำระ / Cr ธนาคาร (111301)`

### เลขบัญชีจริง (SOT)
| แบบ | ค้างจ่าย (booked ตอนบันทึกค่าใช้จ่าย) | เจ้าหนี้สรรพากร ... รอชำระ (reclass ตอนปิดแบบ) |
|---|---|---|
| ภ.ง.ด.1 (เงินเดือน) | 215201 | **215352** |
| ภ.ง.ด.3 (บุคคล) | 215203 | 215353 *(คาด · pattern)* |
| ภ.ง.ด.53 (นิติ) | 215204 | **215355** |

> **ครบวงจร WHT:** บันทึกค่าใช้จ่าย → **Cr 215201/203/204 ค้างจ่าย** → สร้างแบบ ภ.ง.ด. → **reclass เป็น 21535X รอชำระ** → ชำระ → **Dr 21535X / Cr ธนาคาร**. **ยอดในแบบ = ยอด WHT ที่ Cr ค้างจ่ายไว้** ✓

### 3 แบบ (เมนู: ภาษี หัก ณ ที่จ่าย)
- **ภ.ง.ด.53 (นิติ · 5848508):** ดูภาพรวม > เลือกงวด > สร้างแบบ → ดึงรายการหัก ณ ที่จ่าย(นิติ)อัตโนมัติ (เพิ่ม/ลบได้) > อนุมัติ/ร่าง · ตัวอย่าง: จ่าย 8,355 → Dr 215355 / Cr 111301 SCB
- **ภ.ง.ด.3 (บุคคล · 5846736):** flow เหมือน 53 เป๊ะ · ผู้ถูกหัก=บุคคลธรรมดา · ปุ่ม Re-data ดึง "แยก" (split allocation) สำหรับแบบย้อนหลัง
- **ภ.ง.ด.1 (เงินเดือน · 6208782):** 🔴🔴 **ต้องสร้างใบหัก ณ ที่จ่ายเองก่อน** (ประเภท 40(1)1) — **PEAK ยังไม่รองรับ auto-pull จาก PEAK Payroll → PEAK Tax** · ตัวอย่าง PV-202511001: จ่าย 1,500 → Dr 215352 / Cr 111301 SCB

### ดึงข้อมูลหัก ณ ที่จ่าย (5846712) — form-prep ไม่มี GL
- Path: ภาษี หัก ณ ที่จ่าย > ใบหัก ณ ที่จ่าย > ปุ่ม "ดึงข้อมูล" (ตามช่วงวันที่ / ตามเลขเอกสาร) · หรือ auto-pull real-time
- sync-state 3 สี: **แดง**=แก้ใน main PEAK · **ส้ม**=แก้ใน PEAK Tax · **🗑**=ถูกลบใน main PEAK → ปุ่ม [แก้ไข] ถูกปิดจนกด **[อัปเดต]** reconcile ก่อน · dedup (ดึงแล้วไม่ดึงซ้ำ)

### e-filing + Gotcha
- ดาวน์โหลด .txt/PDF → อัปโหลด **RD Prep** (โปรแกรมกรมสรรพากร) หรือยื่นเว็บ RD · ยืนยัน "ยื่นแบบแล้ว" + "รับใบเสร็จแล้ว" + แนบใบเสร็จ
- แถบเตือนส้ม (lookback 6 เดือน หา vendor เอกสารหาย) + 🪄 สรุปรายการควรตรวจสอบ · จ่ายด้วยเช็คไม่ได้ · สร้างครั้งแรกของเดือน = auto "ยื่นปกติ" · **เดือนไหนไม่มีการหัก = ไม่ต้องยื่น ภงด3/53**

# 4) ใบหัก ณ ที่จ่าย (หนังสือรับรอง 50-ทวิ) — form-only

> 🔴 **ทั้งหมว form-only ไม่มี GL** (GL ของ WHT ลงตอนบันทึกค่าใช้จ่ายใน main PEAK แล้ว: Dr ค่าใช้จ่าย / Cr 215201-204 ค้างจ่าย) · ใบ 50-ทวิ = แค่ **พิมพ์แบบราชการให้ผู้ถูกหัก**. **ใช้เป็น reference ฟิลด์ให้ `/r/[token]/wht-form` ของเราได้เลย.**

### สร้างใบหัก ณ ที่จ่าย ด้วยตนเอง (5910658)
- ใช้: ออกใบ 50-ทวิ standalone (จ่าย+หักนอกระบบ แต่ต้องออกหนังสือรับรอง)
- Path: PEAK Tax > ภาษี หัก ณ ที่จ่าย > ใบหัก ณ ที่จ่าย > สร้างใบหัก > **"สร้างแบบละเอียด"**
- **ฟอร์ม 50-ทวิ** ("หนังสือรับรองการหักภาษี ณ ที่จ่าย มาตรา 50 ทวิ"):
  - **เล่มที่ / เลขที่** (รันเลขเอง · doc no. เช่น `#WT53-20251000001` = WT + ประเภทภงด + ปี + running)
  - **ผู้มีหน้าที่หัก** (=บริษัทเรา · auto ชื่อ+เลข 13 หลัก+ที่อยู่ จากข้อมูลกิจการ)
  - **ผู้ถูกหัก** (ชื่อ+เลข 13 หลัก+ที่อยู่ · **ดึงจากฐานผู้ติดต่อ PEAK**)
  - **checkbox ประเภท ภ.ง.ด. 7 ตัว:** ภ.ง.ด.1ก / 1ก พิเศษ / 2 / 3 / 2ก / 3ก / 53
  - **ตารางเงินได้** (ตามมาตรา 40): ประเภทเงินได้ | วันที่จ่าย | จำนวนเงินที่จ่าย | ภาษีที่หัก — แถว: 1.เงินเดือน 40(1) · 2.ค่าธรรมเนียม/นายหน้า 40(2) · 3.ค่าลิขสิทธิ์ 40(3) · 4.(ก)ดอกเบี้ย/(ข)เงินปันผล 40(4) · 5.มาตรา 3 เตรส · 6.อื่นๆ
  - **รวม (auto sum)** + **รวมภาษี (ตัวอักษร)** = baht-text
  - **เงื่อนไขการหัก** dropdown: หัก ณ ที่จ่าย / ออกให้ตลอดไป / ออกให้ครั้งเดียว · อ้างอิง 1 / 2 (35 ตัวอักษร)
- **popup กรอกเงินได้:** ประเภท · คำอธิบาย(auto แก้ได้) · จำนวนที่จ่าย · อัตราภาษี(preset/custom) · ภาษี(auto-calc/กรอกเอง)
- **ปุ่ม:** ยกเลิก / ร่าง / อนุมัติ · หลังสร้าง: **ส่งอีเมล / พิมพ์**
- 🔴 GL: ไม่มี — verbatim *"การสร้างใบหัก ณ ที่จ่าย ด้วยตนเอง ที่ PEAK Tax จะไม่มีการลงบัญชี"*
- ⚠️ ผู้ติดต่อต้องมี **ที่อยู่** ไม่งั้นฟอร์มไม่ขึ้น → แก้ที่ผู้ติดต่อ → Ctrl+F5 → สร้างใหม่

### ใบหัก ณ ที่จ่าย ทั้งปีให้พนักงาน — ม.50 ทวิ ประจำปี (5858871)
- ออก 50-ทวิ สรุปเงินได้+ภาษีทั้งปีให้พนักงาน (คู่กับ ภ.ง.ด.1ก ปลายปี) · ประเภท ภงด=**1ก** · เงินได้=**40(1)1 เงินเดือน**
- 🔴 GL: ไม่มี — form-only แสดงที่ PEAK Tax เท่านั้น

### 3. โมเดล 2-layer (สำคัญ)
- **Bookkeeping layer** = ออกเอกสาร → post GL ทันที (เราทำแล้ว: revenue/expense)
- **Tax-filing layer** = อ่าน tax line-item (SOT = เอกสารที่ post แล้ว) → ประกอบแบบ → ยื่น · **ไม่ post GL ซ้ำ** ยกเว้น **closing-JV ปิดงวด** (JVTX VAT + reclass WHT)
- filing layer ต้องมี **sync-state** (source แก้/ลบ → lock จน reconcile) + **dedup** (ดึงแล้วไม่ซ้ำ)

### 6. e-filing
- PEAK เตรียมตัวเลข → export .txt/PDF → **RD Prep** (WHT) / เว็บกรมสรรพากร (VAT) → user ยื่นเอง → PEAK mark "ยื่นแล้ว" + บันทึกบัญชี. **ไม่ยิง API ตรงกับสรรพากร.**
- **ช่องปรับปรุงระบบเรา:** เก็บ log การส่งใบ 50-ทวิ ทางอีเมล (PEAK ยังไม่เก็บ).
