## 💠 The Document Model — ฝั่งรับเงิน (Revenue / เอกสารขาย)

> คอลเลกชัน `เอกสารขาย - รับเงิน` = https://intercom.help/peak/en/collections/3872163

PEAK มองงานขาย = **สายเอกสารที่ไหลต่อกัน** แต่ละใบสร้าง**ต่อจาก**ใบก่อนหน้าได้ (carry ข้อมูล) และแต่ละใบมี**สถานะ** + บางใบ**ลงบัญชี (debit/credit)** บางใบไม่ลง:

```
ใบเสนอราคา ──┐  (ยังไม่ลงบัญชี · เสนอราคา/เงื่อนไข)
(Quotation)  │
             ▼
        ใบวางบิล ──────┐   (แจ้งยอดให้ลูกค้าเตรียมจ่าย · รวมหลายใบแจ้งหนี้ได้)
        (Billing Note) │
             │         ▼
             └──►  ใบแจ้งหนี้ ──┐   (ตั้งลูกหนี้ · ลงบัญชี Dr.ลูกหนี้ Cr.รายได้+ภาษีขาย)
                  (Invoice)     │   = ใบส่งของ/ใบกำกับภาษี ได้ในใบเดียว (รวมร่าง)
                       │        ▼
                       │   ใบเสร็จรับเงิน ──┐  (รับเงินจริง · Dr.เงินสด/ธนาคar Cr.ลูกหนี้)
                       │   (Receipt)        │   + หัก ณ ที่จ่าย · = ใบกำกับภาษี ได้ในใบเดียว
                       │        │           ▼
                       │        │      ใบกำกับภาษี (Tax Invoice) — VAT ขาย
                       ▼        ▼
                  ใบลดหนี้ / ใบเพิ่มหนี้ (Credit / Debit Note — ปรับยอดหลังออกเอกสาร)

เสริม: ใบรับเงินมัดจำ (Deposit) — รับเงินก่อนส่งของ · ตัดกับใบแจ้งหนี้/เสร็จภายหลัง
```

**หลักที่ต้องจำ (ต่างจากที่เราเข้าใจ):**
- **ใบเสนอราคา = ยังไม่ลงบัญชี** (แค่เสนอ) · **ใบแจ้งหนี้ = จุดตั้งลูกหนี้** (เริ่มลงบัญชี) · **ใบเสร็จ = จุดรับเงินจริง**.
- PEAK รวมได้: **ใบแจ้งหนี้/ใบกำกับภาษี** ใบเดียว · **ใบเสร็จรับเงิน/ใบกำกับภาษี** ใบเดียว (แล้วแต่จังหวะรับเงิน + ต้องออก VAT เมื่อไหร่).
- **ใบวางบิล** = รวมหลายใบแจ้งหนี้เพื่อวางบิลลูกค้ารายเดือน (ตรงกับ billing-run ของเรา).
- ทุกใบ **แก้ไข/ยกเลิก** มีเงื่อนไขต่างกันตาม**สถานะ** (เช่น แก้ใบแจ้งหนี้ "ก่อน/หลังออกใบเสร็จ" คนละวิธี · หัก ณ ที่จ่ายแก้ทีหลังมีขั้นตอนเฉพาะ).

### 🔎 ยืนยันจากภาพหน้าจอจริง (ไม่ใช่แค่ข้อความ · 2026-07-20)
- **รหัสเอกสาร (document type codes) + stepper บนฟอร์มสร้าง:** ทุกฟอร์มสร้างมีแถบ stepper บนสุดโชว์สายเอกสาร **`ใบเสนอราคา (QO) → ใบแจ้งหนี้ (IV) → ใบเสร็จรับเงิน (RE) → ใบกำกับภาษี (TIV)`** = ลูกค้าเห็นชัดว่าเอกสารต่อยอดไปไหนได้ (จุดที่ active = ใบที่กำลังทำ). รหัสอื่น: ใบวางบิล ใบลดหนี้ ใบเพิ่มหนี้ ใบรับเงินมัดจำ.
- **เมนู รายรับ (dropdown) เรียงตามนี้:** ดูภาพรวม · ใบเสนอราคา · ใบรับเงินมัดจำ · **ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้)** · ใบเสร็จรับเงิน · ใบกำกับภาษีขาย · ใบลดหนี้ · ใบเพิ่มหนี้ · ใบวางบิล · นำเข้าเอกสาร. → **🔑 "ใบแจ้งหนี้ = ใบส่งของ = บันทึกลูกหนี้" = เอกสารใบเดียวกันใน PEAK** (ไม่ได้แยก 3 ใบ — ออกทีเดียวได้ทั้งแจ้งหนี้/ส่งของ/ตั้งลูกหนี้).
- **เลขที่เอกสาร format:** `<CODE>-YYYYMM#####` เช่น `QO-20251200001` (รหัส+ปีค.ศ.+เดือน+running 5 หลัก · running แยกซีรีส์ต่อชนิดเอกสาร) + ช่อง **อ้างอิง** (reference · optional).
- **ฟอร์มสร้าง แบ่งเป็น sections (พับได้ · ติ๊กเขียวเมื่อกรอกครบ):** ① ข้อมูลลูกค้า (ชื่อ/ที่อยู่/เบอร์โทร — ดึงจาก master ผู้ติดต่อ) · ② กลุ่มจัดประเภท · ③ ข้อมูลราคาและภาษี (รวม/แยกภาษี) · ④ **รายการ (line items)**. Header: วันที่ · วันที่ใช้ได้ถึง · โครงการ · สกุลเงิน.
- **🔑 คอลัมน์ line item (สำคัญต่อการลงบัญชี):** สินค้า/บริการ · **บัญชี (GL account ต่อบรรทัด)** · จำนวน · ราคา/หน่วย · ส่วนลด/หน่วย · ภาษี(VAT) · มูลค่าก่อนภาษี · **หัก ณ ที่จ่าย (ต่อบรรทัด)**. → PEAK ลงบัญชีอัตโนมัติจาก "บัญชี" ที่ผูกแต่ละบรรทัด (นี่คือกลไกที่ทำให้ออกเอกสาร = ลงบัญชีเอง). ตั้งค่าได้ว่าจะโชว์คอลัมน์ไหน (ตั้งค่าการแสดงผล) + มี "รูปแบบใหม่/เก่า".
- **💡 สำหรับระบบเรา:** ต้องมี (1) **doc-type code + running series แยกต่อชนิด** · (2) **stepper สายเอกสาร** ให้เห็น convert path · (3) **GL-account ต่อ line item** เพื่อ auto-posting · (4) ใบแจ้งหนี้เดียว = แจ้งหนี้+ส่งของ+ตั้งลูกหนี้.

## 📖 (ตัวอย่างเนื้อหาเต็ม) 1. ใบเสนอราคา (Quotation)

**คืออะไร:** เอกสารที่ผู้ขายออกให้ลูกค้า เพื่อพิจารณาราคา+เงื่อนไข **ก่อนตัดสินใจซื้อ** — **ยังไม่มีการลงบัญชี** (ไม่มี debit/credit).

**ขั้นตอนสร้าง:** รายรับ → ใบเสนอราคา → สร้าง → กรอกรายละเอียด → บันทึกร่าง/อนุมัติ → พิมพ์/ส่งอีเมลให้ลูกค้า.

**ฟิลด์หลัก:**
| ฟิลด์ | ความหมาย |
|---|---|
| ชื่อลูกค้า (ผู้ติดต่อ) | เลือกจากรายชื่อที่มี หรือสร้างใหม่ |
| เลขที่เอกสาร | auto-gen (ตั้งรูปแบบได้ในตั้งค่า) |
| วันที่ออก | default วันนี้ · แก้ได้ |
| วันที่ใช้ได้ถึง (หมดอายุ) | default +7 วัน · ตั้งค่าได้ |
| ประเภทราคา | รวม/แยกภาษี |
| สกุลเงิน | รองรับต่างประเทศ |
| สินค้า/บริการ | ผูกรหัสสินค้า หรือพิมพ์ inline |
| จำนวน · ราคา/หน่วย | |
| ส่วนลด/หน่วย | บาท หรือ % |
| ภาษี (VAT) | 7% / 0% / ไม่มี |
| หัก ณ ที่จ่าย | % (ถ้ามี) |
| ส่วนลดรวม (ท้ายเอกสาร) | |
| หมายเหตุ · แนบไฟล์ · แท็ก | |

**สถานะ:** รออนุมัติ → รอตอบรับ → ตอบรับแล้ว / ปฏิเสธ / ยกเลิก. (สลับ ตอบรับแล้ว↔รอตอบรับ ได้ · เปลี่ยนหลายใบพร้อมกันได้)

**ต่อยอดเป็น:** ใบแจ้งหนี้ · ใบเสร็จรับเงิน · (ใบวางบิล) — carry ข้อมูลจากใบเสนอราคาไป.
