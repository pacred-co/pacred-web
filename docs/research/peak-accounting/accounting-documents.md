## 🏗️ THE GL PIPELINE (ภาพรวม — โครงกระดูกระบบบัญชีทั้งหมด)

```
ผังบัญชี (Chart of Accounts · 6-digit tree)          ← ราก
      │  ทุกเอกสาร (IV/RE/PO/EXP/JV) ผูก GL account ต่อ line
      ▼
สมุดรายวัน (Journal · 5 เล่ม · Dr/Cr)                ← post-on-APPROVAL
      │  อนุมัติ = post GL · ร่าง/รออนุมัติ = ยังไม่เข้า
      ▼
บัญชีแยกประเภท (GL / Ledger รายบัญชี)                ← รวม Dr/Cr ต่อบัญชี → คงเหลือ
      ▼
งบทดลอง (Trial Balance)                              ← พิสูจน์ ΣDr = ΣCr
      ├──────────────┐
      ▼              ▼
งบฐานะการเงิน     งบกำไรขาดทุน + งบกระแสเงินสด
(หมวด 1/2/3)       (หมวด 4/5)
      ▼
ปิดปี: กำไร/ขาดทุนสุทธิ → กำไรสะสม (หมวด 3) [auto ขึ้นปีใหม่]
      ▼
XBRL → ยื่น DBD (กรมพัฒนาธุรกิจการค้า · e-Filing ประจำปี)
```

**เมนู "บัญชี" (top-nav) = ชั้น GL ครบ:** ผังบัญชี · บัญชีรายวัน · บัญชีแยกประเภท · งบทดลอง · งบฐานะการเงิน · งบกำไรขาดทุน · งบกระแสเงินสด · DBD e-Filing · สินทรัพย์.

# 1) ผังบัญชี (Chart of Accounts) — เลข 6 หลัก tree 3 ชั้น

**โครงเลขบัญชี (auto-generated · แก้เลขเองไม่ได้):**
```
D1        D2       D3-D4      D5-D6
[หมวด]   [รอง]    [ย่อย]     [running 01-99]
```
- **D1 = หมวดหลัก:** `1`สินทรัพย์ · `2`หนี้สิน · `3`ส่วนของเจ้าของ/ทุน · `4`รายได้ · `5`ค่าใช้จ่าย
- **D2 = รอง:** เช่น 1=หมุนเวียน · 2=ไม่หมุนเวียน (→ D1D2 = 11/12/21/22…)
- **D3-D4 = ย่อย (ชนิดบัญชี):** 31=ลูกหนี้ · 51=VAT · 52=WHT ค้างจ่าย · 54=VAT ซื้อ
- **D5-D6 = running 01-99** (เกิน 99 → ระบบสร้าง "รอง" ใหม่ให้)

**ตัวอย่างจริง (cross-check ตรงทุกตัวกับ revenue/expense/tax):**
`113101` ลูกหนี้การค้า (1·1·31·01) · `410101` รายได้ขาย (4·1·01·01) · `510103` ต้นทุนขาย (5·1·01·03) · `215101/215102` VAT ขาย ภ.พ.30/รอเรียกเก็บ · `115401/115402` VAT ซื้อ/ยังไม่ถึงกำหนด · `215201-204` WHT ค้างจ่าย · `590000` ภาษีเงินได้ (5·9·00·00) · `111101` เงินสด · `111201/111301` ธนาคารกระแส/ออมทรัพย์ · `212101` เจ้าหนี้การค้า.

**ผังบัญชีมาตรฐาน (default · 5 หมวด):**
| หมวด | ชื่อ | รอง | → งบ |
|---|---|---|---|
| 1 สินทรัพย์ | Assets | 11 หมุนเวียน · 12 ไม่หมุนเวียน | งบฐานะการเงิน |
| 2 หนี้สิน | Liabilities | 21 หมุนเวียน · 22 ไม่หมุนเวียน | งบฐานะการเงิน |
| 3 ส่วนของเจ้าของ | Equity | 31 ทุนจดทะเบียน · 32 สำรอง · 33 กำไรสะสม | งบฐานะการเงิน |
| 4 รายได้ | Revenue | 41 ขาย/บริการ · 42 อื่น | งบกำไรขาดทุน |
| 5 ค่าใช้จ่าย | Expenses | 51 ต้นทุน · … · 59 ภาษีเงินได้ | งบกำไรขาดทุน |

**เพิ่มบัญชี (5630272):** เมนู บัญชี > ผังบัญชี > `+ เพิ่มบัญชี` → เลือก 3 dropdown **หลัก/รอง/ย่อย** → ระบบผูกเลข 6 หลักให้เอง (แก้ไม่ได้) + ชื่อไทย/อังกฤษ + ประเภทเงินได้ภาษี/อัตรา WHT.
- 🔴 **บัญชีย่อย (sub-account) ของ contact/การเงิน สร้างมือไม่ได้** — ระบบ gen ให้อัตโนมัติ: `111101` เงินสด → sub `CSH001` (จากตั้งช่องทางการเงิน) · `113101` ลูกหนี้ → `C00001` (ต่อผู้ติดต่อ 1 ราย) · ธนาคาร → `BSV001/BCA005` (ต่อบัญชี). **บัญชีธนาคาร/เงินสด ต้องสร้างจากเมนู "การเงิน" ไม่ใช่หน้าผังบัญชี** → ระบบสร้าง GL ให้ตาม.

**แก้ไข (5630352):** แก้ได้เฉพาะ **ชื่อบัญชี user-created** · บัญชีระบบ (PEAK ให้มา) = locked · บัญชี custom ที่ถูกใช้ในเอกสารแล้ว = แก้ประเภทภาษีไม่ได้ (ต้องยกเลิกเอกสารก่อน) · เลขบัญชี = auto ตลอด.
- 🟡 **ลบบัญชี:** บทความแยก (ไม่อยู่ใน batch) — แพทเทิร์น PEAK: ลบได้เฉพาะ user-created ที่ยังไม่มี transaction · บัญชีระบบ/มี transaction ลบไม่ได้ (ควรยืนยันรอบหน้า).

# 2) สมุดรายวัน (Journal) — 🔴 post-on-approval (หัวใจ)

### กลไก post-on-approval (ยืนยัน — นี่คือ pattern ที่เราล็อกไว้)
เอกสารทุกโมดูล (IV/RE/PO/EXP/JV) อนุมัติ → ระบบ **สร้าง journal entry อัตโนมัติ** ส่งเข้าสมุดรายวันเล่มที่ตรงประเภท → สถานะ `รออนุมัติ` → กด **"อนุมัติ"** → **post เข้า GL + มีผลต่องบทันที**.

**Lifecycle:** `ร่าง → รอตรวจสอบ → รออนุมัติ → อนุมัติแล้ว` (+ `ยกเลิก`)
| สถานะ | ผลต่อ GL/งบ |
|---|---|
| ร่าง / รอตรวจสอบ | ยังไม่เข้า GL |
| **รออนุมัติ / อนุมัติแล้ว** | **มีผลต่องบทันที** |
| ควรตรวจสอบ | ผิดปกติ (Dr≠Cr / ไม่ผูกบัญชี) → §alert |

### 5 เล่มสมุดรายวัน (double-entry books) + prefix
| Prefix | สมุดบัญชี | ที่มา |
|---|---|---|
| **SV** | รายวันขาย | ใบแจ้งหนี้/ใบกำกับ (revenue) |
| **RV** | รายวันรับ | ใบเสร็จรับเงิน (รับชำระ) |
| **UV** | รายวันซื้อ | รายจ่าย/ใบกำกับซื้อ (มี VAT ซื้อ) |
| **PV** | รายวันจ่าย | บันทึกรายจ่าย/จ่ายเงิน |
| **JV** | รายวันทั่วไป | บันทึกบัญชีมือ (manual) |
| **JVTX** | รายวันทั่วไป | JV ปิดภาษีซื้อ-ขาย (period-close · §3) |
| **JVDP** | รายวันทั่วไป | ค่าเสื่อมสินทรัพย์ (ระบบ auto-post เอง) |

**Journal-detail (GL lines จริงจากภาพ):** เลขที่บัญชี (6 หลัก) + บัญชีย่อย + Dr + Cr — เช่น `520307 ค่ารับรอง Dr 1,990` · `115402 ภาษีซื้อยังไม่ถึงกำหนด Dr 139.30`.

### อนุมัติ batch (3954601)
บัญชี > บัญชีรายวัน > รายวันทั้งหมด > แท็บ **"รออนุมัติ"** → ติ๊ก (≤**100/รอบ**) → แถบล่างโชว์ **Σ Dr/Cr** → "อนุมัติ" → ยืนยัน → post GL ทันที → ย้ายไป "อนุมัติแล้ว". (ต้องมีสิทธิ์อนุมัติ)

### ยกเลิก / กลับรายการ (6052803) — 2 คำสั่งคนละตัว
เมนู **"ตัวเลือก"** บน journal-detail: `แก้ไข · คัดลอก · ยกเลิกเอกสาร · ล็อกเอกสาร · กลับรายการ`
- **ยกเลิกเอกสาร** = soft-cancel (mark "ยกเลิก" · **ไม่ลบ** · ตัดออกจากงบ · audit trail คงอยู่). auto-posted → ยกเลิกตรงได้ แต่ **ต้องตามไปยกเลิกเอกสารต้นทางด้วย** (ให้รายได้/ค่าใช้จ่าย+ภาษี ถูก).
- **กลับรายการ** = reverse (สร้าง JV กลับด้าน Dr↔Cr)
- 🔴 **manual JV ที่อนุมัติแล้ว → ต้อง "กลับไปรออนุมัติ" ก่อน ถึงยกเลิก/แก้ได้** (un-approve 2 ขั้น).

### Alert "รายการที่ควรตรวจสอบ" (9755451)
ระบบดักผิดปกติ → ป้ายส้ม "ควรตรวจสอบ" แทนตัวเลข Dr/Cr:
- **ควรตรวจสอบ** (ผิดชัด): Dr≠Cr · **ไม่เลือกผังบัญชี** · ไม่ใส่จำนวน · ลงด้านเดียว
- **น่าสงสัย** (อาจผิด): รายได้ในสมุดจ่าย · รายจ่ายในสมุดขาย · บัญชีย่อยผิด
- 🔴 **กฎ PEAK: แก้จากเอกสารต้นขั้วเสมอ** (ไม่แก้ที่ journal ตรง) — ใบเสร็จออกแล้ว = ยกเลิก+ออกใหม่.

# 3) โอนปิดภาษี manual (JV) + ทะเบียน WHT ถูกหัก + แก้สมุดรายวัน

### 🔴 บันทึกโอนปิดภาษีขาย-ซื้อ ด้วยมือ (6051515) — cross-check JVTX
ปิดงวด VAT ปลายเดือนทำมือได้ (ถ้าใบกำกับครบ ระบบ auto = JVTX). เมนู บัญชี > สมุดรายวัน > สร้าง "บันทึกบัญชี" · สมุด=**รายวันทั่วไป** · ผู้ติดต่อ=**กรมสรรพากร** · เลข `JV-YYYYMM####`.

**กรณีจ่าย (ภาษีขาย > ซื้อ · เช่น 8,400 vs 6,300 → จ่าย 2,100):**
| เลขบัญชี | ชื่อ | Dr | Cr |
|---|---|---:|---:|
| 215101 | ภาษีขาย ภ.พ.30 | 8,400 | — |
| 115401 | ภาษีซื้อ | — | 6,300 |
| **215302** | เจ้าหนี้สรรพากร | — | 2,100 |

→ จ่ายจริง: สร้าง **EXP** บรรทัด `215302` (ภาษี=ไม่มี) → เคลียร์เจ้าหนี้

**กรณีคืน (ภาษีซื้อ > ขาย · เช่น 7,000 vs 5,950 → คืน 1,050):**
| เลขบัญชี | ชื่อ | Dr | Cr |
|---|---|---:|---:|
| 215101 | ภาษีขาย ภ.พ.30 | 5,950 | — |
| **115405** | ลูกหนี้สรรพากร | 1,050 | — |
| 115401 | ภาษีซื้อ | — | 7,000 |

→ ถ้ายกไปทบเดือนหน้าแทนขอคืน ใช้ **115451 เครดิต ภ.พ.30** แทน 115405 · รับคืนจริง: สร้าง **RT** บรรทัด `115405` → เคลียร์ลูกหนี้

> **🔗 cross-check กับ JVTX auto (tax-documents.md):** concept เหมือนเป๊ะ (zero-out 215101/115401 → land net ที่ control account). ต่างแค่**เลข control:** manual demo = `215302` (generic) · auto JVTX = `215351 เจ้าหนี้สรรพากร ภ.พ.30 รอชำระ` (dedicated) — เป็น flavour เดียวกัน ขึ้นกับผังบัญชีบริษัท. **Pacred: period-close = pure account-transfer JV** (auto ตอนอนุมัติ ภ.พ.30 หรือปุ่มทำมือ).

### ทะเบียนภาษี "ถูกหัก" ณ ที่จ่าย (13630187) — receivable · ไม่ post GL
ทะเบียนใบ 50-ทวิ ที่ **ลูกค้าหักเรา** (เราถูกหัก = receivable). เมนู การเงิน > **"ภาษีถูกหัก ณ ที่จ่าย"** (แท็บคู่กับ "ภาษีหัก ณ ที่จ่าย" ที่เป็น payable) · doc `WTR-`/`WTX-` อ้างอิงเอกสารขาย. `+ หัก ณ ที่จ่าย` → กรอกวันที่/เลขใบหัก/แนบไฟล์ 50-ทวิ → ลงทะเบียน → "ได้รับแล้ว".
- 🔴 **ไม่ post journal** — บัญชี `≈115403 ภาษีถูกหัก ณ ที่จ่าย` (prepaid tax/สินทรัพย์) ถูก Dr ตั้งแต่ออกเอกสารขายแล้ว · ทะเบียนแค่ยืนยันได้รับ certificate (ใช้เครดิตตอนยื่น ภ.ง.ด. นิติปลายปี). **≠ WHT ที่เราหักคู่ค้า (215201-204 ค้างจ่าย · payable).**

### แก้ไขสมุดรายวัน (5950683)
แก้ได้: ผังบัญชี · คำอธิบาย · กลุ่มจัดประเภท (โครงการ/แผนก/สาขา) · แก้ไม่ได้: ลายเซ็นผู้อนุมัติ. 🔴 **อนุมัติแล้วต้อง "กลับไปรออนุมัติ" ก่อน** → แก้ → re-post ตอนอนุมัติซ้ำ.

# 4) งบการเงิน (roll-up จาก GL)

### บัญชีแยกประเภท (GL / Ledger · 5962613)
ยอด+เคลื่อนไหวรายบัญชี. บัญชี > บัญชีแยกประเภท > เลือกช่วงวัน + คลิกบัญชี. **คอลัมน์:** วันที่ | **สมุดรายวัน (ลิงก์กลับ journal)** | คำอธิบาย (ฝังบัญชีย่อย inline) | Dr | Cr | คงเหลือ.
- **แถวแรก = "ยอดยกมา"** (opening balance สะสม) → คงเหลือ = running balance ลงมา
- GL ↔ journal ผูกสองทาง (คลิกลิงก์กลับได้)
- Export: เลือก ทุกบัญชี/ช่วงบัญชี/เลือกบัญชี + toggle แสดงบัญชีย่อย + ซ่อนยอด 0

### งบทดลอง (Trial Balance · 12968842)
รวมยอด Dr/Cr **ทุกบัญชี** → พิสูจน์ ΣDr=ΣCr → สะพานขึ้น 3 งบ. **toggle บัญชีย่อย (หัวใจ sub-account):**
- **แสดงบัญชีย่อย:** บัญชีหลัก=0 · ยอดดันลง sub (เช่น `111301`=0 แต่ `BSV002` Dr 107,000 · `BSV001` Cr 425,816.60)
- **ไม่แสดง:** โชว์ยอดสุทธิบัญชีหลัก (`111301` Cr 318,816.60 = 425,816.60−107,000)
- ต่างแค่ **ระดับรายละเอียด** ยอดสุทธิเท่ากัน

### งบฐานะการเงิน แบบเปรียบเทียบ (6533529)
**สินทรัพย์ = หนี้สิน + ทุน.** บัญชี > งบฐานะการเงิน > "เปรียบเทียบย้อนหลัง" (วัน/สัปดาห์/เดือน/ไตรมาส/ปี · **สูงสุด 12 งวด** = 12 คอลัมน์สิ้นงวด).
- โครง: สินทรัพย์ (หมุนเวียน/ไม่หมุนเวียน + รวมย่อย) · หนี้สิน+ส่วนผู้ถือหุ้น · **วงเล็บ = ติดลบ** · แถว "รวม" = subtotal ตัวหนา
- ทุกบรรทัด **[+] ขยาย** + **คลิกชื่อบัญชี = drill-down ไป GL** (ปิดวงจร roll-up ↔ drill-down)
- toggle ซ่อนบัญชีไม่เคลื่อนไหว + "แสดงเลขที่ผังบัญชี" + ขยาย/ย่อทั้งหมด

# 5) ปิดปี + XBRL/DBD + Dashboard

### กำไรสะสม (Retained Earnings · 6091582) — 🔴 year-end close
กำไรสะสม = หมวด **3 ส่วนของผู้ถือหุ้น**. 2 ประเภท:
- **จัดสรรแล้ว** (Appropriated) — สำรองตามกฎหมาย/นโยบาย · **ห้ามปันผล**
- **ยังไม่จัดสรร** (Unappropriated) — ปันผลได้ · ติดลบ = วงเล็บ (ขาดทุนสะสม)

🔴 **PEAK ปิดยอดกำไร→กำไรสะสม "อัตโนมัติ" ตอนขึ้นปีบัญชีใหม่** (closing entry ไม่ทำมือ) → **ระบบเราต้องมี routine ปิดปี:** โอน กำไร/ขาดทุนสุทธิ (P&L หมวด 4−5) → **กำไรสะสมยังไม่จัดสรร (หมวด 3)** ทันทีที่ขึ้นปีใหม่. (บทความไม่โชว์เลข GL ของ RE — ตั้งเองได้ เช่น 3xxxxx)

### งบไม่บาลานซ์ (Dr≠Cr · 8176987)
🔴 **สาเหตุหลักเดียว = journal มีบรรทัดที่ไม่ได้ระบุผังบัญชี (เลขบัญชีว่าง).**
> **กฎ validation ระบบเรา: ทุก line ต้องมี GL account เสมอ — ห้ามว่าง (fail-closed).**
หา: บัญชีรายวัน > พิมพ์รายงาน > Export Excel > Filter คอลัมน์ "เลขที่ผังบัญชี" = **(Blanks)** → แก้ผ่าน ตัวเลือก > แก้ไข.

### XBRL ยื่น DBD (6002248)
ไฟล์งบประจำปียื่นกรมพัฒนาธุรกิจการค้า. เมนู บัญชี > **DBD e-Filing** > สร้าง. **Wizard 4 สเต็ป:** รอบรายงาน (งวดปัจจุบัน+งวดก่อนเปรียบเทียบ) → ข้อมูลทั่วไป (หุ้นจดทะเบียน · **รองรับเฉพาะบริษัทจำกัด+ธุรกิจทั่วไป** · งบ P&L ขั้นเดียว/หลายขั้น) → หมายเหตุ (ดึงจากงบทดลอง) → e-Filing (ดาวน์โหลด XBRL + Excel สอบทาน).
- **เอกสาร 4 ชิ้น:** งบฐานะการเงิน · งบกำไรขาดทุน · งบแสดงการเปลี่ยนแปลงส่วนผู้ถือหุ้น · หมายเหตุ
- 🔴 **ต้องปิดงบ/มีงบทดลองครบก่อน** · ปีแรกดึงจาก XBRL นำเข้า ปีถัดไปดึงจากปีก่อน · (จริง: ยื่นภายใน 1 เดือนหลังอนุมัติงบ + ผ่านผู้สอบบัญชี)

### Dashboard ภาพรวมการเงิน (13628022) — view-layer อ่านอย่างเดียว
เมนู **การเงิน > ภาพรวม** (คนละหน้ากับ "บัญชี"). 5 ส่วน: (1) เงินสด/ธนาคาร ≤4 บัญชี (จาก bank sub-account · แดง=overdraft) (2) ตารางเงินเข้า-ออก + ค้างรับ/จ่ายเกินเวลา + คาดว่าเข้า/ออก (AR/AP aging + ประมาณการ · กดตัวเลข drill-down) (3) % กระทบยอดธนาคาร (4) top 3 บัญชียอดสูงสุด (5) เช็ครับ/จ่าย.
- 🔴 **ไม่ post GL** — อ่านจาก bank balances + AR/AP aging + reconciliation.

## 💎 พิมพ์เขียว GL engine ของ Pacred (สรุปสร้างระบบเอง)

### 1. Chart of Accounts
- ทุก GL account = **6 หลัก tree 3 ชั้น** (D1 หมวด1-5 · D2 รอง · D3-4 ย่อย · D5-6 running) · เลข **auto-gen** ตาม dropdown หลัก/รอง/ย่อย (แก้เลขเองไม่ได้)
- **sub-account (บัญชีย่อย)** ผูก dimension: ลูกหนี้→ต่อ contact · ธนาคาร→ต่อบัญชี · เงินสด→ต่อช่องทาง — gen อัตโนมัติจากเมนู การเงิน/ผู้ติดต่อ ไม่ใช่ผังบัญชี
- หมวด 1/2/3 → งบฐานะการเงิน · 4/5 → งบกำไรขาดทุน

### 2. Journal engine (post-on-approval — หัวใจ)
- **1 GL ledger + 1 status machine:** ทุกเอกสารอนุมัติ → เขียน `journal_entry` (header: เลขที่ · เล่ม/prefix · วันที่ · อ้างอิงเอกสารต้นทาง) + `journal_lines` (account_code · sub_account · Dr · Cr) → post GL. **draft/pending = ไม่แตะ GL.**
- **Invariant บังคับก่อน post:** `ΣDr = ΣCr` **AND** ทุก line มี `account_code` (ไม่งั้น flag "ควรตรวจสอบ" · กัน trial balance เพี้ยน)
- **Reverse = สร้าง entry กลับด้าน ไม่ลบ** (soft-cancel + audit trail) · manual JV ต้องถอยสถานะก่อนยกเลิก/แก้
- **Batch approve** ≤100/รอบ + โชว์ Σ Dr/Cr
- 5 เล่มตาม prefix (SV/RV/UV/PV/JV) · ค่าเสื่อม auto-post เป็น JV

### 3. Statements = view-layer roll-up
- GL ledger (running balance + opening) → trial balance (ΣDr=ΣCr · toggle sub-account) → balance sheet / P&L / cash-flow · drill-down คลิกชื่อบัญชี → GL

### 4. Period-close
- **VAT (รายเดือน):** JV zero-out `Dr 215101 / Cr 115401` → net ที่ control (`215351` auto / `215302` manual) → จ่าย EXP หรือ คืน RT (`115405` ลูกหนี้)
- **Year-end (รายปี):** โอน กำไร/ขาดทุนสุทธิ → กำไรสะสมยังไม่จัดสรร (หมวด 3) อัตโนมัติตอนขึ้นปีใหม่
- **DBD/XBRL (รายปี):** 4 งบ → wizard → ยื่นกรมพัฒนาธุรกิจการค้า
