## 🔄 สายเอกสาร (workflow chain) — ยืนยันจาก stepper บนฟอร์มจริง

```
ใบเสนอราคา (QO)  ── ยังไม่ลงบัญชี (แค่เสนอราคา)
     │  (ตอบรับแล้ว → ต่อยอด)
     ├──────────────► ใบวางบิล (BN) ── รวมหลายใบแจ้งหนี้ วางบิลลูกค้ารายเดือน (= billing-run เรา)
     ▼                     │
ใบแจ้งหนี้ (IV)  ◄──────────┘   ★ ลงบัญชี: Dr.ลูกหนี้การค้า  Cr.รายได้ + ภาษีขาย(VAT)
   = ใบส่งของ = บันทึกลูกหนี้ (ใบเดียวกันใน PEAK · ออกทีเดียวได้หมด)
     │  (รับเงิน → ต่อยอด)
     ▼
ใบเสร็จรับเงิน (RE)  ★ ลงบัญชี: Dr.เงินสด/ธนาคาร (+ ภาษีถูกหัก ณ ที่จ่าย)  Cr.ลูกหนี้การค้า
   = ออกใบกำกับภาษี (TIV) ในใบเดียวได้ · เลือกจังหวะออก VAT ได้
     ▼
ใบกำกับภาษีขาย (TIV) ── ภาษีขาย (แยกออกได้กรณีออกทีหลัง / รวมหลายแจ้งหนี้เป็น 1 ใบ)

ปรับยอดหลังออกเอกสาร:  ใบลดหนี้ (CN · ลดยอด/คืนเงิน)  ·  ใบเพิ่มหนี้ (DN · เพิ่มยอด)
รับเงินก่อนของ:        ใบรับเงินมัดจำ (Deposit) → ตัดกับใบแจ้งหนี้/ใบเสร็จภายหลัง
```

**กติกาเงินที่ต้องจำ:** ใบเสนอราคา=ไม่ลงบัญชี · **ใบแจ้งหนี้=จุดตั้งลูกหนี้ (เริ่มลงบัญชี)** · **ใบเสร็จ=จุดรับเงินจริง (ตัดลูกหนี้)** · VAT/หัก ณ ที่จ่าย = ต่อบรรทัด · PEAK ลงบัญชีอัตโนมัติจาก "บัญชี (GL) ที่ผูกแต่ละ line item".

## 1. ใบเสนอราคา (Quotation · QO)

<!-- batch quo-a · ดูรูป 14 รูป -->

> **ภาพรวม doc-type.** ใบเสนอราคา = เอกสารที่ผู้ขายออกให้ลูกค้าเพื่อพิจารณาราคา/เงื่อนไข **ก่อน** ตัดสินใจซื้อ — เป็นจุดเริ่มของสายเอกสารขาย แต่ **ยังไม่มีการลงบัญชี** (ไม่มี Dr/Cr, ไม่กระทบตัวเลขงบการเงิน).
> **รหัสเอกสาร = QO** · เลขรันอัตโนมัติรูปแบบ **`QO-YYYYMM#####`** (เห็นจริง `QO-20251200001`, `QO-20260100001`) — YYYY=ปี ค.ศ., MM=เดือน, #####=รัน 5 หลัก. แก้เลขเองได้ (ช่องเลขที่เอกสาร คลิกได้ · จำกัด 8 อักขระส่วนรัน).
> **สายเอกสารต่อยอด (stepper บนหัวเอกสาร):** `ใบเสนอราคา (QO)` ···▸ `ใบแจ้งหนี้ (IV)` ···▸ `ใบเสร็จรับเงิน (RE)` ···▸ `ใบกำกับภาษี (TIV)`. แต่ละขั้นมีปุ่ม `+` (วงกลมน้ำเงิน) เพื่อสร้างเอกสารขั้นถัดไปจากใบเสนอราคา.
> **เมนู:** แถบเมนูบนสุด = หน้าหลัก / **รายรับ** / รายจ่าย / ผู้ติดต่อ / สินค้า / การเงิน / บัญชี / คลังเอกสาร / ตั้งค่า. ใบเสนอราคาอยู่ใต้ **รายรับ**.
> **แท็บหน้า list ใบเสนอราคา:** ล่าสุด / ทั้งหมด / ร่าง / รออนุมัติ / รอตอบรับ / พ้นกำหนด(สีแดง) / ยอมรับแล้ว / ออกบางส่วน / ออกครบแล้ว / ปฏิเสธแล้ว / ยกเลิก. **คอลัมน์ list:** เลขที่เอกสาร · ลูกค้า · วันที่ออก · ใช้ได้ถึง · มูลค่าสุทธิ · สถานะ (มี dot สีตามสถานะ) / คำสั่ง (ปุ่มเขียว `ตอบรับ ▾` ต่อแถว). มีปุ่มม่วง `+ สร้างใบเสนอราคา ▾` และ `🖨 พิมพ์รายงาน` ด้านบน.

### วิธีสร้างใบเสนอราคาใน PEAK (article 5966405)

**ใช้ทำอะไร:** ออกใบเสนอราคาเสนอลูกค้า (ราคา + เงื่อนไข) ก่อนปิดการขาย. ไม่ลงบัญชี — เป็นเอกสารตั้งต้นของสายเอกสารรายรับ.

**ขั้นตอน (จากภาพจริง):**
1. **เข้าหน้าสร้าง** — เมนู `รายรับ` → hover `ใบเสนอราคา` → submenu ที่ลอยออกมา: `+ สร้าง` (ปุ่มสีม่วง) / `ยอมรับ` / `รอตอบรับ` / `พ้นกำหนด` / `ดูทั้งหมด` / `ล่าสุด` → กด `+ สร้าง`.
2. **หน้า "สร้างใบเสนอราคา"** เป็นการ์ด/แถบ section เรียงลงมา (มี stepper QO▸IV▸RE▸TIV อยู่บนสุด, มุมขวามี `⭐ เพิ่มรายการโปรด` + `⚙️ ตั้งค่าการแสดงผล`):
   - **① เลขที่เอกสาร (มุมขวาบน):** ช่อง `อ้างอิง` (ระบุเอกสารอ้างอิง เช่น เลขใบสั่งซื้อ ถ้ามี · จำกัด 32 อักขระ) + `เลขที่เอกสาร *` (รันอัตโนมัติ `QO-20251200001` · แก้ได้).
   - **② ข้อมูลลูกค้า:** `ชื่อลูกค้า *` (พิมพ์เพื่อค้นหาผู้ติดต่อ หรือสร้างผู้ติดต่อใหม่ · ดึงที่อยู่/เบอร์โทรมาอัตโนมัติ) · `วันที่ออก *` (default วันนี้ เช่น 23/12/2025 · แก้ย้อน/ล่วงหน้าได้) · `วันที่ใช้ได้ถึง *` (default +7 วัน เช่น 30/12/2025).
   - **③ กลุ่มจัดประเภท** (Package Pro Plus): `พนักงานขาย` / `ช่องทางการขาย` / `แผนก` / `โครงการ` (default "ยังไม่ระบุ" ทุกช่อง) — จัดประเภทรายได้ตามมิติ.
   - **④ ข้อมูลราคาและภาษี:** `ประเภทราคา *` = **แยกภาษี** (default · 100 รายได้ + 7 VAT = 107) หรือ **รวมภาษี** (107 ในราคาแล้ว → 100 รายได้ + 6.54... VAT) · `ราคานี้เป็นค่าเงิน` = สกุลเงิน (default THB).
   - **⑤ รายการ (line items):** ตารางคอลัมน์ `สินค้า/บริการ *` (เลือกรหัสสินค้า เช่น P00002 - สินค้า B) · `บัญชี` (ดึงจากรหัสสินค้า เช่น 410101 - รายได้จากการขาย) · `จำนวน *` (มีปุ่ม +/− · โชว์ "เหลือ: 32.00") · `ราคา/หน่วย *` · `ส่วนลด/หน่วย` (บาท/%) · `ภาษี` (7% / 0% / ไม่มี) · `มูลค่าก่อนภาษี` (คำนวณให้ เช่น 3.00×3,000=9,000) · `หัก ณ ที่จ่าย` (default "ยังไม่ระบุ") · ถังขยะลบแถว. ใต้ตาราง: `+ เพิ่มรายการใหม่` · `+ เพิ่มคำอธิบายรายการ` · ตัวเลือก `แสดง [50 ▾] รายการ` · `หน้า [1 ▾]`.
   - **⑥ สรุปข้อมูล:** `ส่วนลดรวม` (0.00 บาท · ✏️แก้ได้ · โชว์เมื่อการระบุภาษีทุกแถวเหมือนกัน) · `มูลค่าที่คำนวณภาษี 7%` (เช่น 9,000.00 บาท) · `ภาษีมูลค่าเพิ่ม 7%` (630.00 บาท · ✏️ กดดินสอแก้ยอดภาษีได้) · กล่องน้ำเงินเข้มขวา `จำนวนเงินทั้งสิ้น 9,630.00 บาท`.
   - **⑦ หมายเหตุสำหรับลูกค้า** (พับ/ขยาย) · **⑧ แนบไฟล์ในเอกสารนี้** (พับ/ขยาย) · **⑨ แท็ก** ("กรุณาเลือกแท็กที่ต้องการ").
3. **บันทึกหรืออนุมัติ (ปุ่มล่างขวา / Sticky Bar):**
   - `ยกเลิก` (ปุ่มขาว).
   - **`บันทึกร่าง ▲`** (ปุ่มม่วง) — dropdown: `บันทึกร่าง และสร้างใหม่`. → เอกสารสถานะ **ร่าง**.
   - **`อนุมัติใบเสนอราคา ▲`** (ปุ่มเขียว) — dropdown: `อนุมัติและพิมพ์` / `อนุมัติและสร้างใหม่` / `ส่งอนุมัติ` (ส่งให้หัวหน้า/ผู้มีอำนาจอนุมัติ). → เอกสารสถานะ **รอตอบรับ**.
4. **พิมพ์/ส่ง** — บนหน้ารายละเอียด (หลังอนุมัติ) มีปุ่ม `ส่งเอกสาร ▾` (เลือกอีเมล) · `พิมพ์ ▾` (ส้ม · dropdown พิมพ์ใบส่งของชั่วคราว/พิมพ์ซองเอกสาร · แก้ธีมสี/หัวเอกสาร/คอลัมน์/ฟอนต์ได้) · `ตัวเลือก ▾` (ม่วง).

**การลงบัญชี:** ไม่มี Dr/Cr — ใบเสนอราคาไม่ลงบัญชี, ไม่กระทบงบการเงิน.

**Gotchas:** ① `ส่วนลดรวม` กรอกไม่ได้ = แถวรายการระบุภาษีไม่เหมือนกันทุกแถว. ② รายการเกิน 50 แล้วต้องแทรก → เปลี่ยน "แสดง" เป็น 100 แล้วลากย้ายลำดับ. ③ ไม่เห็นช่อง เงินมัดจำ / หัก ณ ที่จ่าย → ไปเปิด toggle ที่ `⚙️ ตั้งค่าการแสดงผล`.

**ตั้งค่าการแสดงผลข้อมูล (dialog จากปุ่ม ⚙️):** ตั้งค่าส่วนแรก — `อ้างอิงถึง` / `กลุ่มจัดประเภท` / `สกุลเงิน` (เปิด-ปิด). ตั้งค่ารายการ — เลือก `รูปแบบใหม่ / รูปแบบเก่า` + toggle `สินค้า/บริการ` / `บัญชี` / `จำนวน` / `ส่วนลด/หน่วย` / `ภาษี` / `มูลค่าก่อนภาษี` / `หัก ณ ที่จ่าย` → ปุ่ม `ยกเลิก` / `บันทึก ▾`.

### แก้ไขเอกสารใบเสนอราคา (สถานะรอตอบรับ) (article 6014404)

**ใช้ทำอะไร:** แก้ไขข้อมูลบนใบเสนอราคาที่ออกไปแล้ว. **แก้ได้เฉพาะสถานะ "รอตอบรับ" เท่านั้น** — ถ้าเอกสารถูก "ยอมรับแล้ว" ต้องย้อนสถานะกลับเป็น "รอตอบรับ" ก่อน (ดู article 9798379).

**ขั้นตอน (จากภาพจริง):**
1. เมนู `รายรับ` → `ใบเสนอราคา` → กดแท็บ `รอตอบรับ` (โชว์จำนวน badge เช่น "1") เพื่อค้นเอกสาร. list โชว์คอลัมน์ เลขที่เอกสาร/ลูกค้า/วันที่ออก/ใช้ได้ถึง/มูลค่าสุทธิ + ปุ่มเขียว `ตอบรับ ▾` ต่อแถว.
2. คลิก `เลขที่เอกสาร` (เช่น QO-20260100001) เข้าหน้ารายละเอียด.
3. คลิก dropdown `ตัวเลือก ▾` (ปุ่มม่วง มุมขวาบน — เข้าได้ทั้งแถบบนและ Sticky Bar ล่าง) → เลือก `แก้ไข`.
4. แก้ข้อมูลตามต้องการ (สถานะ "รอตอบรับ" เปลี่ยนได้ทุกช่อง) → คลิก `บันทึก`.

**สถานะที่แก้ได้:** รอตอบรับ (และ ร่าง). **แก้ไม่ได้:** ยอมรับแล้ว / ปฏิเสธแล้ว / ยกเลิก / เมื่อมีเอกสารต่อยอดแล้ว.

### เปลี่ยนสถานะใบเสนอราคาจาก "ยอมรับแล้ว" กลับเป็น "รอตอบรับ" (article 9798379)

**ใช้ทำอะไร:** เมื่อกดตอบรับ (ยอมรับแล้ว) ไปแล้ว แต่ต้องการแก้เอกสาร ต้องย้อนสถานะกลับเป็น "รอตอบรับ" ก่อนถึงจะแก้ได้.

**ขั้นตอน (จากภาพจริง):**
1. เมนู `รายรับ` → `ใบเสนอราคา` → `ดูทั้งหมด`.
2. เลือกใบเสนอราคาสถานะ **ยอมรับแล้ว** ที่ต้องการเปลี่ยน (คลิกเลขที่เอกสาร).
3. หน้ารายละเอียด คลิกปุ่ม `ตัวเลือก ▾` → เลือก `กลับไปรอตอบรับ`.
4. **Dialog ยืนยัน:** หัวข้อ "ยืนยัน ใบเสนอราคา กลับไปสถานะรอตอบรับ" · ข้อความ "คุณกำลังแก้ไข ใบเสนอราคา QO-... กลับไปสถานะรอตอบรับ กรุณายืนยันเพื่อทำรายการ" → ปุ่ม `ยกเลิก` / `ยืนยัน` (เขียว). → สถานะเปลี่ยน ยอมรับแล้ว → รอตอบรับ. (หน้ารายละเอียดมี ribbon มุมขวาโชว์ "ใบเสนอราคา (ยอมรับแล้ว)" และ tab `ข้อมูลเอกสาร` / `ข้อมูลประวัติ`.)

### การกดปฏิเสธใบเสนอราคา (article 10319166)

**ใช้ทำอะไร:** ลูกค้าปฏิเสธ/ไม่อนุมัติใบเสนอราคา → เปลี่ยนสถานะจาก "รอตอบรับ" เป็น "ปฏิเสธแล้ว" ทันที เพื่อจัดการเอกสาร + วิเคราะห์ Conversion Rate ของทีมขาย.

**ขั้นตอน (จากภาพจริง):**
1. เมนู `รายรับ` → `ใบเสนอราคา` → `ดูทั้งหมด`.
2. เลือกเอกสารที่ลูกค้าไม่อนุมัติ (สถานะ รอตอบรับ) คลิกเข้าไป.
3. ไปที่ **หน้าตอบรับในเอกสาร** — ทำได้ 3 วิธี:
   - **3.1** ที่ stepper Timeline เอกสาร คลิกการ์ด `ใบเสนอราคา QO ▲` → dropdown โชว์เลขเอกสาร+ยอด+สถานะ (รอตอบรับ) → กดปุ่มลูกศร `>` → เลือก `💬 ตอบรับ`.
   - **3.2** คลิกปุ่ม `ตัวเลือก ▾` → เลือก `ตอบรับ`.
   - **3.3** คลิกปุ่ม `ตอบรับ` (ปุ่มเขียวด้านล่าง/ต่อแถวใน list).
4. **Dialog "ตอบกลับใบเสนอราคาแล้ว":** radio เลือก `○ ยอมรับ` / `● ปฏิเสธ` → เลือก **ปฏิเสธ** → กรอก `วันที่ปฏิเสธ` (default วันนี้ · แก้ได้) → ปุ่ม `ยกเลิก` / `ยืนยัน` (เขียว). → สถานะเปลี่ยนเป็น **ปฏิเสธแล้ว** (เอกสารไปแท็บ ปฏิเสธแล้ว).

> หมายเหตุ: dialog เดียวกันนี้ ถ้าเลือก `● ยอมรับ` = กดยอมรับใบเสนอราคา (สถานะ → ยอมรับแล้ว, กรอกวันที่ยอมรับ).

### การยกเลิกใบเสนอราคา (article 4552774)

**ใช้ทำอะไร:** ยกเลิกใบเสนอราคาที่ไม่ใช้แล้ว → สถานะเป็น "ยกเลิก". รองรับ 2 กรณี.

**กรณีที่ 1 — ยกเลิกโดยยังไม่มีเอกสารอ้างอิงต่อยอด (ยังไม่มี IV/RE):**
1. เข้าใบเสนอราคาที่ต้องการ → คลิก `ตัวเลือก ▾` → เลือก `ยกเลิกเอกสาร`.
2. **Dialog "ยืนยันการยกเลิกใบเสนอราคา":** ข้อความ "คุณกำลังจะยกเลิกใบเสนอราคา QO-... นี้ กรุณายืนยันเพื่อทำรายการ" → ปุ่ม `ยกเลิก` / `ยืนยัน` (เขียว).
3. **ผลลัพธ์:** เอกสารเปลี่ยนเป็นสถานะ **ยกเลิก** (dot สีเทา · ไปแท็บ ยกเลิก).

**เงื่อนไข:** ถ้าใบเสนอราคากด "ยอมรับแล้ว" อยู่ ต้องย้อนกลับเป็น "รอตอบรับ" ก่อน (article 9798379) จึงจะจัดการ/ยกเลิกได้ตามปกติ. *(จากภาพ dialog: เอกสารสถานะ "ยอมรับแล้ว" ก็สามารถกดยกเลิกได้เมื่อยังไม่มีเอกสารอ้างอิง — ปุ่มยกเลิกอยู่ใต้ตัวเลือก.)*

**กรณีที่ 2 — ยกเลิกเมื่อมีเอกสารอ้างอิงต่อยอดแล้ว:** ต้องยกเลิกย้อนกลับตามลำดับสายเอกสาร (จากปลายน้ำมาต้นน้ำ):
1. **ใบเสร็จรับเงิน (RE)** — `ตัวเลือก` → `ยกเลิกเอกสาร`.
2. **ใบแจ้งหนี้ (IV)** — ยกเลิกการชำระเงินก่อน (กด `⋮` 3 จุด → ยกเลิกการชำระ) แล้วจึง `ยกเลิกเอกสาร`.
3. **ใบเสนอราคา (QO)** — ทำตามกรณีที่ 1.

**การลงบัญชี:** ใบเสนอราคาไม่ลงบัญชี → การยกเลิกไม่มีผล Dr/Cr. (การยกเลิก IV/RE ที่ลงบัญชีแล้วจะกลับรายการบัญชีของเอกสารนั้นๆ ตาม doc-type ของมัน.)

**Gotcha:** เมื่อยกเลิก/ลบแล้วสร้างใหม่ ระบบจะรันเลขที่เอกสารฉบับใหม่ต่อจากลำดับล่าสุดอัตโนมัติ (แก้เลขเองได้). เอกสารที่ยกเลิกยังอยู่ในแท็บ "ยกเลิก" เพื่อการอ้างอิง (ไม่ถูกลบทิ้ง).

## หมายเหตุรวม (batch quo-b — งานเสริมของใบเสนอราคา)

ทุกงานในชุดนี้อยู่ใต้เมนู **รายรับ (Income)** → **ใบเสนอราคา** เป็นหลัก (ยกเว้นใบสั่งซื้อที่อยู่ใต้ **รายจ่าย**) — ไม่มีการลงบัญชี Dr/Cr โดยตรง เพราะใบเสนอราคาเป็นเอกสารก่อนการขาย (pre-sale) ยังไม่กระทบบัญชี/สต็อก การลงบัญชีจะเกิดเมื่อออกใบแจ้งหนี้/ใบเสร็จต่อยอดเท่านั้น

**รหัสเอกสาร:** ใบเสนอราคา = **QO** · รูปแบบเลขที่ = `QO-YYYYMM#####` (เช่น `QO-20250800005`) · ใบสั่งซื้อ = **PO** (`PO-YYYYMM#####`)

**สายเอกสารต่อยอด (stepper บนหัวเอกสาร):**
`ใบเสนอราคา (QO)` ··· `ใบแจ้งหนี้ (IV)` ··· `ใบเสร็จรับเงิน (RE)` ··· `ใบกำกับภาษี (TIV)`
ปุ่ม `+` สีน้ำเงินระหว่างการ์ดใช้สร้างเอกสารขั้นถัดไปได้ทันที

### คัดลอกใบเสนอราคา เพื่อออกใบใหม่ (article 4548443)

**ใช้ทำอะไร:** สร้างใบเสนอราคาใบใหม่จากใบเดิมทั้งหมดในคลิกเดียว โดยไม่ต้องกรอกข้อมูลลูกค้า/รายการซ้ำ — ประหยัดเวลา ลดข้อผิดพลาด ใช้ได้แม้ใบเดิมถูกยกเลิกไปแล้ว

**ขั้นตอน (จากภาพจริง):**
1. เมนู **รายรับ → ใบเสนอราคา → ดูทั้งหมด** (submenu ของ ใบเสนอราคา มี: + สร้าง / ยอมรับ / รอตอบรับ / พ้นกำหนด / ดูทั้งหมด / ล่าสุด)
2. ในหน้า list คลิก **เลขที่เอกสาร** (ลิงก์สีน้ำเงิน เช่น QO-20250800005) เข้าหน้ารายละเอียด
3. บนหน้ารายละเอียดเอกสาร มุมขวาบนมี 3 ปุ่ม: `✈ ส่งเอกสาร` (น้ำเงิน) · `🖨 พิมพ์` (ส้ม) · `☰ ตัวเลือก` (ม่วง) → กด **ตัวเลือก**
4. Dropdown ตัวเลือก เรียงจากบน: `🔒 แก้ไข` (เทา/ล็อกเมื่อยอมรับแล้ว) · **`📄 คัดลอก`** · สร้างใบรับเงินมัดจำ · สร้างใบแจ้งหนี้ · สร้างใบเสร็จรับเงิน · สร้างใบสั่งซื้อ · กลับไปรอตอบรับ · เปลี่ยนเป็นออกครบแล้ว · ยกเลิกเอกสาร · 🔒 ล็อกเอกสาร → เลือก **คัดลอก**
5. ระบบสร้างใบเสนอราคาใหม่อีก 1 ใบ (สถานะ ร่าง/รอตอบรับ) พร้อมให้แก้ไขข้อมูลต่อ

**การลงบัญชี:** ไม่มี (คัดลอกเอกสารเฉย ๆ)
**เงื่อนไข/gotcha:** คัดลอกได้ทุกสถานะ รวมถึงใบที่ **ยกเลิกแล้ว** (ใช้คัดลอกสร้างใหม่แทนการแก้ใบที่ยกเลิก)

### เพิ่มรายการโปรด ใบเสนอราคา (article 6014184)

**ใช้ทำอะไร:** บันทึกชุดข้อมูลใบเสนอราคา (ลูกค้า/สินค้า/จำนวน/ราคา/ส่วนลด) เป็น "รายการโปรด" (favorite/template) เพื่อกดเรียกใช้ซ้ำได้เร็วในครั้งถัดไป — เหมาะกับงานที่ออกใบซ้ำ ๆ ให้ลูกค้าเดิมหรือสินค้าชุดเดิม

**ขั้นตอน (จากภาพจริง):**
1. เมนู **รายรับ → ใบเสนอราคา → + สร้าง** เข้าหน้า **สร้างใบเสนอราคา**
2. กรอกข้อมูลลูกค้า + รายการสินค้า/บริการ + จำนวน + ราคา (+ ส่วนลด) ให้เรียบร้อยก่อน
3. มุมขวาบน (ใต้ stepper) มีปุ่ม **`⭐ เพิ่มรายการโปรด`** และ `⚙ ตั้งค่าการแสดงผล` → กด **เพิ่มรายการโปรด**
4. เปิด dialog **"เพิ่มรายการโปรด - ใบเสนอราคา"** กรอก:
   - **ชื่อรายการโปรด** (จำเป็น * · สูงสุด 92 ตัวอักษร) — เช่น "สินค้า B"
   - **คำอธิบายรายการโปรด** (สูงสุด 500 ตัวอักษร · ไม่บังคับ)
   - **บันทึกผู้ติดต่อไว้** — radio: `● บันทึก` / `○ ไม่บันทึก` (เลือกว่าจะจำชื่อกิจการ/ลูกค้าด้วยไหม)
   - **บันทึกจำนวน, ราคาและส่วนลดสินค้า/บริการไว้** — radio: `● บันทึก` / `○ ไม่บันทึก`
5. กด **เพิ่มรายการ** (ปุ่มเขียวมุมขวาล่าง · ปุ่ม ยกเลิก อยู่ซ้าย)
6. รายการโปรดจะปรากฏเป็นการ์ดใต้หัวข้อ **"รายการโปรดของใบเสนอราคา"** ด้านบนหน้าสร้างใบเสนอราคา (มีไอคอน PEAK + ชื่อ + เมนู ⋮) — คลิกการ์ดเพื่อดึงข้อมูลมาใช้ในใบใหม่

**การลงบัญชี:** ไม่มี (เป็น template เก็บไว้เรียกซ้ำ)
**สถานะ:** ไม่มีสถานะเอกสาร (เป็นรายการโปรด ไม่ใช่เอกสารจริง)

### ตรวจสอบสถานะใบเสนอราคา (article 7200095 · รหัสเอกสาร QO)

**ใช้ทำอะไร:** ติดตามว่าใบเสนอราคาแต่ละใบเดินไปถึงขั้นไหน และจัดการเปลี่ยนสถานะผ่านแถบเครื่องมือด้านล่าง (Sticky Bar)

**ขั้นตอน (จากภาพจริง):**
1. เมนู **รายรับ → ใบเสนอราคา → ดูทั้งหมด**
2. หน้า list มี **แถบสถานะ (tab)** ให้คลิกกรอง: `↺ ล่าสุด · ทั้งหมด · ร่าง · รออนุมัติ · รอตอบรับ · พ้นกำหนด · ยอมรับแล้ว · ออกบางส่วน · ออกครบแล้ว · ปฏิเสธแล้ว …` (แต่ละ tab มี badge ตัวเลขจำนวน) — มีตัวกรอง วันที่ออก + ช่องค้นหา (ชื่อ/เลขที่) + ค้นหาขั้นสูง
3. คลิกเข้าเอกสาร → จัดการผ่าน **Sticky Bar ด้านล่าง** (ปุ่มเปลี่ยนตามสถานะปัจจุบัน) — บนหน้ารายละเอียดมีธง (banner มุมขวา) บอกสถานะ เช่น "ใบเสนอราคา (รอตอบรับ)" / "(ยอมรับแล้ว)"

**สถานะทั้งหมด (คอลัมน์ สถานะ · มี dot สี):**
| สถานะ | ความหมาย |
|---|---|
| ล่าสุด | เอกสารที่สร้างล่าสุด 100 รายการ (เป็น tab filter ไม่ใช่สถานะจริง) |
| ทั้งหมด | ทุกฉบับ ทุกสถานะ |
| ร่าง | บันทึกร่างไว้ ยังไม่ใช้งาน |
| รออนุมัติ | รอผู้มีอำนาจอนุมัติ (badge สีส้ม) |
| รอตอบรับ | ส่งให้ลูกค้าแล้ว รอลูกค้าตอบรับภายในวันที่ใช้ได้ถึง (badge สีน้ำเงิน · dot ○) |
| พ้นกำหนด | เลย "วันที่ใช้ได้ถึง" แล้วยังไม่ตอบรับ (สีแดง) |
| ยอมรับแล้ว | ลูกค้าตอบรับแล้ว → สร้างใบแจ้งหนี้ต่อได้ (dot ● น้ำเงิน) |
| ออกบางส่วน | นำไปออกเอกสารต่อบางส่วน ยังไม่ครบ (dot ○ เขียว · badge เขียว) |
| ออกครบแล้ว | ออกเอกสารต่อครบทุกรายการ |
| ปฏิเสธแล้ว | เอกสารถูกปฏิเสธ |
| ยกเลิก | ยกเลิกแล้ว ใช้ต่อไม่ได้ แต่ยังคัดลอกสร้างใหม่ได้ |

**ปุ่มบน Sticky Bar เปลี่ยนตามสถานะ:**
- สถานะ **รอตอบรับ** → Sticky Bar แสดง `✈ ส่งเอกสาร` · `🖨 พิมพ์` · `☰ ตัวเลือก` · **`💬 ตอบรับ`** (ปุ่มเขียวขวาสุด — กดเพื่อเปลี่ยนเป็นยอมรับแล้ว + บันทึก "วันที่ยอมรับ")
- สถานะ **ยอมรับแล้ว** → ปุ่มขวาสุดเปลี่ยนเป็น **`สร้างใบแจ้งหนี้`** (เขียว)
- ต่อไปที่หน้าใบแจ้งหนี้ → ปุ่ม `รับชำระเงิน`

**ตรวจว่าออกเอกสารต่อแล้วหรือยัง:** ดู stepper/Timeline บนหัวเอกสาร (การ์ด QO→IV→RE→TIV) — การ์ดที่ทำแล้วจะมี dot สี/เครื่องหมาย ✓ · หรือดู tab "ข้อมูลประวัติ" ในหน้ารายละเอียด

### เปลี่ยนสถานะใบเสนอราคาและใบสั่งซื้อหลายรายการ (bulk status change · article 11461514)

**ใช้ทำอะไร:** เปลี่ยนสถานะเอกสารหลายใบพร้อมกันเป็น **"ออกครบแล้ว"** — ใช้กับ **ใบเสนอราคา (QO)** และ **ใบสั่งซื้อ (PO)** เพื่อปิดงานเอกสารที่ออกต่อครบแล้วเป็นชุด

**เมื่อใช้ได้:**
- ใบเสนอราคา: อยู่ในสถานะ **ยอมรับแล้ว** หรือ **ออกบางส่วน**
- ใบสั่งซื้อ: อยู่ในสถานะ **อนุมัติแล้ว** หรือ **ออกบางส่วน**

**ขั้นตอน — ใบเสนอราคา (จากภาพจริง):**
1. เมนู **รายรับ → ใบเสนอราคา → ดูทั้งหมด**
2. คลิก tab **ยอมรับแล้ว** หรือ **ออกบางส่วน**
3. **ติ๊ก checkbox** หน้าแต่ละแถวที่ต้องการ (หรือ checkbox หัวตารางเลือกทั้งหน้า) — เลือกได้สูงสุด **100 รายการ** · แถวที่เลือกจะไฮไลต์ฟ้า + มี badge แดงบอกจำนวน
4. **Sticky Bar ด้านล่างโผล่ขึ้น:** ซ้ายบอก `เลือก 3 รายการ (สูงสุด 100 รายการ) · รวมมูลค่าสุทธิ 12,741.00` · ขวามีปุ่ม `ยกเลิก` · `🖨 พิมพ์เอกสาร` · **`เปลี่ยนเป็นออกครบแล้ว`** (เขียว)
5. กด **เปลี่ยนเป็นออกครบแล้ว** → ระบบตรวจเงื่อนไข แล้วให้ยืนยัน `อัปเดตสถานะ` / `เปลี่ยนเป็นออกครบแล้ว`

**ขั้นตอน — ใบสั่งซื้อ (จากภาพจริง):**
1. เมนู **รายจ่าย → ใบสั่งซื้อ → ดูทั้งหมด** (submenu ใบสั่งซื้อ: อนุมัติแล้ว / รออนุมัติ / ดูทั้งหมด / ล่าสุด)
2. คลิก tab **อนุมัติแล้ว** หรือ **ออกบางส่วน**
3. ติ๊ก checkbox แถวที่ต้องการ (คอลัมน์ คำสั่ง ด้านขวาแต่ละแถว = ปุ่ม `บันทึกซื้อสินค้า`)
4. Sticky Bar ล่างโผล่: `เลือก 3 รายการ (สูงสุด 100 รายการ) รวมมูลค่าสุทธิ 337.05` + `ยกเลิก` · `🖨 พิมพ์เอกสาร` · **`เปลี่ยนเป็นออกครบแล้ว`**
5. กด **เปลี่ยนเป็นออกครบแล้ว**

**สถานะเป้าหมาย:** เปลี่ยนได้เป็น **"ออกครบแล้ว"** เท่านั้น
**การลงบัญชี:** ไม่มี (เปลี่ยนสถานะเอกสารอย่างเดียว)
**Gotcha/เงื่อนไข:**
- ใบเสนอราคาแต่ละใบ **ต้องมีใบแจ้งหนี้/ใบเสร็จรับเงินออกต่อแล้ว** ถึงจะเปลี่ยนเป็นออกครบได้
- ใบสั่งซื้อต้อง **บันทึกซื้อสินค้าต่อจากใบสั่งซื้อแล้ว**

**ใช้ทำอะไร:** หลังลูกค้ากด "ยอมรับ" ใบเสนอราคา ต่อยอดเป็นใบแจ้งหนี้เพื่อเรียกเก็บเงิน โดยระบบดึงข้อมูลจากใบเสนอราคามาให้อัตโนมัติ (ไม่ต้องพิมพ์ซ้ำ)

**ขั้นตอน (จากภาพจริง):**
1. **รายรับ → ใบเสนอราคา → ยอมรับ** (เมนูย่อยของใบเสนอราคามี: สร้าง / ยอมรับ / รอตอบรับ / พ้นกำหนด / ดูทั้งหมด / ล่าสุด) — หรือเข้าแท็บ **"ยอมรับแล้ว"** ในลิสต์ใบเสนอราคา
2. เปิดใบเสนอราคาที่สถานะ **ยอมรับแล้ว** (หน้าเอกสารมีธง "ใบเสนอราคา (ยอมรับแล้ว)" + `วันที่ยอมรับ`). มี **2 ช่องทางสร้าง:**
   - **ช่องทางที่ 1:** กดปุ่ม **(+)** บนการ์ด "ใบแจ้งหนี้ IV" ใน stepper ด้านบน
   - **ช่องทางที่ 2:** กดปุ่ม **สร้างใบแจ้งหนี้** (ปุ่มเขียว) บนแถบเครื่องมือลอยล่างจอ (Sticky Bar)
3. ระบบให้ระบุ **"วันที่ตอบรับใบเสนอราคา"** → กด **ยืนยัน**
4. ขึ้น **Pop-up "สร้างใบแจ้งหนี้จากใบเสนอราคา #QO-....."** ให้เลือก **สร้างเอกสารด้วยวิธี** (radio):
   - **ตามใบเสนอราคา** — สร้างตามยอดทั้งหมด (100%)
   - **แบ่งตามสินค้า/บริการแต่ละแถว** — ระบุจำนวน/ราคาที่จะออกต่อบรรทัด (คำนวณจากยอดคงเหลือแต่ละแถว)
   - ปุ่ม **ยกเลิก** / **ยืนยัน** (เขียว)
5. เข้าฟอร์มสร้างใบแจ้งหนี้ที่ **เติมข้อมูลอัตโนมัติ** จากใบเสนอราคา: หัวฟอร์มโชว์ **"อ้างอิงจากระบบ: QO-20260200001"** + ช่อง อ้างอิง = เลข QO + เลขที่เอกสารใหม่ IVT-.... · ชื่อ/ที่อยู่ลูกค้า · วันที่ · ประเภทราคา/ภาษี/สกุลเงิน · **รายการสินค้า/บริการ** ทุกแถวมี checkbox หน้าแถว (ติ๊กเลือกแถวที่จะออก) และถ้าแบ่งตามแถวจะโชว์ **"เหลือ : 59.00"** ใต้ช่องจำนวน (ยอดคงเหลือที่ยังไม่ได้ออก). แก้ไข/เพิ่มรายการได้
6. บันทึก: **ยกเลิก / บันทึกร่าง ⌄ / อนุมัติใบแจ้งหนี้ ⌄**

**ฟิลด์สำคัญ:** วันที่ตอบรับใบเสนอราคา (ตอนกดสร้าง) · วิธีสร้าง (ตามใบเสนอราคา / แบ่งตามแถว) · checkbox เลือกแถว · จำนวนต่อแถว (มี "เหลือ" กำกับ) · toggle การออกใบกำกับภาษี

**สถานะ:** ใบเสนอราคา `รอตอบรับ → ยอมรับแล้ว` · ใบแจ้งหนี้ที่สร้าง = **รอรับชำระ**

**การลงบัญชี (Dr/Cr):** เหมือนใบแจ้งหนี้ปกติ (กรณีตั้งหนี้: `Dr.ลูกหนี้ / Cr.รายได้` + `Cr.ภาษีขายรอเรียกเก็บ`; เงื่อนไขวันที่ให้ผล SV/RV เช่นเดียวกับ article 5966489)

**เงื่อนไข/ข้อควรระวัง:** เลขที่เอกสาร + วันที่ครบกำหนด ตั้งเองได้ผ่านไอคอนฟันเฟือง · ตั้งหมายเหตุลูกค้าได้ · โดยปกติออกใบแจ้งหนี้หลังส่งมอบสินค้า/บริการเสร็จ · stepper จะเชื่อม QO → IV ให้เห็นความสัมพันธ์

### สร้างใบแจ้งหนี้หลายใบจากใบเสนอราคาใบเดียว (article 4548865)

**ใช้ทำอะไร:** ธุรกิจที่ **แบ่งส่งมอบ/แบ่งเรียกเก็บเป็นงวด** (เช่นงานโปรเจกต์) ออกใบแจ้งหนี้หลายใบทยอยจากใบเสนอราคาฉบับเดียว จนครบมูลค่า

**ขั้นตอน (จากภาพจริง):**
1. **รายรับ → ใบเสนอราคา → ดูทั้งหมด** → เลือกแท็บสถานะ **"ยอมรับแล้ว"** (ลิสต์ใบเสนอราคามีแท็บ: ล่าสุด / ทั้งหมด / ร่าง / ... / **ออกบางส่วน** / **ออกครบแล้ว** / ปฏิเสธแล้ว)
2. ที่แถวใบเสนอราคา กดปุ่มคำสั่ง **สร้างใบแจ้งหนี้ ⌄** (ปุ่มเขียวในคอลัมน์คำสั่ง) — หรือเปิดใบแล้วกดในหน้าเอกสาร
3. ระบุ **วันที่ตอบรับใบเสนอราคา** → **ยืนยัน**
4. Pop-up **"สร้างใบแจ้งหนี้จากใบเสนอราคา #QO-....."** → เลือกวิธีจาก **3 รูปแบบ** (radio):
   - **ตามใบเสนอราคา** (ทั้งยอด)
   - **แบ่งตามสินค้า/บริการแต่ละแถว** (ระบุจำนวน/ราคาต่อบรรทัดตามยอดคงเหลือ — เหมาะกับทยอยส่งมอบ)
   - **แบ่งเปอร์เซ็นต์** (ออกเป็น % ของยอดงาน)
   - ปุ่ม **ยกเลิก / ยืนยัน**
5. ในฟอร์ม แก้ไขจำนวน/ราคาต่อบรรทัด (แถวที่มี checkbox ✓) แล้วบันทึก: **บันทึกร่าง / อนุมัติ / อนุมัติและพิมพ์** (ปุ่มล่างมีเลขเอกสาร เช่น `ใบแจ้งหนี้ IV-20250800008` · ปุ่ม ยกเลิก / บันทึกร่าง ⌄ / อนุมัติใบแจ้งหนี้ ⌄)
6. **สร้างใบถัดไปสำหรับยอดคงเหลือ:** ทำซ้ำข้อ 1–5 — ระบบแสดง **ยอดคงเหลือ (เหลือ/คงเหลือ)** อัตโนมัติต่อบรรทัด จนกว่าจะออกครบ

**แนวคิดยอดคงเหลือ + สถานะใบเสนอราคา:**
- เมื่อออกใบแจ้งหนี้บางส่วน → ใบเสนอราคาเปลี่ยนเป็น **"ออกเอกสารต่อบางส่วน"** (แท็บ "ออกบางส่วน" มีตัวนับ เช่น 5)
- เมื่อออกครบตามมูลค่า → เปลี่ยนเป็น **"ออกครบแล้ว"**

**การตั้งค่าเงื่อนไข (สำคัญ — จากภาพหน้า Settings):** เมนู **ตั้งค่า → ตั้งค่านโยบายบัญชี → การออกเอกสารจากใบเสนอราคา**:
- **"มูลค่าเอกสารทั้งหมดเกินใบเสนอราคา"**: `ไม่ได้` / `ได้ แจ้งเตือนก่อน`
- **เงื่อนไขการเปลี่ยนแปลงข้อมูลตามวิธีแบ่งตามสินค้า/บริการแต่ละแถว** (ตั้งต่อฟิลด์): `จำนวน`, `ราคา/หน่วย`, `ส่วนลด/หน่วย` → เลือก `ห้ามแก้ไข` / `ลดได้` / `เพิ่มได้` / `ลดและเพิ่มได้`; และ `เพิ่มรายการใหม่` → `ไม่ได้` / `ได้`
- ปุ่ม **ยกเลิก / บันทึก** (เขียว)

**ฟิลด์สำคัญ:** วิธีสร้าง (3 แบบ) · จำนวน/ราคาต่อแถว (ผูกกับ "เหลือ") · เงื่อนไขนโยบายด้านบน

**สถานะ:** ใบเสนอราคา: ยอมรับแล้ว → **ออกเอกสารต่อบางส่วน** → **ออกครบแล้ว** · ใบแจ้งหนี้แต่ละใบ: รอรับชำระ

**การลงบัญชี (Dr/Cr):** แต่ละใบลงเหมือนใบแจ้งหนี้ปกติ (ตั้งหนี้: `Dr.ลูกหนี้ / Cr.รายได้ + Cr.ภาษีขายรอเรียกเก็บ`; กรณีรับล่วงหน้าใช้ 212103 รายได้รับล่วงหน้า)

**ข้อควรระวัง:** เอกสารที่ล็อกแล้วยังพิมพ์รายใบได้ตามปกติ (ล็อกกันแก้ไข/ยกเลิก ไม่กันการพิมพ์)

### สร้างใบแจ้งหนี้ แบบแบ่งชำระ (article 5885222)

**ใช้ทำอะไร:** ใบแจ้งหนี้ยอดเดียว แต่ลูกค้า **ทยอยจ่ายเป็นงวด ๆ** (เช่นมัดจำ + ยอดคงเหลือ). ทำโดยออกใบแจ้งหนี้ยอดเต็มก่อน แล้วบันทึก "รับชำระ" ทีละงวดจนครบ

**ขั้นตอน (จากภาพจริง):**
1. เปิดใบแจ้งหนี้ (สถานะ **รอรับชำระ**) → แท็บ **"ข้อมูลการชำระ"** — หน้านี้โชว์: `มูลค่าเต็มใบแจ้งหนี้` (เช่น 10,700.00), `มูลค่าสุทธิที่ต้องรับชำระทั้งสิ้น`, `มูลค่าที่รับชำระแล้ว 0 รายการ`, กล่อง `มูลค่าที่รอรับชำระอีก` (10,700.00) + `วันที่ครบกำหนด` + ปุ่ม **รับชำระ** (เขียว)
2. กด **รับชำระ** → เปิดกล่อง **"รับชำระเงิน #IVT-....."** :
   - เลือกโหมด **พื้นฐาน / ขั้นสูง** (radio) · **วันที่รับชำระ** *
   - **ช่องทางการรับชำระเงิน** (checkbox): `รับชำระโดย` (เลือกบัญชีเงิน เช่น "ไทยพาณิชย์ - ...SCB"), **จำนวนเงินที่รับชำระ** (กรอกยอดงวดนี้ เช่น 3,000.00), `หมายเหตุ`
   - checkbox **ถูกหัก ณ ที่จ่าย** (ถ้าลูกค้าหักภาษี ณ ที่จ่าย)
   - สรุป: `รับชำระด้วยเงินรวม` / กล่องเขียว **รับชำระรวมทั้งสิ้น** (3,000.00) / `ต้องรับชำระเงินอีก 7,700.00 บาท`
   - ปุ่ม **ยกเลิก / รับชำระเงิน ⌄** (เขียว)
3. **บันทึกงวดถัดไป:** กลับหน้าข้อมูลการชำระ ระบบโชว์ยอดค้างเหลือ + ปุ่ม **รับชำระ** อีกครั้ง → ทำซ้ำจนครบยอด (สถานะไล่ ยังไม่รับชำระ → รับบางส่วน → รับชำระเต็มจำนวน)
4. **ออกใบเสร็จ:** เลือกได้ 2 แบบ — (ก) **แยกตามการชำระ** (ใบเสร็จต่อแต่ละงวด) หรือ (ข) **รวมยอดทั้งหมด** (ออกได้ต่อเมื่อรับชำระครบเต็มจำนวนแล้ว)

**สายเอกสาร:** `ใบแจ้งหนี้ IV` → (รับชำระเงินหลายงวด) → `ใบเสร็จรับเงิน RE` (แยกตามงวด หรือ รวมยอด) → `ใบกำกับภาษี TIV`

**ฟิลด์สำคัญ:** วันที่รับชำระ* · รับชำระโดย (บัญชีเงิน) · จำนวนเงินที่รับชำระ (ยอดงวด) · checkbox ถูกหัก ณ ที่จ่าย

**สถานะ:** รอรับชำระ → รับบางส่วน → รับชำระเต็มจำนวน (พร้อมออกใบเสร็จ)

**การลงบัญชี (Dr/Cr):** บทความไม่ได้ระบุ journal โดยตรง — แต่ตามกลไก แต่ละครั้งที่กด "รับชำระเงิน" จะเกิด RV (`Dr.เงิน/ธนาคาร / Cr.ลูกหนี้การค้า`) ตามยอดงวดนั้น จนล้างลูกหนี้หมด (สอดคล้องกับตรรกะ RV ใน article 5966489)

**ข้อควรระวัง:** ออกใบเสร็จแบบ "รวมยอดทั้งหมด" ได้เฉพาะเมื่อรับชำระครบแล้ว · ถ้าต้องการ **ออกเอกสารเรียกเก็บแยกตามงวด/ตาม % ของงาน** ให้ใช้ **ใบวางบิล (Billing Statement)** เป็นเอกสารแยก (สร้างใบแจ้งหนี้ยอดเต็ม 100% ก่อน → เมนูใบวางบิล → เลือกใบแจ้งหนี้ → สร้างเป็นใบวางบิล → ระบุจำนวนเงินแต่ละงวด)

### สร้างใบแจ้งหนี้ จากหน้ารายรับ (article 4548975)

**ใช้ทำอะไร:** ทางเข้าสร้างใบแจ้งหนี้แบบยืนเดี่ยวจาก **หน้ารายรับ** โดยตรง (ฟอร์มเดียวกับ article 5966489 ทุกประการ) — เป็นเส้นทางลัดหลักที่พนักงานใช้บ่อยสุด

**ขั้นตอน (จากภาพจริง):**
1. เมนู **รายรับ** → เมนูย่อย **ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้)** → **+ สร้าง** (flyout ยังมีลัดตามสถานะ: รอชำระเงิน / ชำระแล้ว / พ้นกำหนด / ดูทั้งหมด / ล่าสุด). อีกทางบนหน้าเดียวกัน = ปุ่ม **+ สร้างใบแจ้งหนี้ ⌄** (ม่วง) ข้างปุ่ม **🖨 พิมพ์รายงาน**
2. เข้าฟอร์ม **"สร้างใบแจ้งหนี้ / ใบกำกับภาษี"** (เหมือน 5966489): section ข้อมูลลูกค้า / กลุ่มจัดประเภท (โครงการ·พนักงานขาย·กลุ่มธุรกิจ) / ข้อมูลราคาและภาษี (ประเภทราคา · **การออกใบกำกับภาษี toggle** · สกุลเงิน) / รายการ (สินค้า·บัญชี·จำนวน·ราคา/หน่วย·ส่วนลด/หน่วย·ภาษี·มูลค่าก่อนภาษี·หัก ณ ที่จ่าย). ในภาพ toggle "ออกใบกำกับภาษี" เปิด (สีน้ำเงิน) และหัก ณ ที่จ่าย = 3% (90.00)
3. บันทึก: **ยกเลิก / บันทึกร่าง ⌄ / อนุมัติใบแจ้งหนี้ ⌄** (ปุ่มล่างมีเลขเอกสาร เช่น `ใบแจ้งหนี้ IVT-20250800004`)

**หน้าภาพรวมรายรับ (บริบท):** ฝั่งขวามีการ์ด **"เอกสารที่ออก"** สรุปยอดตามชนิดเอกสาร/ช่วงเวลา (ใบเสนอราคาที่ออก / รอลูกค้าตอบรับ / ลูกค้ายอมรับแล้ว / ออกเอกสารต่อบางส่วน ฯลฯ) · การ์ดซ้ายสรุป รอรับชำระ / พ้นกำหนดชำระ (เช่น พ้นกำหนด 16 รายการ 111,389.32 บาท)

**ฟิลด์สำคัญ + สถานะ + การลงบัญชี:** เหมือน article 5966489 ทุกอย่าง (สถานะ รอรับชำระ · Journal SV รายวันขาย · Dr.ลูกหนี้การค้า / Cr.รายได้ / Cr.ภาษีขายรอเรียกเก็บ)

**หมายเหตุ:** article นี้เป็น "ทางเข้าที่ 2" ของการสร้างใบแจ้งหนี้ยืนเดี่ยว — เนื้อหาซ้อนกับ 5966489 ต่างกันแค่จุดเริ่ม (เข้าจากเมนูรายรับ/หน้าภาพรวมรายรับ)

**ฟิลด์/ปุ่มสำคัญ:** ปุ่ม **ย้อนกลับ** (กลับไปขั้นเลือกเอกสารในใบวางบิล) · **ติ๊กถูก ✔️** (เลือก/นำใบแจ้งหนี้เข้า-ออกใบวางบิล) · **ถัดไป / บันทึก** · ช่อง **จำนวนเงินวางบิล** (แก้ยอดวางบิลต่อใบได้)

**สถานะ:** รอรับชำระ / รับชำระแล้ว (ใบแจ้งหนี้) · ใบวางบิล = รอรับชำระ

**การลงบัญชี (Dr/Cr):** บทความไม่ระบุ (เน้นขั้นตอนปฏิบัติ)

**เงื่อนไข/ข้อควรระวัง — เอกสารที่แก้ไขใบแจ้งหนี้ไม่ได้ทันที ต้องปลด/ยกเลิก downstream ก่อน:**
- มีการกดรับชำระ + ออกใบเสร็จแล้ว → ต้องยกเลิกใบเสร็จก่อน
- ถูกนำไปสร้างใบวางบิลแล้ว → ต้องยกเลิกใบวางบิล **หรือ** ปลดใบแจ้งหนี้ออกจากใบวางบิลก่อน
- ถ้าใบวางบิลมีการชำระเงิน/ออกใบเสร็จแล้ว → ยกเลิกใบวางบิลไม่ได้จนกว่าจะยกเลิกการชำระ/ใบเสร็จของใบวางบิลก่อน
- **หลักรวม:** "ผู้ใช้งานต้องทำการยกเลิกเอกสารที่เชื่อมโยงกันก่อน" จึงจะแก้ไขใบแจ้งหนี้ได้

**ขั้นตอน (จากภาพจริง)**

วิธีที่ 1 — เทียบด้วยสูตร XLOOKUP บน Excel (แนะนำ):
1. **ดาวน์โหลดรายงานใบเสร็จรับเงินจาก PEAK** ราย 1 เดือน — ที่หน้า **รายรับ > ใบเสร็จรับเงิน** กดปุ่มม่วง **"พิมพ์รายงาน"** (มุมบนซ้าย ข้างปุ่ม "+ สร้างใบเสร็จรับเงิน") ได้ไฟล์ Excel ชื่อรายงาน = "รายงานใบเสร็จรับเงิน" มีคอลัมน์ **ลำดับที่ · เลขที่เอกสาร (RT-อออออ) · วันที่ออก · อ้างอิง** (คอลัมน์ "อ้างอิง" = เลขคำสั่งซื้อจากแพลตฟอร์ม เช่น 250726HYSRAP3) → sheet ชื่อ "รายงานใบเสร็จรับเงิน"
2. **ดาวน์โหลดคำสั่งซื้อจากแพลตฟอร์ม** ช่วงเวลาคาบเกี่ยวข้ามเดือน (เช่น 01/09–10/10) — บน Shopee: เมนู "คำสั่งซื้อของฉัน" > แท็บสถานะ (เช่น "กำลังจัดส่ง") > กดปุ่ม **"ดาวน์โหลด"** (มุมขวาบน) > popup **"ดาวน์โหลดคำสั่งซื้อ..."** เลือก **ช่วงเวลา** (เช่น 2025/08/11 – 2025/09/10) > กด **"ดาวน์โหลด"** (ปุ่มส้ม)
3. เอาคำสั่งซื้อแพลตฟอร์มไปวางเป็น Sheet 2 (เปลี่ยนชื่อ "Check Order") และ Sheet 3 ("Data Order")
4. **ลบแถวหัวตารางที่ Merge cell ออก** (สำคัญ — เซลล์ Merge จะทำให้สูตรพัง)
5. แทรกคอลัมน์ใหม่ (คอลัมน์ E) หัวว่า **"ตรวจสอบรายการเปิดเอกสาร"** (เน้นสีเหลือง)
6. ใส่สูตรในคอลัมน์ E ของ Check Order:
   `=XLOOKUP(A2, รายงานใบเสร็จรับเงิน!D:D, รายงานใบเสร็จรับเงิน!B:B, "ไม่พบรายการ")`
   (A2 = เลขคำสั่งซื้อแพลตฟอร์ม · D:D = คอลัมน์อ้างอิงในรายงาน PEAK · B:B = เลขที่เอกสารใบเสร็จ · ถ้าไม่เจอคืนค่า "ไม่พบรายการ")
7. รายการที่ขึ้น "ไม่พบรายการ" → ไปเช็คบนแพลตฟอร์มว่าเป็นออเดอร์ **คืนสินค้า/สูญหาย** (ระบบไม่สร้างเอกสารให้ = ปกติ) หรือเป็นออเดอร์ตกหล่นจริง → ต้องดึงเข้าเอง

วิธีที่ 2 — ตรวจแบบ manual:
1. ดาวน์โหลดรายงานแพลตฟอร์มให้ตรงกับ **ตั้งค่าการเชื่อมต่อ** — ที่ **ตั้งค่า > ตั้งค่าเชื่อมต่อระบบภายนอก > เชื่อมต่อแอปพลิเคชันภายนอก > Shopee** จะเห็นรายชื่อร้านค้าที่เชื่อมต่อ พร้อมปุ่มจุดสามจุด (⋮) → เมนู **รีเฟรชการเชื่อมต่อ / ตั้งค่าการเชื่อมต่อ / ยกเลิกการเชื่อมต่อ** (การเชื่อมต่อ Shopee มีอายุไม่เกิน 365 วัน ต้องรีเฟรชก่อนหมดอายุ)
2. เอาเลขคำสั่งซื้อไปค้นในช่อง **"อ้างอิง"** ของหน้าใบเสร็จ — ที่ **รายรับ > ใบเสร็จรับเงิน** กด **"ค้นหาขั้นสูง"** (ระดับ ขั้นสูง/ขั้นสูงมาก) ใส่ **วันที่ออก** (เช่น 01/08/2025–30/09/2025) และช่อง **อ้างอิง** ("ระบุเอกสารอ้างอิง ถ้ามี") > กด **"ค้นหา"**
3. ถ้าไม่พบ → ตรวจว่าตั้งค่าการเชื่อมต่อตรงกับข้อมูลที่ดาวน์โหลดหรือไม่ + เช็คสถานะคืน/สูญก่อนดึงเอง

**ฟิลด์สำคัญ**
- **หมายเลขคำสั่งซื้อ** (แพลตฟอร์ม) = คีย์หลักที่ใช้ join
- **อ้างอิง** (บนใบเสร็จ PEAK) = ช่องที่เก็บหมายเลขคำสั่งซื้อ
- คอลัมน์รายงานแพลตฟอร์มที่เห็น: หมายเลขคำสั่งซื้อ · สถานะการสั่งซื้อ (เช่น "สำเร็จแล้ว") · Hot Listing · สถานะการคืนเงิน/คืนสินค้า (N=ไม่คืน) · วันที่ทำการสั่งซื้อ · เวลาการชำระสินค้า · ช่องทางการชำระเงิน (บัตรเครดิต/บัตรเดบิต · เก็บเงินปลายทาง · QR พร้อมเพย์ · SPayLater · Shopee Credit Down Payment)

**การตั้งค่าสถานะที่ให้ระบบดึงมาสร้างเอกสาร (4 ตัวเลือก)**
1. ตั้งแต่สถานะที่ต้องจัดส่ง
2. ตั้งแต่สถานะกำลังจัดส่ง
3. ตั้งแต่สถานะจัดส่งสำเร็จ
4. **เฉพาะสถานะสำเร็จ** ← แนะนำ (ตัดออเดอร์คืน/สูญออก)

**การลงบัญชี (Dr/Cr):** ไม่มี — บทความนี้เป็นงานตรวจสอบ/กระทบยอด ไม่ได้ลงบัญชี

**เงื่อนไข/ข้อควรระวัง**
- ต้องดาวน์โหลดรายงาน **คาบเกี่ยวข้ามเดือน** (ออเดอร์จัดส่งสำเร็จเดือนถัดไปจะรับรู้รายได้เดือนถัดไป)
- ตั้งค่าสถานะการเชื่อมต่อ **ต้องตรงกับ** ช่วงสถานะที่โหลดรายงาน
- ออเดอร์ **คืน/สูญ** ระบบไม่สร้างเอกสาร = ปกติ ไม่ต้องตกใจ
- เซลล์ **Merge** ในหัวตารางทำให้สูตร XLOOKUP พัง → ต้องลบก่อน

### 2. บันทึกบัญชีหนี้สงสัยจะสูญ และหนี้สูญใน PEAK

**ใช้ทำอะไร**
บันทึกรายการปรับปรุงเกี่ยวกับลูกหนี้ที่เก็บไม่ได้ ผ่านเมนู **บัญชี > สมุดรายวัน (บัญชีรายวัน / JV)** — สมุดบัญชี "รายวันทั่วไป"
- **หนี้สงสัยจะสูญ** = ประมาณการลูกหนี้ที่คาดว่าจะเก็บไม่ได้ (ตั้งค่าเผื่อ)
- **หนี้สูญ** = ลูกหนี้ที่ติดตามทวงถามจนถึงที่สุดแล้วแต่ไม่ได้รับชำระ (ตัดจำหน่ายจริง)

**ขั้นตอน (จากภาพจริง)**
เอกสารที่บันทึกคือ **บัญชีรายวัน เลขที่ #JV-อออออออ** (เช่น JV-202401001) หัวเอกสารแยกแท็บ **ข้อมูลเอกสาร / ข้อมูลประวัติ** มีปุ่ม **พิมพ์** (ส้ม) และ **ตัวเลือก** (ม่วง) มุมขวาบน · สถานะแสดงจุดเขียว **"อนุมัติแล้ว"**
- ส่วนหัว **ข้อมูลบัญชี**: สมุดบัญชี = "รายวันทั่วไป" · วันที่รายการ (เช่น 31/12/2023) · เอกสารอ้างอิง · ใบกำกับภาษีอ้างอิง
- ส่วน **คำอธิบายรายการ**: ผู้ติดต่อ · คำอธิบายบันทึกบัญชี (เช่น "ตั้งสำรองหนี้สงสัยจะสูญ" / "ตัดจำหน่ายหนี้สูญ" / "กลับรายการค่าเผื่อหนี้สงสัยจะสูญ" / "หนี้สูญได้รับคืน")

### 3. แก้ไขหัก ณ ที่จ่าย บนใบแจ้งหนี้ — กรณีออกใบเสร็จแล้ว

**ใช้ทำอะไร**
แก้ยอด **หัก ณ ที่จ่าย** ที่บันทึกผิดบนใบแจ้งหนี้ ในกรณีที่ **ออกใบเสร็จรับเงินไปแล้ว** — ระบบล็อกเอกสารเพื่อความปลอดภัยทางบัญชี ต้อง "ปลดล็อก" ตามลำดับ: **ยกเลิกใบเสร็จ → ยกเลิกการรับชำระ → แก้ไขใบแจ้งหนี้ → บันทึก**

**ขั้นตอน (จากภาพจริง)**

เฟส 1 — ยกเลิกใบเสร็จ + ยกเลิกการรับชำระ:
1. ไป **รายรับ > ใบแจ้งหนี้** (หัวข้อ "ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้)") > แท็บ **"รับชำระแล้ว"** — แถบแท็บมี: ล่าสุด · ทั้งหมด · ร่าง · รออนุมัติ · รอรับชำระ · เกินเวลารับชำระ · **รับชำระแล้ว** · ยกเลิก · เอกสารอัตโนมัติ
2. แถวใบแจ้งหนี้ (เช่น IV-20260100003) คอลัมน์ "คำสั่ง" มีปุ่มน้ำเงิน **"ดูใบเสร็จ"** → กด
3. เข้าหน้า **ใบเสร็จรับเงิน/ใบกำกับภาษี เลขที่ #RT-อออออ** (สถานะเขียว "รับชำระแล้ว") → กดปุ่มม่วง **"ตัวเลือก"** (มุมขวาบน ข้างปุ่ม ส่งเอกสาร/พิมพ์) → dropdown มี: แก้ไข(🔒เทา) · คัดลอก(🔒เทา) · สร้างใบลดหนี้ · สร้างใบเพิ่มหนี้ · **ยกเลิกเอกสาร** · ล็อกเอกสาร → เลือก **"ยกเลิกเอกสาร"**
4. ยืนยันผ่าน pop-up → กด **"ยืนยัน"**
5. กลับไปที่ใบแจ้งหนี้เดิม → แท็บ **"ข้อมูลการชำระ"** → บล็อกการรับชำระเงินครั้งที่ 1 มุมขวาบนมีไอคอน **จุดสามจุด (⋮)** → เมนู **"ยกเลิก"** (การรับชำระ)
6. ยืนยัน popup **"ยืนยันยกเลิกการชำระเงิน"** → กดปุ่มฟ้า **"ยืนยัน"**

เฟส 2 — แก้ไข + บันทึก:
7. กลับแท็บ **"ข้อมูลเอกสาร"** → กด **"ตัวเลือก"** → ตอนนี้ **"แก้ไข"** กดได้แล้ว (ไม่ล็อก) → เลือก **แก้ไข**
8. ในฟอร์มแก้ไข ที่ตาราง **รายการ** คอลัมน์ **หัก ณ ที่จ่าย** กดลูกศร ▾ เลือกอัตราจาก dropdown (**1.5% · 2% · 3% · 5% · 10% · กำหนดเอง**) หรือแก้จำนวนเงิน → กดปุ่มม่วง **"บันทึก"** (มุมขวาล่าง)

**หมายเหตุ:** ถ้าคอลัมน์ "หัก ณ ที่จ่าย" ไม่แสดง → เปิด **ตั้งค่าการแสดงผล** เปิด toggle "หัก ณ ที่จ่าย" ให้เป็น "เปิด" แล้วบันทึกก่อน

**ฟิลด์สำคัญ (บนฟอร์มใบแจ้งหนี้)**
- ตารางรายการ: **สินค้า/บริการ · บัญชี (เลขบัญชี เช่น 410101) · จำนวน · หน่วย · ราคา/หน่วย · ส่วนลด/หน่วย · ภาษี (เช่น 7%) · มูลค่าก่อนภาษี · หัก ณ ที่จ่าย**
- ประเภทราคา = แยกภาษี · ประเภทใบกำกับภาษี = เป็นใบกำกับภาษี · ราคานี้เป็นค่าเงิน = THB
- เงินมัดจำ: ปุ่ม "+ เลือกเงินมัดจำ"

**สถานะที่เปลี่ยน**
- ใบเสร็จ (RE): รับชำระแล้ว → **ยกเลิก**
- การรับชำระ: บันทึกแล้ว → **ยกเลิก**
- ใบแจ้งหนี้ (IV): ล็อก(รับชำระแล้ว) → ปลดล็อก(รอรับชำระ) → ล็อกใหม่หลังบันทึก
- (stepper บนหัว: ใบแจ้งหนี้✓ – ใบเสร็จรับเงิน – ใบกำกับภาษี · เดิมติ๊กครบ 3 ขั้น พอยกเลิกใบเสร็จ ขั้น RE/TIV จะหลุด)

**การลงบัญชี:** การยกเลิกใบเสร็จ+การรับชำระ = กลับรายการบันทึกบัญชีเดิม (สมุดรับเงิน RV) ; บันทึกใหม่ = สร้างรายการบัญชีที่แก้ WHT ถูกต้อง

**ข้อควรระวัง**
- **ต้องยกเลิก 2 ชั้น** (ทั้งใบเสร็จ + การรับชำระ) — ยกเลิกอย่างเดียวไม่พอ ปุ่ม "แก้ไข" ยังเทา/ล็อกอยู่
- ต้องเปิด toggle "หัก ณ ที่จ่าย" ในตั้งค่าการแสดงผลก่อน ถ้าช่องไม่โผล่
- วงจร ล็อก→ปลดล็อก→ล็อกใหม่ตอนบันทึก

### 4. แก้ไขหัก ณ ที่จ่าย บนใบแจ้งหนี้ — กรณียังไม่ได้ออกใบเสร็จรับเงิน

**ใช้ทำอะไร**
แก้ยอด **หัก ณ ที่จ่าย** ที่บันทึกผิด กรณี **รับชำระเงินแล้ว แต่ยังไม่ได้ออกใบเสร็จรับเงิน** — เคสง่ายกว่าข้อ 3 เพราะยังไม่มีใบเสร็จให้ยกเลิก แค่ยกเลิก "การรับชำระ" ชั้นเดียวก็แก้ได้

**ขั้นตอน (จากภาพจริง)**

เฟส 1 — ยกเลิกการรับชำระ:
1. ไป **รายรับ > ใบแจ้งหนี้** > แท็บ **"รับชำระแล้ว"**
2. กดเลขที่เอกสารใบแจ้งหนี้ (เช่น IVT-20260100001) — สังเกตคอลัมน์คำสั่งเป็นปุ่มเขียว **"สร้างใบเสร็จ"** (แปลว่ายังไม่ออกใบเสร็จ ต่างจากข้อ 3 ที่เป็น "ดูใบเสร็จ")
3. แท็บ **"ข้อมูลการชำระ"** → บล็อกการรับชำระเงินครั้งที่ 1 (แสดง "รับชำระเต็มจำนวนแล้ว" + ฝั่งขวา "ยังไม่ออกใบเสร็จ" + ปุ่มเขียว "ออกใบเสร็จ") → กดไอคอน **จุดสามจุด (⋮)** มุมขวาบนของบล็อก → เมนู **"ยกเลิก"**
4. กด **"ยืนยัน"** ใน popup "ยืนยันยกเลิกการชำระเงิน"

เฟส 2 — แก้ไข + บันทึก:
5. กลับแท็บ **"ข้อมูลเอกสาร"** — สถานะเปลี่ยนเป็นจุดฟ้า **"รอรับชำระ"** → กด **"ตัวเลือก"** → dropdown มี: **แก้ไข** · คัดลอก · สร้างใบลดหนี้ · สร้างใบเพิ่มหนี้ · สร้างใบวางบิล · ยกเลิกเอกสาร · ล็อกเอกสาร → เลือก **"แก้ไข"** (ไม่ถูกล็อก เพราะไม่มีใบเสร็จมาล็อกไว้)
6. ในฟอร์ม ที่ตาราง **รายการ** คอลัมน์ **หัก ณ ที่จ่าย** (มีไอคอนดินสอ ✎) แก้อัตรา/จำนวน
7. กด **"บันทึก"**

**หมายเหตุ (gotcha):** ถ้าคอลัมน์ "หัก ณ ที่จ่าย" ไม่แสดง → กดปุ่ม **"ตั้งค่าการแสดงผล"** (มุมขวา) เปิด popup **"ตั้งค่าการแสดงผลข้อมูล"** → เลื่อนไปที่ toggle **"หัก ณ ที่จ่าย"** เปิดเป็น **"เปิด"** → กด **"บันทึก"** (toggle อื่นในภาพ: หมายเหตุ · แนบไฟล์ · แท็ก · สินค้า/บริการ · บัญชี · จำนวน · ส่วนลด/หน่วย · ภาษี · มูลค่าก่อนภาษี — มีรูปแบบใหม่/รูปแบบเก่าให้เลือก)

**ฟิลด์สำคัญ:** เหมือนข้อ 3 (ตารางรายการ + คอลัมน์หัก ณ ที่จ่าย) · ประเภทใบกำกับภาษี = "ไม่ใช่ใบกำกับภาษี" (ในเคสตัวอย่าง IVT นี้)

**สถานะที่เปลี่ยน**
- การรับชำระ: บันทึกแล้ว → **ยกเลิก**
- ใบแจ้งหนี้: รับชำระแล้ว → **รอรับชำระ** (แก้ไขได้)
- stepper: ใบเสร็จรับเงิน (RE) และใบกำกับภาษี (TIV) ยังไม่ติ๊ก (มีปุ่ม `+` รอสร้าง)

**การลงบัญชี:** ยกเลิกการรับชำระ = กลับรายการสมุดรับเงิน (RV-อออออ) ; แก้แล้วบันทึกใหม่จะได้รายการบัญชี WHT ที่ถูกต้อง

**ข้อควรระวัง / ต่างจากข้อ 3**
- เคสนี้ **ยกเลิกชั้นเดียว** (เฉพาะการรับชำระ) — ไม่ต้องยกเลิกใบเสร็จ เพราะยังไม่ได้ออก
- ปุ่ม "แก้ไข" กดได้ทันทีหลังยกเลิกการรับชำระ (ไม่เทา)
- ยังต้องเปิด toggle "หัก ณ ที่จ่าย" ในตั้งค่าการแสดงผล ถ้าช่องไม่โผล่
- อัตรา WHT ให้เลือก: 1.5% · 2% · 3% · 5% · 10% · กำหนดเอง (เหมือนข้อ 3)

### สร้างใบเสร็จรับเงิน ใน PEAK (id 5981304)

**ใช้ทำอะไร:** ออกใบเสร็จรับเงิน (หรือ ใบเสร็จรับเงิน/ใบกำกับภาษี) แบบ**ไม่อ้างอิง**ต้นทางใด ๆ — ใช้เมื่อรับเงินจากลูกค้าและออกเอกสารในขั้นตอนเดียว (ไม่ได้ทำใบเสนอราคา/ใบแจ้งหนี้มาก่อน)

**ขั้นตอน (จากภาพจริง):**
1. เมนูบนสุด **รายรับ** ⟶ ขยายเมนูย่อย **ใบเสร็จรับเงิน** ⟶ **+ สร้าง** (ปุ่มม่วง) — เมนู รายรับ มีลำดับ: ดูภาพรวม / ใบเสนอราคา / ใบรับเงินมัดจำ / ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้) / **ใบเสร็จรับเงิน** / ใบกำกับภาษีขาย / ใบลดหนี้ / ใบเพิ่มหนี้ / ใบวางบิล / นำเข้าเอกสาร
2. หน้าฟอร์มหัวเรื่อง **"สร้างใบเสร็จรับเงิน / ใบกำกับภาษี"** แบ่งเป็น section (มี ✓ วงกลมเขียวหน้าแต่ละ section):
   - **ข้อมูลลูกค้า** — เลือก **ชื่อลูกค้า** (บังคับ *) จาก dropdown (มีลิงก์ "เลือกรายการล่าสุดของผู้ติดต่อนี้") → auto ดึง **ที่อยู่** + **เบอร์โทร** · ช่อง **วันที่ออก** (บังคับ *) · ช่อง **อ้างอิง** (ระบุเอกสารอ้างอิงถ้ามี) · ช่อง **เลขที่เอกสาร** (แก้ได้ผ่านไอคอนฟันเฟือง ⚙) แสดง `RT-20250800004`
   - **กลุ่มจัดประเภท** — dropdown 4 ช่อง: พนักงานขาย / ช่องทางการขาย / แผนก / โครงการ (ค่าเริ่ม "ยังไม่ระบุ")
   - **ข้อมูลราคาและภาษี** — **ประเภทราคา** (บังคับ *; เช่น "แยกภาษี") · **การออกใบกำกับภาษี** = สวิตช์ toggle "ออกใบกำกับภาษี" (เปิด=ออกใบเสร็จ/ใบกำกับภาษี) · **ราคานี้เป็นค่าเงิน** (เช่น THB)
   - **รายการ** — ตารางคอลัมน์: **สินค้า/บริการ · บัญชี · จำนวน · ราคา/หน่วย · ส่วนลด/หน่วย · ภาษี · มูลค่าก่อนภาษี · หัก ณ ที่จ่าย** (แต่ละแถวเลือกสินค้า P0000x + ผังบัญชีรายได้ 4xxxxx เช่น 410201 รายได้จากการให้บริการ) · ปุ่ม "+ เพิ่มรายการใหม่" / "+ เพิ่มคำอธิบายรายการ" · ตั้ง "แสดง 50 รายการ / หน้า"
3. **ส่วนรับชำระเงิน** (section "รับชำระเงิน" มี radio **พื้นฐาน / ขั้นสูง**):
   - หัวข้อ **"รับชำระเงินครั้งที่ 1"** · ช่อง **วันที่ชำระ** (บังคับ *)
   - ☑ **ช่องทางการรับชำระเงิน** → **รับชำระโดย** (dropdown บัญชี เช่น "ไทยพาณิชย์ - 1234567888907777 SCB") · **จำนวนเงินที่รับชำระ** · **หมายเหตุ**
   - ☑ **ถูกหัก ณ ที่จ่าย** → **การนำส่ง**: ◉ ปกติ / ○ e-Withholding Tax · **อัตราภาษี** (dropdown เช่น 3%) · **มูลค่าถูกหัก ณ ที่จ่าย** (เช่น 90.00)
   - แถบสรุปมุมขวา: **รับชำระด้วยเงินรวม** / **ถูกหัก ณ ที่จ่าย** / กล่องเขียว **รับชำระรวมทั้งสิ้น XXX บาท** / **ต้องรับชำระเงินอีก 0.00 บาท** (ต้องเป็น 0 จึงอนุมัติได้)
4. แถบล่างสุด (sticky · ซ้ายโชว์ "ใบเสร็จรับเงิน RT-..."): ปุ่ม **ยกเลิก** (ขาว) · **บันทึกร่าง** (ม่วง มี dropdown) · **อนุมัติใบเสร็จรับเงิน** (เขียว มุมขวาสุด มี dropdown) → กดอนุมัติเพื่อออกเอกสารจริง

**ฟิลด์สำคัญ:** ชื่อลูกค้า* · วันที่ออก* · เลขที่เอกสาร (RT-) · ประเภทราคา* · toggle ออกใบกำกับภาษี · รับชำระโดย (ช่องทาง) · หัก ณ ที่จ่าย
**สถานะ:** ร่าง (กด "บันทึกร่าง") → **รับชำระแล้ว** (กด "อนุมัติใบเสร็จรับเงิน") — หลังอนุมัติแถบไทม์ไลน์แสดงสถานะแทนปุ่มสร้าง
**การลงบัญชี (Dr/Cr):** อนุมัติแล้วสร้างบันทึกบัญชีชุดขาย (ตัวอย่างเลข **SV-2026xxxxx**) — **Dr เงินสด/ธนาคาร** (เท่ายอดรับจริง) + **Dr ภาษีถูกหัก ณ ที่จ่าย** (สินทรัพย์ 90.00) / **Cr รายได้** (มูลค่าก่อนภาษี) / **Cr ภาษีขาย** (VAT 7%) — คลิกเลขบันทึกบัญชีดูรายละเอียดได้
**เงื่อนไขแก้/ยกเลิก:** หลังอนุมัติแก้ได้เฉพาะ **เลขที่เอกสาร · อ้างอิง · กลุ่มจัดประเภท · รายการสินค้า/บริการ · หมายเหตุเอกสาร** — **แก้วันที่โดยตรงไม่ได้** (ต้องยกเลิกแล้วคัดลอกเพื่อเปลี่ยนวันที่) · ช่อง "ส่วนลดรวม" จะแสดงเมื่อการระบุภาษีของทุกแถวเหมือนกัน · ถ้าไม่เห็นคอลัมน์บัญชีให้เปิดผ่าน "ตั้งค่าการแสดงผล"

### สร้างใบเสร็จรับเงิน ต่อจากใบแจ้งหนี้ ใน PEAK (id 6003552)

   - radio **พื้นฐาน / ขั้นสูง** · **วันที่รับชำระ** (บังคับ *)
   - ☑ **ช่องทางการรับชำระเงิน**: **รับชำระโดย** (บัญชีธนาคาร) · **จำนวนเงินที่รับชำระ** · **หมายเหตุ**
   - ☐ **ถูกหัก ณ ที่จ่าย** (ติ๊กถ้าลูกค้าหัก)
   - สรุป: **รับชำระรวมทั้งสิ้น XXX บาท / ต้องรับชำระเงินอีก 0.00** → ปุ่ม **ยกเลิก · รับชำระเงิน** (มี dropdown) → ระบบ**ลงบันทึกบัญชีอัตโนมัติ**
3. **หลังรับชำระ (สถานะ "รับชำระแล้ว"):** แท็บข้อมูลการชำระโชว์ **"รับชำระเต็มจำนวนแล้ว"** + บล็อก "รับชำระเงินครั้งที่ 1" (ช่องทาง + ยอด) + **บันทึกบัญชี: RV-2026xxxxx** + ข้อความแดง **"ยังไม่ออกใบเสร็จ"** + ปุ่มเขียว **"ออกใบเสร็จ"** → กดปุ่มนี้
4. เข้าฟอร์ม **"สร้างใบเสร็จรับเงิน / ใบกำกับภาษี"** (ดึงข้อมูลจากใบแจ้งหนี้อัตโนมัติ):
   - หัวโชว์ **อ้างอิงจากระบบ** = เลขใบแจ้งหนี้ต้นทาง (`IV-20260100003`) · **อ้างอิง** · **เลขที่เอกสาร** = `RT-20260100002`
   - ข้อมูลลูกค้า/รายการ/ราคา/ภาษี ดึงมาครบ (P00002 สินค้า B, บัญชี 410101 รายได้จากการขาย, จำนวน/ราคา/ภาษี 7%) · **สรุปข้อมูล**: ส่วนลดรวม / มูลค่าที่คำนวณภาษี 7% / ภาษีมูลค่าเพิ่ม 7% / **จำนวนเงินทั้งสิ้น** (กล่องน้ำเงินเข้ม)
   - toggle **การออกใบกำกับภาษี = ออกใบกำกับภาษี** (เปิดถ้าต้องการใบกำกับภาษีพ่วง)
   - แก้ได้เฉพาะ **เลขที่เอกสาร** + **วันที่ออก** (ข้อจำกัด: ตั้งวันที่**ก่อนวันออกใบแจ้งหนี้ไม่ได้**) + หมวดหมู่จัดประเภท
5. กด **"อนุมัติใบเสร็จรับเงิน"** (เขียว มุมขวาล่าง)

**ความสามารถพิเศษ:** รวมใบแจ้งหนี้หลายใบเป็นใบเสร็จใบเดียว → ใช้ผ่านฟีเจอร์ **ใบวางบิล (BN)** (ดูหัวข้อ id 6841140)
**สถานะ:** ใบแจ้งหนี้: รอรับชำระ → รับชำระแล้ว · ใบเสร็จ: ยังไม่ออกใบเสร็จ → (อนุมัติ) รับชำระแล้ว
**การลงบัญชี (Dr/Cr):** ใบแจ้งหนี้ลงขายไว้แล้ว (Dr ลูกหนี้ / Cr รายได้ / Cr ภาษีขาย) · ตอน **"รับชำระเงิน"** สร้าง **RV-...** = **Dr เงินสด/ธนาคาร + Dr WHT / Cr ลูกหนี้การค้า** · การ "ออกใบเสร็จ" ไม่ลงบัญชีซ้ำ (แค่ออกเอกสารหลักฐานรับเงิน)
**เงื่อนไข:** ออกใบเสร็จจากใบแจ้งหนี้ได้ต่อเมื่อรับชำระแล้ว · ตั้งวันที่ใบเสร็จก่อนวันใบแจ้งหนี้ไม่ได้

### สร้างใบเสร็จรับเงินฉบับร่าง ต่อจากใบแจ้งหนี้ ใน PEAK (id 13963013)

**ใช้ทำอะไร:** สร้าง **ใบเสร็จรับเงินฉบับร่าง (Draft Receipt)** ต่อจากใบแจ้งหนี้ **ก่อน**บันทึกรับชำระเงินจริง — ใช้ให้ลูกค้าตรวจสอบหรือดำเนินการภายในองค์กร (เพราะ PEAK **ไม่อนุญาต**ให้ออกใบเสร็จจากหน้าข้อมูลการชำระในใบแจ้งหนี้ถ้ายังไม่บันทึกรับชำระ — ต้องสร้างเป็น "ร่าง" แทน)

**ขั้นตอน (จากภาพจริง):**
1. **รายรับ ⟶ ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้) ⟶ ดูทั้งหมด** → เลือกใบแจ้งหนี้ที่ต้องการ
2. ที่แถบ stepper ด้านบน กดกล่อง **"ใบเสร็จรับเงิน (RE)"** (หรือปุ่ม + ) → เข้าฟอร์มสร้างใบเสร็จ
3. ในส่วน **การรับชำระเงิน** เลือก **วิธีการรับชำระเงิน** (ช่องทาง/บัญชีธนาคาร) — เหมือนฟอร์มปกติ
4. กดปุ่ม **"บันทึกร่าง"** (แทนการกด "อนุมัติใบเสร็จรับเงิน")

**สถานะและผลลัพธ์ (จากภาพจริง):**
- ที่แถบ stepper กล่อง **"ใบเสร็จรับเงิน (RE)" เปลี่ยนเป็น สีส้ม** = **ฉบับร่าง** (เทียบ IV=น้ำเงิน, TIV=เขียว)
- หน้า "ข้อมูลการชำระ" ของใบแจ้งหนี้: **บันทึกบัญชี = "-"** (ยัง**ไม่มีบันทึกบัญชี**) แม้จะโชว์เลขที่ใบเสร็จ `RE-20260300002` และช่องทางรับชำระที่เลือกไว้
- คลิกดูเอกสารร่างเพื่อตรวจรายละเอียดได้

**ข้อควรระวังสำคัญ (gotcha):**
1. **ปิดการแสดงผลช่องทางการเงินบนใบร่าง:** เปิด Online view ของใบเสร็จร่างจะโชว์ธนาคารที่เลือกตอนสร้างร่าง → ควรปิดโดย: กด **"แก้ไข"** → กด **ไอคอนดินสอ ✏️ สีฟ้า** ที่ส่วนชำระเงิน → กดปุ่มสีฟ้าให้**เปลี่ยนเป็นสีเทา** (ปิด) → กด **"บันทึก"**
2. **รายการกระทบยอด (Reconciliation):** ที่ **การเงิน ⟶ เงินสด/ธนาคาร ⟶ (บัญชี)** ⟶ แท็บ "รายการบันทึก" ค้นด้วยเลขเอกสารในช่วงวันที่ จะขึ้น **"ไม่พบรายการ"** — เพราะเอกสารยัง**เป็นฉบับร่าง** จึงไม่สร้างรายการกระทบยอดที่ธนาคาร

**ความแตกต่างร่าง vs อนุมัติ:** ร่าง = **ไม่กระทบบัญชี/ไม่กระทบยอดธนาคาร** จนกว่าจะมีการบันทึกรับชำระเงินจริง · เมื่อบันทึกรับชำระจริงแล้ว ใบเสร็จจะเปลี่ยนสถานะเป็น **อนุมัติ** และระบบลงบันทึกบัญชี (RV) ตามปกติ
**สถานะ:** ฉบับร่าง (ส้ม · ไม่มีบัญชี) → อนุมัติ (เมื่อรับชำระจริง)

### สร้างใบเสร็จรับเงิน ต่อจากใบเสนอราคา ใน PEAK (id 8703065)

**ใช้ทำอะไร:** สร้างใบเสร็จรับเงินโดยดึงข้อมูลจาก **ใบเสนอราคา (QO)** โดยตรง (**ข้ามใบแจ้งหนี้**) — ทำได้เมื่อใบเสนอราคา**ถูกตอบรับ (ยอมรับ) แล้ว**เท่านั้น

**ขั้นตอน (จากภาพจริง):**
1. **ขั้นตอนที่ 1 — ตอบรับใบเสนอราคา:** **รายรับ ⟶ ใบเสนอราคา ⟶ รอตอบรับ** → เลือกรายการ → กด **"ตอบรับ"** → popup **"ตอบกลับใบเสนอราคาแล้ว"**: radio ◉ **ยอมรับ** / ○ ปฏิเสธ + **วันที่ยอมรับ** → กด **"ยืนยัน"** (แท็บสถานะของใบเสนอราคา: ล่าสุด / ทั้งหมด / ร่าง / รออนุมัติ / รอตอบรับ / พ้นกำหนด / **ยอมรับแล้ว** / ออกบางส่วน / ออกครบแล้ว / ปฏิเสธแล้ว)
2. **ขั้นตอนที่ 2 — สร้างใบเสร็จ (2 วิธี):**
   - **วิธีที่ 1 (จากลิสต์สถานะ):** ไปแท็บ **"ยอมรับแล้ว"** → ที่คอลัมน์ **คำสั่ง** ของแถวนั้นกดปุ่ม **"สร้างใบแจ้งหนี้ ▾"** → เมนู dropdown มี: สร้างใบรับเงินมัดจำ / **สร้างใบเสร็จรับเงิน** → เลือก "สร้างใบเสร็จรับเงิน"
   - **วิธีที่ 2 (จากตัวเอกสาร):** เปิดใบเสนอราคา → ที่ stepper กดปุ่ม **"+"** บนกล่อง **ใบเสร็จรับเงิน (RE)** (QO ⟶ IV(+) ⟶ **RE(+)** ⟶ TIV) — ถ้ายังไม่ได้ตอบรับ จะมี popup **"สร้างใบเสร็จรับเงินจากใบเสนอราคา #QO-..."** เตือน "กรุณาระบุวันที่ลูกค้าตอบรับใบเสนอราคาก่อนสร้างเอกสาร" + ช่อง **วันที่ยอมรับ** → กด **ยืนยัน**
3. **ขั้นตอนที่ 3 — เลือกวิธีสร้าง (popup):**
   - ◉ **ตามใบเสนอราคา** — สร้างเอกสารตาม**ยอดทั้งหมด**ของใบเสนอราคา
   - ○ **แบ่งตามสินค้า/บริการแต่ละแถว** — คำนวณจาก**ยอดคงเหลือตามแต่ละแถว** (ทยอยออกได้) → กด **ยืนยัน**
4. **ขั้นตอนที่ 4 — ระบุช่องทางรับชำระ:** ตรวจข้อมูลเอกสารใบเสร็จ (ดึงจาก QO) → เลือก **ช่องทางเพื่อรับชำระเงิน** → กด **"อนุมัติใบเสร็จรับเงิน"**

**ฟิลด์สำคัญ:** วันที่ยอมรับ (ตอบรับ QO) · เลขที่เอกสาร (RT-) · ช่องทางรับชำระ · toggle ออกใบกำกับภาษี
**สถานะ:** ใบเสนอราคา: รอตอบรับ → ยอมรับแล้ว → (สร้างใบเสร็จ) ออกบางส่วน/ออกครบแล้ว · ใบเสร็จ: (อนุมัติ) **รับชำระแล้ว**
**การลงบัญชี (Dr/Cr):** ใบเสนอราคา**ไม่ลงบัญชี** (เป็นเอกสารเสนอราคา) · เมื่ออนุมัติใบเสร็จจึงสร้างบันทึกบัญชี (ชุดขาย + รับเงิน) = **Dr เงินสด/ธนาคาร + Dr WHT / Cr รายได้ / Cr ภาษีขาย** — คลิกเลขที่บันทึกบัญชีดูรายละเอียดได้
**เงื่อนไขแก้/ยกเลิก:** เมื่อใบเสร็จสถานะ **"รับชำระแล้ว" แก้ข้อมูลไม่ได้** → ต้อง **ยกเลิกเอกสาร** แล้ว **คัดลอก**มาสร้างใบใหม่

### สร้างใบเสร็จรับเงิน ต่อจากใบวางบิล ใน PEAK (id 6841140)

**ใช้ทำอะไร:** สร้างใบเสร็จรับเงินโดยดึงข้อมูลจาก **ใบวางบิล (BN)** — ใช้เมื่อรวมใบแจ้งหนี้หลายใบไว้ในใบวางบิลใบเดียวแล้วรับเงินรวมทีเดียว → ออกใบเสร็จใบเดียว · ทำได้เมื่อ**รับชำระเงินแล้ว**เท่านั้น (ยังไม่รับชำระ = ออกใบเสร็จไม่ได้)

**ขั้นตอน (จากภาพจริง):**
1. **รายรับ ⟶ ใบวางบิล ⟶ ดูทั้งหมด** → เลือกใบวางบิล (เช่น `BN-20260100002`) · stepper: **BN(น้ำเงิน) ⟶ RE(+) ⟶ TIV** · แท็บ **ข้อมูลเอกสาร** โชว์ **รายการ** = ตารางใบแจ้งหนี้ที่รวมไว้ (คอลัมน์: ลำดับที่ · เลขที่เอกสาร · วันที่ออก · วันที่ครบกำหนด · มูลค่าสุทธิ · จำนวนเงินวางบิล · หัก ณ ที่จ่าย — เช่น `IV-20260100001` 5,350.00 / `IV-20260100004` 2,129.30)
2. **กรณี 3.1 — ยังไม่รับชำระ** (สถานะ "รอรับชำระ" หรือ "เกินเวลารับชำระ"): ไปแท็บ **"ข้อมูลการชำระ"** → กด **"รับชำระ"** → popup **"รับชำระเงิน #BN-..."** (หัวบอก "มูลค่าลูกหนี้ที่สามารถรับชำระได้ทั้งสิ้น XXX บาท") — มีโหมด **พื้นฐาน / ขั้นสูง**:
   - **วันที่รับชำระ** (บังคับ *)
   - ☑ **ช่องทางการรับชำระเงิน**: **รับชำระโดย** (บัญชีธนาคาร) · **จำนวนเงินที่รับชำระ** · **หมายเหตุ** · ลิงก์ **"+ เพิ่มช่องทางการรับชำระเงิน"** (รับหลายช่องทาง)
   - (เฉพาะโหมด **ขั้นสูง**) ☐ **ค่าธรรมเนียม หรือรายการปรับปรุง** (ค่าธรรมเนียมธนาคาร/ปรับปรุง) · ☐ **ตัดชำระกับเอกสาร** (อ้างอิงตัดกับเอกสารอื่น)
   - ☑ **ถูกหัก ณ ที่จ่าย**: การนำส่ง ◉ ปกติ / ○ e-Withholding Tax · **อัตราภาษี** (เช่น 3%) · **มูลค่าถูกหัก ณ ที่จ่าย** (เช่น 150.00)
   - สรุป: รับชำระด้วยเงินรวม / ถูกหัก ณ ที่จ่าย / **รับชำระรวมทั้งสิ้น XXX** / ต้องรับชำระเงินอีก 0.00 → ปุ่ม **ยกเลิก · รับชำระเงิน ▾** → ระบบ**บันทึกบัญชีอัตโนมัติ**
3. **กรณี 3.2 — รับชำระแล้ว:** ไปแท็บ "ข้อมูลการชำระ" (สถานะ "รับชำระแล้ว" · โชว์ **บันทึกบัญชี: RV-2026xxxxx** · ข้อความแดง "ยังไม่ออกใบเสร็จ") → กดปุ่มเขียว **"ออกใบเสร็จ"**
4. **ขั้นตอนที่ 4 — อนุมัติ:** ระบบดึงข้อมูลจากใบวางบิล **แก้ได้เฉพาะ 2 ฟิลด์**: **เลขที่เอกสาร** + **วันที่ออก** (ควรตรงกับวันรับชำระ) · ถ้าต้องออกใบกำกับภาษีให้**ติ๊ก "ออกใบกำกับภาษี"** → กด **"อนุมัติใบเสร็จรับเงิน"**

**ผลลัพธ์:** ได้ **"ใบเสร็จรับเงิน"** (หรือ **"ใบเสร็จรับเงิน/ใบกำกับภาษี"** ถ้าเปิดใบกำกับภาษี) ครอบคลุมยอดรวมของใบแจ้งหนี้ทุกใบในใบวางบิล
**สายเอกสาร:** ใบแจ้งหนี้ (IV หลายใบ) ⟶ **ใบวางบิล (BN)** ⟶ **ใบเสร็จรับเงิน (RE)** ⟶ ใบกำกับภาษี (TIV)
**สถานะ:** ใบวางบิล: รอรับชำระ/เกินเวลารับชำระ → รับชำระแล้ว · ใบเสร็จ: ยังไม่ออกใบเสร็จ → (อนุมัติ) ออกแล้ว
**การลงบัญชี (Dr/Cr):** การกด **"รับชำระเงิน"** สร้าง **RV-2026xxxxx** = **Dr เงินสด/ธนาคาร (ยอดรับจริง) + Dr ภาษีถูกหัก ณ ที่จ่าย (WHT) / Cr ลูกหนี้การค้า** (ปิดยอดลูกหนี้ของทุกใบแจ้งหนี้ในใบวางบิล) · การออกใบเสร็จไม่ลงบัญชีซ้ำ

### บทความ 2 · สร้างใบเสร็จรับเงิน/ใบกำกับภาษี ใน PEAK (id 6022651 · ฉบับเน้นฟอร์มสร้าง + toggle ภาษี)

### บทความ 3 · ยกเลิกใบเสร็จรับเงิน ใน PEAK (id 6012814)

**ใช้ทำอะไร:** เมื่อออกใบเสร็จ/ใบกำกับภาษีผิด **แก้มูลค่าสุทธิ · ราคา/หน่วย · จำนวน ไม่ได้** → ต้อง **ยกเลิกเอกสาร** แล้วสร้างใหม่ (สามารถนำเลขเอกสารเดิมกลับมาใช้ได้).

**ขั้นตอน (จากภาพจริง):**
1. เมนู `รายรับ > ใบเสร็จรับเงิน > ดูทั้งหมด` → เข้าหน้ารายการ "ใบเสร็จรับเงิน" (แท็บ: ล่าสุด · ทั้งหมด · **ร่าง (n)** · **รออนุมัติ (n)** · รับชำระแล้ว · **ยกเลิก** · เอกสารอัตโนมัติ · ตัวกรอง วันที่ออก + ช่องค้นหา + ค้นหาขั้นสูง · คอลัมน์: เลขที่เอกสาร · ลูกค้า · วันที่ออก · มูลค่าสุทธิ · สถานะ) → คลิกเลขที่เอกสารที่จะยกเลิก.
2. ที่หน้ารายละเอียด กดปุ่ม **ตัวเลือก** (ปุ่มม่วง มุมขวาบน) → dropdown เด้งออกมี: **แก้ไข** (เทา/ล็อก 🔒 เพราะอนุมัติแล้ว) · **คัดลอก** · **สร้างใบลดหนี้** · **สร้างใบเพิ่มหนี้** · **ยกเลิกเอกสาร** · **ล็อกเอกสาร** → กด **ยกเลิกเอกสาร**.
3. สถานะเอกสารเปลี่ยนเป็น **ยกเลิกแล้ว** (ในหน้ารายการ แท็บ **ยกเลิก** จะแสดงสถานะ ● ยกเลิก จุดเทา).

**การลงบัญชี (Dr/Cr):** เมื่อยกเลิก ระบบ **กลับรายการบัญชีให้อัตโนมัติ** (reverse) — สมุดรายวัน (RV) ที่บันทึกไว้ตอนออกใบเสร็จจะถูกทำเครื่องหมาย "ยกเลิก". เท่ากับล้างเดบิต/เครดิตที่บันทึกไว้ (กลับด้านรายการรับเงิน/รายได้/VAT ขาย).

**เอกสารที่เชื่อมโยง:** ถ้าใบเสร็จนี้อ้างอิงจากใบแจ้งหนี้/ใบเสนอราคา → **การยกเลิก/ลบใบเสร็จจะไม่มีผลกับใบแจ้งหนี้หรือใบเสนอราคา** (ต้นทางยังอยู่).

**เงื่อนไข/ข้อจำกัด (ยกเลิกไม่ได้ ให้ดูสัญลักษณ์ล็อกในแท็บ "ข้อมูลการชำระ"):**
- **ล็อกสีน้ำตาล 🔒** = ถูกล็อกโดยแบบภาษี **ภ.พ.30** → ต้องยกเลิกแบบ ภ.พ.30 ของเดือนนั้นก่อน.
- **ล็อกสีฟ้า 🔒** = ถูกล็อกโดย **การกระทบยอดธนาคาร (bank reconcile)** → ต้องยกเลิกการกระทบยอดก่อน.
- **ล็อกสีแดง 🔒** = ถูกล็อกโดย **งวดบัญชี (period)** → ต้องปลดล็อกงวดที่ ตั้งค่า ก่อน.
- **ยกเลิกทีละหลายใบพร้อมกันไม่ได้** (ต้องทำทีละใบ · ยกเว้นไฟล์ที่นำเข้าจาก Excel).
- เลขเอกสารเดิมนำกลับมาใช้กับใบเสร็จใหม่ได้ผ่าน **แก้ไขเลขที่เอกสาร**.

**สถานะ:** รับชำระแล้ว → **ยกเลิก** (ยกเลิกแล้ว).

### บทความ 4 · แก้ไขใบเสร็จรับเงินส่วนใดได้บ้าง ใน PEAK (id 14073987)

**ใช้ทำอะไร:** อธิบายว่าหลังอนุมัติแล้ว **แก้ฟิลด์ใดได้ / ฟิลด์ใดแก้ไม่ได้** ในใบเสร็จรับเงิน.

**ขั้นตอน (จากภาพจริง):** เข้าหน้ารายละเอียดใบเสร็จ → กด **ไอคอนดินสอสีฟ้า (✏️)** ที่วางอยู่ข้างฟิลด์ที่แก้ได้ (ในภาพมีวงกลมแดงชี้ที่ดินสอตรงหัวเอกสาร · ข้าง อ้างอิง · ข้าง กลุ่มจัดประเภท · ข้าง รายการ) → แก้แล้วบันทึก.

**ฟิลด์ที่แก้ได้ (กรณีใบเสร็จแบบแยก / ออกเดี่ยว):**
- เลขที่เอกสาร · อ้างอิง · กลุ่มจัดประเภท (พนักงานขาย/ช่องทาง/แผนก/โครงการ) · **รายการ** (ชื่อสินค้า/บริการ · คำอธิบาย · ผังบัญชี — แก้เฉพาะข้อความ/บัญชี ไม่ใช่จำนวนเงิน) · หมายเหตุสำหรับลูกค้า · แนบไฟล์ในเอกสาร · แท็ก
- ในแท็บ **ข้อมูลการชำระ**: วันที่ชำระ · ช่องทางการรับชำระเงิน · ยอดเงินที่รับชำระ · ค่าธรรมเนียม

**กรณีใบเสร็จที่อ้างอิงจากเอกสารอื่น (เช่น สร้างจากใบแจ้งหนี้ · สาย QO▸IV▸RE▸TIV):** แก้ได้เหมือนข้างต้น **ยกเว้นแก้ "รายการ" ไม่ได้** (line item อ้างอิงจากเอกสารต้นทาง — ในภาพหน้า RE-ที่มาจาก IV ไม่มีดินสอที่หัวข้อ "รายการ").

**ฟิลด์ที่แก้ไม่ได้:** ข้อมูลลูกค้า · ข้อมูลราคาและภาษี · **จำนวน** · **ราคาสินค้า/บริการ (ราคา/หน่วย)** · ส่วนลด · **ภาษี (VAT)** · หัก ณ ที่จ่าย.

**ต้องทำอย่างไรถ้าต้องแก้ฟิลด์ที่แก้ไม่ได้:** **ยกเลิกเอกสารแล้วสร้างใหม่** โดยใช้เลขเอกสารเดิม (ดูบทความ 3).

**ข้อควรรู้:** ตอนอยู่สถานะ **ร่าง** แก้ได้ทุกฟิลด์ · เมื่ออนุมัติแล้วจึงจำกัดเฉพาะที่มีดินสอ ✏️ (ปุ่ม "แก้ไข" ในเมนู ตัวเลือก จะถูกล็อกเทา).

### บทความ 5 · คัดลอกใบเสร็จรับเงิน ใน PEAK (id 6034544)

**ใช้ทำอะไร:** "คัดลอกใบเสร็จรับเงิน เป็นวิธีที่สะดวกและรวดเร็วในการสร้างเอกสารใหม่จากข้อมูลเดิม" — เมื่อต้องออกใบเสร็จที่ข้อมูลคล้ายเดิมโดยไม่ต้องกรอกใหม่ทั้งหมด.

**ขั้นตอน (จากภาพจริง):**
1. เมนู `รายรับ > ใบเสร็จรับเงิน > ดูทั้งหมด` → เลือก/เปิดใบเสร็จที่ต้องการคัดลอก (ในภาพ = RE-20250800001).
2. ที่หน้ารายละเอียด กดปุ่ม **ตัวเลือก** (ม่วง) → dropdown → เลือก **คัดลอก** (ไอคอนกระดาษซ้อน · อยู่บนสุดของเมนู ใต้ "แก้ไข" ที่ถูกล็อก · เหนือ สร้างใบลดหนี้ / สร้างใบเพิ่มหนี้ / ยกเลิกเอกสาร / ล็อกเอกสาร).

**ข้อมูลที่คัดลอกไป:** ระบบโหลดข้อมูลทั้งหมดของใบเสร็จเข้าสู่ **ฉบับร่างใหม่** ยกเว้น 2 อย่าง — **เลขที่เอกสาร** (auto รันเลขต่อจากใบเสร็จล่าสุด) และ **วันที่ออก** (default = วันที่ทำรายการปัจจุบัน) · ทั้งสองยังแก้ได้.

**สถานะเริ่มต้นของสำเนา:** เป็น **ร่าง** — ตรวจสอบแล้วกดปุ่มอนุมัติเพื่อออกเป็นใบเสร็จจริง.

**ข้อควรรู้:** สำเนาที่ได้เป็น **เอกสารประเภทใบเสร็จรับเงินเสมอ** (ไม่มีตัวเลือกเปลี่ยนประเภทเอกสารตอนคัดลอก). ถ้าต้นฉบับเป็น RE ล้วน (ในภาพ ประเภทใบกำกับภาษี = "ยื่นยอดขาย ภ.พ.30 (ไม่ใช่ใบกำกับภาษี)") สำเนาจะเป็นแบบเดียวกัน — ถ้าต้องการออกใบกำกับภาษีต่อยอด ใช้ปุ่ม **＋** ที่ป้าย "ใบกำกับภาษี (TIV)" บนสายเอกสาร.

## ✳️ เก็บเพิ่ม (main-loop · หลังชนลิมิต · core 4 หมวดที่เหลือ)

### 2. ใบรับเงินมัดจำ (Deposit · id 5781997)
- **คืออะไร:** รับชำระค่าสินค้า/บริการ **ล่วงหน้า** (มัดจำ) เป็นหลักประกันก่อนส่งของ/ออกใบแจ้งหนี้เต็ม.
- **flow:** ใบเสนอราคา (ใส่เงื่อนไขมัดจำใน หมายเหตุ) → ตอบรับ → **เก็บมัดจำ** (ออกใบแจ้งหนี้ขอมัดจำ **หรือ** ออกใบเสร็จเลยถ้าจ่ายทันที) → ส่งของจริงแล้ว **เก็บส่วนที่เหลือ** (ใบแจ้งหนี้/ใบเสร็จ).
- **🔑 การลงบัญชี:** บรรทัดมัดจำผูกบัญชี **212104 "เงินรับล่วงหน้า – เงินมัดจำ"** (หนี้สิน) · **VAT 7%** (ตามคำสั่งกรมสรรพากร ป.73/2541) · ตอนออกใบแจ้งหนี้/ใบเสร็จเต็มใบ → **ตัดมัดจำเป็นค่าลบ (debit)** กลับ = matching กับยอดจริง.
- **สถานะ:** ยอมรับแล้ว → ออกบางส่วน (มัดจำ+ส่วนที่เหลือแยกใบ) → ปิดด้วยใบเสร็จสุดท้าย. (มี ยกเลิก/แก้/mobile — เก็บรอบหน้า)
- **💡 ระบบเรา:** ต้องมีบัญชีหนี้สิน "เงินรับล่วงหน้า" + VAT ตอนรับมัดจำ + กลไก**ตัดมัดจำ**ตอนออกบิลเต็ม (เราเก็บมัดจำ/ฝากโอนอยู่แล้ว — ต้อง map เป็น deposit-liability + ตัดทีหลัง).

### 6. ใบลดหนี้ (Credit Note · id 6320743 อ้างอิงเอกสาร)
- **คืออะไร:** ลดยอดหนี้ที่ออกเอกสารไปแล้ว (ของคืน/ของเสีย/คิดเกิน) — อ้างอิงจาก ใบแจ้งหนี้/ใบเสร็จ/ใบรับมัดจำ เดิม.
- **ขั้นตอน:** รายรับ → ใบลดหนี้ → สร้าง **หรือ** จากใบแจ้งหนี้/ใบเสร็จเดิม กด ตัวเลือก → สร้างใบลดหนี้ → เลือกเอกสารเดิม → ใส่ **สาเหตุ** (คำอธิบาย auto ตามสาเหตุ) · **ขอบเขต** (ทั้งใบ/เฉพาะบางบรรทัด) · **รับคืนสินค้า** (ใช่=สต๊อกเพิ่ม / ไม่=ไม่ปรับสต๊อก).
- **🔑 2 โหมดตัดยอด:** (a) ต้นฉบับ**ยังไม่จ่าย** → offset ตัดลูกหนี้อัตโนมัติ · (b) ต้นฉบับ**จ่ายแล้ว** → เลือกช่องทาง**คืนเงิน**ให้ลูกค้า (เงินสด/ธนาคารออก).
- **การลงบัญชี:** กลับรายการ Dr.รายได้ + Dr.ภาษีขาย / Cr.ลูกหนี้ (หรือ Cr.เงินสด ถ้าคืนเงิน).
- **ข้อจำกัด:** ออกใบลดหนี้จากใบแจ้งหนี้ที่**ผูกมัดจำ**ไม่ได้ · (แบบไม่อ้างอิง 6320790 · คืนเงิน 6360586 · ยกเลิก 9345091 — เก็บรอบหน้า)
- **💡 ระบบเรา:** เรา**ไม่มี**เอกสารลดหนี้ — ตอนนี้แก้บิล/refund ตรงๆ. ระบบบัญชีจริงต้องออก "ใบลดหนี้" อ้างอิงบิลเดิม + กลับรายการบัญชี (คืนภาษีขายด้วย).

### 7. ใบเพิ่มหนี้ (Debit Note · id 6324516 จากเอกสารเดิม)
- **คืออะไร:** เพิ่มยอดหนี้จากใบแจ้งหนี้เดิม (เช่นส่งของขาด/ตกลง 15 แต่ออกบิล 10 → เพิ่ม 5).
- **ขั้นตอน:** รายรับ → ใบเพิ่มหนี้ → สร้าง → **อ้างอิงจากเอกสารเดิม** (กรอง วันที่ออก/ประเภท/ชื่อลูกค้า → เลือกเลขเอกสาร) → ใส่ **สาเหตุ** + ปรับสต๊อก(เพิ่ม/ไม่) + ติ๊กบรรทัดที่คิดเพิ่ม + จำนวน/ราคา → เลือกจ่ายทันที(ช่องทาง)/ยังไม่ชำระ(ตั้งลูกหนี้) → **อนุมัติใบเพิ่มหนี้**.
- **🔑 การลงบัญชี:** จ่ายทันที = Dr.เงินสด/Cr.รายได้ · ยังไม่ชำระ = **Dr.ลูกหนี้/Cr.รายได้** (+ บัญชี 212103 กรณีวันจ่าย≠วันออก). ออกใบกำกับภาษีพ่วงได้.
- **💡 ระบบเรา:** เหมือนใบลดหนี้ — เราไม่มี ต้องมีตอนทำบัญชีจริง (คิดเพิ่ม+ภาษีขายเพิ่ม อ้างอิงบิลเดิม).

### 8. ใบวางบิล (Billing Note · id 5781303 สร้าง) — 🎯 = billing-run ของเรา
- **คืออะไร:** **รวมหลายเอกสารที่ยังไม่จ่าย** (ใบแจ้งหนี้ + ใบลดหนี้ + ใบเพิ่มหนี้ + ใบรับมัดจำ) เป็น**ใบเดียว**เพื่อวางบิล+รับชำระ+ออกใบเสร็จรวม (แทนเก็บทีละใบ · = billing รายเดือน).
- **ขั้นตอน:** รายรับ → ใบวางบิล → สร้าง → **กรองเลือกเอกสาร** (ช่วงวันที่ · ประเภท · **รหัสผู้ติดต่อ**) → ระบบดึงเอกสารค้างจ่ายที่ match → **ติ๊กเลือกใบ + ระบุยอดวางบิล** (แก้ยอดต่อใบได้ = วางบิลบางส่วน) → อนุมัติ → ไป **ข้อมูลการชำระเงิน** → กด "รับชำระเงิน" (วันที่+ช่องทาง) → กด "ออกใบเสร็จ" ต่อได้เลย.
- **🔑 เงื่อนไขรวม:** เอกสารต้อง **บริษัท/ผู้ติดต่อเดียวกัน** + **สกุลเงินเดียวกัน** เท่านั้น.
- **การลงบัญชี:** ตัวใบวางบิลเองเป็นเอกสารรวม/แจ้งยอด (ลูกหนี้ตั้งตั้งแต่ใบแจ้งหนี้แล้ว) · จุดลงเงินจริง = ใบเสร็จที่ออกต่อ.
- **💡 ระบบเรา:** **ตรงกับ billing-run เป๊ะ** — เรารวมหลายออเดอร์/แทรคของลูกค้าเดียวเป็นใบวางบิล FRI แล้วรับชำระ→ออกใบเสร็จ FRG. สิ่งที่ PEAK มีเพิ่ม = วางบิล**บางส่วนต่อใบ** + รวม**ใบลดหนี้/เพิ่มหนี้/มัดจำ**เข้าไปในใบวางบิลได้ด้วย. (แบบแบ่งชำระ 6264575 · แก้ก่อน/หลังรับเงิน 6021413/6054560 · พิมพ์ 6043628 · ยกเลิก 6043684 — เก็บรอบหน้า)
