# PEAK — เอกสารขาย-รับเงิน (Revenue Documents) — คู่มือความรู้ครบวงจร

รวมความรู้ฝั่ง **รายรับ (เอกสารขาย-รับเงิน)** ของ PEAK ทั้ง 8 ชนิดเอกสาร (68 บทความ · จับภาพจริง 17 ชุด) — ครอบคลุมตั้งแต่การสร้าง แก้ไข ยกเลิก ย้อนสถานะ การรับชำระแบบพิเศษ นำเข้าไฟล์ ไปจนถึงคู่บัญชี (Dr/Cr) ที่ระบบลงให้อัตโนมัติ.
เอกสารนี้ใช้เป็นพิมพ์เขียวสำหรับสร้างระบบเอกสารขาย-รับเงินของ Pacred เอง — เก็บทั้ง doc-code, running-series, state-machine, convert-chain, การลงบัญชีต่อ line, VAT/WHT และกฎการแก้/ยกเลิกตามสถานะ.

## 🔄 สายเอกสาร (workflow chain)

**รหัสเอกสาร (doc codes) ที่เห็นจริงในระบบ:**

| เอกสาร | รหัส/prefix | รูปแบบเลขที่ | ลงบัญชีไหม |
|---|---|---|---|
| ใบเสนอราคา | **QO** | `QO-YYYYMM#####` | ❌ ไม่ลง |
| ใบรับเงินมัดจำ (ไม่ออกใบกำกับ) | **DR** | `DR-YYYYMM#####` | ✅ (พัก 212104) |
| ใบรับเงินมัดจำ/ใบกำกับภาษี | **DRT** | `DRT-YYYYMM#####` | ✅ (พัก 212104 + VAT) |
| ใบแจ้งหนี้ (ล้วน) | **IV** | `IV-YYYYMM#####` | ✅ (SV) |
| ใบแจ้งหนี้/ใบกำกับภาษี | **IVT** | `IVT-YYYYMM#####` | ✅ (SV) |
| ใบเสร็จรับเงิน (ล้วน) | **RE** | `RE-YYYYMM#####` | ✅ (RV) |
| ใบเสร็จรับเงิน/ใบกำกับภาษี | **RT** | `RT-YYYYMM#####` | ✅ (RV) |
| ใบกำกับภาษี (แยก/เดี่ยว · ม.86/4) | **TIV** | `TIV-YYYYMM#####` | ✅ (JVTX ย้าย VAT) |
| ใบลดหนี้ (Credit Note) | **CN / CNT** | `CNT-YYYYMM#####` | ✅ (กลับด้าน) |
| ใบเพิ่มหนี้ (Debit Note) | **DBN / DBNT** | `DBN(T)-YYYYMM#####` | ✅ (SV+RV) |
| ใบวางบิล (Billing Note) | **BN** | `BN-YYYYMM#####` | ❌ ตอนออก (เรียกเก็บ) · ✅ ตอนรับชำระ |
| ใบจ่ายเงินมัดจำ (ฝั่งรายจ่าย) | **DP** | `DP-YYYYMM#####` | — |
| batch นำเข้าไฟล์ | **IM+doc** | `IMRT/IMQO/IMIV-YYMM###` | — |
| สมุดรายวันขาย | **SV** | `SV-YYYYMM###` | Dr ลูกหนี้/Cr รายได้+ภาษีขาย |
| สมุดรายวันรับ | **RV** | `RV-YYYYMM###` | Dr เงิน/Cr ลูกหนี้ |
| สมุดรายวันทั่วไป | **JV / JVTX** | `JV-YYYYMM###` | ปรับปรุง/ย้าย VAT |

**สายต่อยอด (stepper บนหัวเอกสาร):**

```
                         ┌──────────────► ใบลดหนี้ (CN)   ← ลดยอดหลังออก
                         │                ใบเพิ่มหนี้ (DBN) ← เพิ่มยอดหลังออก
  ใบเสนอราคา (QO) ──┬──► ใบแจ้งหนี้ (IV/IVT) ──► ใบเสร็จรับเงิน (RE/RT) ──► ใบกำกับภาษี (TIV)
   (pre-sale·ไม่ลงบช.) │        (ตั้งลูกหนี้·SV)          (รับเงิน·RV)         (แยก·ย้าย VAT)
                         └──► ใบเสร็จรับเงิน (RE)   ← ขายสด ข้าม IV
                                    ▲
  ใบรับเงินมัดจำ (DR/DRT) ──[+ เลือกเงินมัดจำ]──► หักเข้า IV/RE (ตัดได้ ≤10 ใบ)

  หลาย IV (ยังไม่ชำระ) ──► ใบวางบิล (BN) ──► ใบเสร็จรับเงิน (RE) ──► ใบกำกับภาษี (TIV)
     (ลูกค้า+สกุลเงินเดียวกัน · รวมเรียกเก็บ+ออกใบเสร็จ 1 ใบ)
```

- **จุดที่ลงบัญชี:** IV/RE/DR/TIV/CN/DBN ลงบัญชีเมื่อ **อนุมัติ/รับชำระ** · **QO และ BN ไม่ลงบัญชีตอนออก** (QO = ก่อนขาย, BN = เอกสารเรียกเก็บ — หนี้ยังอยู่ที่ IV ต้นทาง; BN ลงบัญชีตอน \"รับชำระ\" เป็น RV).
- **มัดจำ:** พักที่ `212104 เงินรับล่วงหน้า-เงินมัดจำ` แล้วหักล้างตอนออกบิลจริง (ผ่าน \"เลือกเงินมัดจำ\" หรือใส่ยอดติดลบ).
- **ลดหนี้/เพิ่มหนี้:** ใช้แก้ยอด **หลังออกเอกสารไปแล้ว** (ลด/คืน = CN · เก็บเพิ่ม = DBN).

## 1. ใบเสนอราคา (Quotation · QO)

> **สรุปชนิดเอกสาร.** ใบเสนอราคา = เอกสารเสนอราคา/เงื่อนไขให้ลูกค้าพิจารณา **ก่อนปิดการขาย** — เป็นต้นสายเอกสารขาย แต่ **ไม่ลงบัญชี** (ไม่มี Dr/Cr, ไม่กระทบงบ/สต็อก). รหัส **QO** · เลข `QO-YYYYMM#####`.
> **สถานะ (state-machine):** ร่าง → รออนุมัติ → **รอตอบรับ** → **ยอมรับแล้ว** / **ปฏิเสธแล้ว** → ออกบางส่วน → ออกครบแล้ว → ยกเลิก.
> **แก้ไข:** ได้เฉพาะ \"รอตอบรับ\" (และร่าง) — ถ้า \"ยอมรับแล้ว\" ต้อง **ย้อนกลับเป็นรอตอบรับ** ก่อน (ตัวเลือก → กลับไปรอตอบรับ).
> **การลงบัญชี:** — (ไม่มี).

### วิธีสร้างใบเสนอราคาใน PEAK (article 5966405)

**ใช้ทำอะไร:** ออกใบเสนอราคาเสนอลูกค้า (ราคา + เงื่อนไข) ก่อนปิดการขาย. ไม่ลงบัญชี — เป็นเอกสารตั้งต้นของสายเอกสารรายรับ.

**ขั้นตอน (จากภาพจริง):**
1. **เข้าหน้าสร้าง** — เมนู `รายรับ` → hover `ใบเสนอราคา` → submenu: `+ สร้าง` / ยอมรับ / รอตอบรับ / พ้นกำหนด / ดูทั้งหมด / ล่าสุด → กด `+ สร้าง`.
2. **หน้า "สร้างใบเสนอราคา"** เป็น section เรียงลงมา (มี stepper QO▸IV▸RE▸TIV · มุมขวา `⭐ เพิ่มรายการโปรด` + `⚙️ ตั้งค่าการแสดงผล`):
   - **① เลขที่เอกสาร (ขวาบน):** `อ้างอิง` (≤32 อักขระ) + `เลขที่เอกสาร *` (auto `QO-20251200001` · แก้ได้).
   - **② ข้อมูลลูกค้า:** `ชื่อลูกค้า *` (ค้นหา/สร้างผู้ติดต่อ · ดึงที่อยู่/เบอร์อัตโนมัติ) · `วันที่ออก *` (default วันนี้) · `วันที่ใช้ได้ถึง *` (default +7 วัน).
   - **③ กลุ่มจัดประเภท** (Pro Plus): พนักงานขาย / ช่องทางการขาย / แผนก / โครงการ (default "ยังไม่ระบุ").
   - **④ ข้อมูลราคาและภาษี:** `ประเภทราคา *` = **แยกภาษี** (100+7=107) หรือ **รวมภาษี** · `ราคานี้เป็นค่าเงิน` (THB).
   - **⑤ รายการ:** ตาราง `สินค้า/บริการ * · บัญชี · จำนวน * · ราคา/หน่วย * · ส่วนลด/หน่วย · ภาษี · มูลค่าก่อนภาษี · หัก ณ ที่จ่าย` · ปุ่ม `+ เพิ่มรายการใหม่` · `+ เพิ่มคำอธิบายรายการ`.
   - **⑥ สรุปข้อมูล:** ส่วนลดรวม · มูลค่าที่คำนวณภาษี 7% · ภาษีมูลค่าเพิ่ม 7% (✏️แก้ได้) · กล่อง **จำนวนเงินทั้งสิ้น**.
   - **⑦ หมายเหตุสำหรับลูกค้า · ⑧ แนบไฟล์ · ⑨ แท็ก**.
3. **บันทึก/อนุมัติ (Sticky Bar):**
   - `ยกเลิก` · **`บันทึกร่าง ▲`** (→ สถานะ **ร่าง**) · **`อนุมัติใบเสนอราคา ▲`** (dropdown: อนุมัติและพิมพ์ / อนุมัติและสร้างใหม่ / ส่งอนุมัติ → สถานะ **รอตอบรับ**).
4. **พิมพ์/ส่ง** — `ส่งเอกสาร ▾` (อีเมล) · `พิมพ์ ▾` (ใบส่งของชั่วคราว/ซองเอกสาร · แก้ธีม/หัว/คอลัมน์/ฟอนต์) · `ตัวเลือก ▾`.

**Gotchas:** ① `ส่วนลดรวม` กรอกไม่ได้ = แถวรายการภาษีไม่เหมือนกันทุกแถว. ② ไม่เห็นช่องเงินมัดจำ/หัก ณ ที่จ่าย → เปิด toggle ที่ `⚙️ ตั้งค่าการแสดงผล`.

### แก้ไขเอกสารใบเสนอราคา (สถานะรอตอบรับ) (article 6014404)

**แก้ได้เฉพาะสถานะ "รอตอบรับ"** — ถ้า "ยอมรับแล้ว" ต้องย้อนกลับก่อน (9798379).
1. `รายรับ → ใบเสนอราคา → รอตอบรับ` → คลิกเลขที่เอกสาร (เช่น QO-20260100001).
2. `ตัวเลือก ▾` → `แก้ไข` → แก้ → `บันทึก`.
**แก้ไม่ได้:** ยอมรับแล้ว / ปฏิเสธแล้ว / ยกเลิก / เมื่อมีเอกสารต่อยอดแล้ว.

### เปลี่ยนสถานะ "ยอมรับแล้ว" → "รอตอบรับ" (article 9798379)

1. `รายรับ → ใบเสนอราคา → ดูทั้งหมด` → เลือกใบ "ยอมรับแล้ว".
2. `ตัวเลือก ▾` → `กลับไปรอตอบรับ`.
3. Dialog "ยืนยัน ใบเสนอราคา กลับไปสถานะรอตอบรับ" → `ยืนยัน`.

### การกดปฏิเสธใบเสนอราคา (article 10319166)

ลูกค้าปฏิเสธ → สถานะ "รอตอบรับ" → "ปฏิเสธแล้ว" (ใช้วิเคราะห์ Conversion Rate).
1. เปิดเอกสาร (รอตอบรับ) → เข้าหน้าตอบรับ 3 ทาง: (3.1) stepper การ์ด `ใบเสนอราคา QO ▲` → `>` → `💬 ตอบรับ` · (3.2) `ตัวเลือก ▾` → `ตอบรับ` · (3.3) ปุ่ม `ตอบรับ`.
2. Dialog "ตอบกลับใบเสนอราคาแล้ว": เลือก **● ปฏิเสธ** → กรอก `วันที่ปฏิเสธ` → `ยืนยัน`.
> เลือก `● ยอมรับ` = กดยอมรับ (→ ยอมรับแล้ว, กรอกวันที่ยอมรับ).

### การยกเลิกใบเสนอราคา (article 4552774)

**กรณี 1 — ยังไม่มีเอกสารอ้างอิงต่อยอด:** เข้าใบ → `ตัวเลือก ▾` → `ยกเลิกเอกสาร` → Dialog ยืนยัน → `ยืนยัน` → สถานะ **ยกเลิก** (dot เทา). (ถ้า "ยอมรับแล้ว" ต้องย้อนเป็นรอตอบรับก่อน.)
**กรณี 2 — มีเอกสารอ้างอิงต่อยอดแล้ว:** ยกเลิกย้อนจากปลายน้ำ → RE (ตัวเลือก→ยกเลิก) → IV (ยกเลิกการชำระ→ยกเลิกเอกสาร) → QO (ตามกรณี 1).
**Gotcha:** เอกสารในแท็บ "ยกเลิก" **ลบไม่ได้** (เก็บอ้างอิง) · ยกเลิก/ลบแล้วสร้างใหม่ ระบบรันเลขต่อจากล่าสุด (แก้เลขเองได้).

### คัดลอกใบเสนอราคา เพื่อออกใบใหม่ (article 4548443)

`รายรับ → ใบเสนอราคา → ดูทั้งหมด` → เปิดเอกสาร → `☰ ตัวเลือก` → `📄 คัดลอก` → ได้ใบใหม่ (ร่าง/รอตอบรับ) พร้อมแก้ต่อ. **คัดลอกได้ทุกสถานะ รวมถึงใบที่ยกเลิกแล้ว** (ใช้แทนการแก้ใบยกเลิก).

### เพิ่มรายการโปรด ใบเสนอราคา (article 6014184)

ในหน้าสร้าง → กรอกข้อมูลก่อน → `⭐ เพิ่มรายการโปรด` → dialog: **ชื่อรายการโปรด*** (≤92) · คำอธิบาย (≤500) · บันทึกผู้ติดต่อไว้ (บันทึก/ไม่) · บันทึกจำนวน/ราคา/ส่วนลด (บันทึก/ไม่) → `เพิ่มรายการ`. การ์ดโผล่ใต้ "รายการโปรดของใบเสนอราคา" · คลิกดึงมาใช้ซ้ำ. (ไม่มีสถานะ/ไม่ลงบัญชี).

### ตรวจสอบสถานะใบเสนอราคา (article 7200095)

`รายรับ → ใบเสนอราคา → ดูทั้งหมด` → tab สถานะ (badge จำนวน): ล่าสุด/ทั้งหมด/ร่าง/รออนุมัติ/รอตอบรับ/พ้นกำหนด/ยอมรับแล้ว/ออกบางส่วน/ออกครบแล้ว/ปฏิเสธแล้ว.
**ปุ่ม Sticky Bar เปลี่ยนตามสถานะ:** รอตอบรับ → `💬 ตอบรับ` · ยอมรับแล้ว → `สร้างใบแจ้งหนี้`.
ตรวจว่าออกต่อแล้วยัง: ดู stepper (dot สี/✓) หรือ tab "ข้อมูลประวัติ".

### เปลี่ยนสถานะหลายรายการ → "ออกครบแล้ว" (bulk · article 11461514)

ใช้กับ **QO** (ยอมรับแล้ว/ออกบางส่วน) และ **PO ใบสั่งซื้อ** (อนุมัติแล้ว/ออกบางส่วน).
`ดูทั้งหมด` → tab → **ติ๊ก checkbox** (≤100 รายการ) → Sticky Bar ล่าง (`เลือก N รายการ · รวมมูลค่าสุทธิ`) → `เปลี่ยนเป็นออกครบแล้ว` (เขียว) → ยืนยัน.
**เงื่อนไข:** แต่ละใบต้องมี IV/RE ออกต่อแล้ว (QO) หรือบันทึกซื้อแล้ว (PO) · ไม่ตรงเงื่อนไข → Pop-up เตือน (เปลี่ยนเฉพาะที่ตรง).

### นำเข้าใบเสนอราคา (Excel · article 6038985 · Pro Plus+)

**คอลัมน์ Excel (A→Q):** ลำดับที่* · วันที่ · อ้างอิงถึง · ลูกค้า · เลขทะเบียน 13 หลัก · เลขสาขา 5 หลัก · ประเภทราคา · สินค้า/บริการ · บัญชี · คำอธิบาย · จำนวน* · ราคาต่อหน่วย* · ส่วนลดต่อหน่วย · อัตราภาษี · ถูกหัก ณ ที่จ่าย · หมายเหตุ · กลุ่มจัดประเภท.
**Gotcha:** ลำดับที่ซ้ำ = รวมเป็นเอกสารเดียว · เตือนเลขซ้ำ · ทศนิยม ≤4 ตำแหน่ง · ตรวจ/ลบก่อนสร้างจริงได้.

## 2. ใบรับเงินมัดจำ (Deposit · DR/DRT)

> **สรุปชนิดเอกสาร.** เงินมัดจำ = รับชำระล่วงหน้าบางส่วน ทำได้ **2 แนวทาง** — (1) ใช้ document-type **ใบรับเงินมัดจำ (DR/DRT)** + ระบบ "ตัดเงินมัดจำ" อัตโนมัติผ่าน dialog "เลือกเงินมัดจำ" (ตัดได้ ≤10 ใบ) · (2) ทำเป็น line item บนบิลปกติ โดยเลือกผังบัญชี **212104 เงินรับล่วงหน้า-เงินมัดจำ** แล้วล้างด้วยยอดติดลบ.
> **รหัส:** DR (ไม่ออกใบกำกับ) / DRT (ใบกำกับภาษี) · **สถานะ:** รอรับชำระ → รอหักมัดจำ (รับเงินแล้ว) → ออกบางส่วน/หักมัดจำแล้ว.
> **การลงบัญชี:** รับมัดจำ → Dr ธนาคาร / Cr **212104** + Cr 215101 ภาษีขาย.

### บันทึกรับเงินมัดจำ (line-item · article 5781997)

ผูกจาก QO → ให้ลูกค้าตอบรับ (stepper QO⋯IV⋯RE⋯TIV · `💬 ตอบรับ`) → เรียกเก็บมัดจำ 2 ทาง: **(2a ยังไม่จ่าย → IV)** ลบรายการสินค้าเดิม → `+ เพิ่มรายการใหม่`: สินค้า=ว่าง, **บัญชี 212104**, ราคา=ยอดมัดจำ, **ภาษี 7%** → อนุมัติ → รับชำระ → ออกใบเสร็จ · **(2b จ่ายทันที → RE)** สร้างใบเสร็จ ใส่รายการมัดจำแบบเดียวกัน → อนุมัติ.
**ล้างมัดจำส่วนที่เหลือ:** จาก QO (ออกบางส่วน) สร้างบิลใหม่ ใส่สินค้าจริง + **เพิ่มรายการล้างมัดจำ** (บัญชี 212104, **ราคา/หน่วย = ยอดติดลบ**, เปิด `ออกใบกำกับภาษี`) → ยอดสุทธิ = ราคาเต็ม − มัดจำ.
**Dr/Cr:** รับมัดจำเป็นเงิน → Dr ธนาคาร / Cr 212104 + Cr 215101 · บิลตั้งหนี้ → Dr ลูกหนี้ / Cr 212104 + Cr ภาษีขาย · ล้างมัดจำ → บรรทัดติดลบ 212104 (Dr 212104).
**Gotcha:** บรรทัดมัดจำ **ไม่ระบุสินค้า** (ไม่ตัดสต็อก) · ต้องลบสินค้าให้เหลือแต่บรรทัดมัดจำ · ส่วนที่เหลือใช้ยอดติดลบ + เปิดใบกำกับ.

### ใบรับเงินมัดจำ (เอกสารเฉพาะ DR/DRT · article 9439114)

3 ขั้น: (ข้ามได้) สร้าง QO → **สร้างใบรับเงินมัดจำ** (`รายรับ → ใบรับเงินมัดจำ → + สร้าง`) → **สร้าง IV/RE** สำหรับยอดเหลือ แล้วกด **`+ เลือกเงินมัดจำ`** ดึงมาหัก. หน้า detail มี stepper `ใบรับเงินมัดจำ (DR) ⋯ ใบกำกับภาษี (TIV)` + แท็บพิเศษ **ข้อมูลการหักเงินมัดจำ**.
**Dr/Cr:** ตั้งหนี้ → Dr ลูกหนี้ / Cr 212104 · รับเงิน → Dr ธนาคาร / Cr 212104 · หักเข้าบิล → Dr 212103/212104 / Cr เงินมัดจำล่วงหน้า.
**Gotcha:** วันที่ใบมัดจำ ≤ วันที่ IV จึงหักได้ · ไม่ตัดสต็อก/ไม่รองรับสกุลต่างประเทศ · เอกสารถัดไป "ออกครบแล้ว" = เพิ่มไม่ได้ · **ประเภทภาษีต้องตรงกัน** (14770836).

### บันทึกรับและคืนเงินมัดจำ (article 6306182)

ตัวอย่างทัวร์ 15,000: มัดจำ 5,000 ก่อน → จ่ายเต็ม → คืนมัดจำ.
1. **รับมัดจำ:** `ใบเสร็จรับเงิน → สร้าง` บัญชี **212104**, 5,000, ภาษี 7%, เปิดออกใบกำกับ → อนุมัติ (RT).
2. **รับส่วนเหลือ:** ใบเสร็จใหม่ บัญชีบริการ 410201 ค่าทัวร์ 15,000 + `+ เลือกเงินมัดจำ` หัก → อนุมัติ.
3. **คืนมัดจำ:** เข้าใบเสร็จมัดจำ (RT) → `☰ ตัวเลือก` → **สร้างใบลดหนี้**.
**Dr/Cr (RV รับค่าบริการ):** Dr 111301 ธนาคาร 16,050 / Cr 410201 15,000 / Cr 215101 1,050 · รับมัดจำ: Dr ธนาคาร 5,350 / Cr 212104 5,000 + Cr ภาษีขาย 350 · คืน (ใบลดหนี้): Cr ธนาคาร / Dr 212104 + Dr ภาษีขาย.

### ยกเลิกใบรับเงินมัดจำ (article 12025068)

**ถอยจากปลายเข้าต้น:** ยกเลิก ใบเสร็จ → การรับชำระ → การเชื่อม/หักมัดจำ → เอกสารมัดจำ.
- **กรณี 1 (รอรับชำระ):** `☰ ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน`.
- **กรณี 2 (รอหักมัดจำ):** แท็บ `ข้อมูลการชำระ` → การ์ดชำระ `⋮ → ยกเลิก` → กลับ `ข้อมูลเอกสาร → ตัวเลือก → ยกเลิกเอกสาร`.
- **กรณี 3 (เชื่อมเอกสารแล้ว):** 3a เชื่อม IV → แท็บ `ข้อมูลการหักเงินมัดจำ` → คลิก IV → แก้ไข → **เอาเครื่องหมายถูกหน้าใบมัดจำออก** (ถ้า IV มีใบเสร็จ ต้องยกเลิกใบเสร็จ+การชำระก่อน) → กลับมายกเลิกที่ใบมัดจำ · 3b เชื่อม RE → คลิก RE → ตัวเลือก → ยกเลิกเอกสาร (ใบเสร็จยกเลิกได้อย่างเดียว) → กลับมายกเลิกมัดจำ.
**Gotcha:** เอกสารในแท็บยกเลิก **ลบไม่ได้** · ต้องปลดหักมัดจำ+ยกเลิกการรับชำระให้หมดก่อน.

### แก้เมื่อกดตัดเงินมัดจำแล้วเตือน "ยอดเงินมัดจำไม่มี/ยกเว้นภาษี" (article 14770836)

**ต้นเหตุ:** **ยอดภาษีของ 2 เอกสารไม่ตรง** — ใบมัดจำ "ไม่มี/ยกเว้นภาษี" ตัดกับบิลมี VAT 7% ไม่ได้ (dialog โชว์แถบแดง เลือกไม่ได้).
**แก้:** (1) ยกเลิกใบมัดจำเดิม (รอหักมัดจำ = ยกเลิกการรับชำระก่อน · รอรับชำระ = ยกเลิกได้เลย) → (2) สร้างใหม่ ไม่ระบุรหัสสินค้า เลือก **212104** + ใส่ภาษีให้ตรงบิล → (3) ตัดกับ IV: **วิธี 1** `+ เพิ่มรายการใหม่` 212104 **ยอดติดลบ** ไม่ระบุภาษี · **วิธี 2** รับชำระ → `ขั้นสูง` → "ค่าธรรมเนียม/รายการปรับปรุง".
**Gotcha:** 2 วิธีต่างกันแค่ผลบนหน้าพิมพ์.

### PEAK Mobile — เอกสารเงินมัดจำ (article 15652563)

e-Document+ · แอป ≥0.2.190. รับ (DR) + จ่าย (DP). **สร้าง** จากแท็บล่าง รายรับ/รายจ่าย → `+ สร้างใบรับ/จ่ายเงินมัดจำ` → กรอก → สร้าง → **อนุมัติ** → "รอหักเงินมัดจำ". **ตัดยอด** ในหน้าสร้าง IV → section `💰 เงินมัดจำ` → `+ เลือกเงินมัดจำ` → หน้า "เลือกเงินมัดจำ" (checkbox หลายใบ · มูลค่าคงเหลือ/หักมัดจำ · VAT 7% ✏️) → ยืนยัน → บิลติดลบ = ยอด−มัดจำ. ภาษีไม่ตรง = แถบแดง (แก้ยอดภาษีเป็น 0). ประวัติ: แท็บ "ประวัติการตัดยอดมัดจำ".

## 3. ใบแจ้งหนี้ (Invoice · IV/IVT)

> **สรุปชนิดเอกสาร.** ใบแจ้งหนี้ = ใบส่งของ + บันทึกลูกหนี้ในเล่มเดียว — หัวใจของ flow ขายเชื่อ (ตั้งลูกหนี้). รหัส **IV** (ไม่ออกใบกำกับ) / **IVT** (ออกใบกำกับ · toggle "ออกใบกำกับภาษี").
> **สถานะ:** ร่าง → รออนุมัติ → **รอรับชำระ** → เกินเวลารับชำระ → **รับชำระแล้ว** → ยกเลิก. + tab "เอกสารอัตโนมัติ" (recurring).
> **แก้ไข/ยกเลิก:** ผูกอะไรแล้วล็อก — ต้อง **ถอยปลายน้ำก่อน** (ใบเสร็จ → การรับชำระ → เอกสารอ้างอิง → ใบแจ้งหนี้).
> **การลงบัญชี:** อนุมัติ → **SV** (Dr 113101 ลูกหนี้ / Cr 410xxx รายได้ + Cr 215101/215102 ภาษีขาย [+ Dr 510103 ต้นทุน / Cr 114102 สินค้า]) · รับชำระ → **RV**.

### สร้างใบแจ้งหนี้ (IV001 / IV011 / article 5966… ชุด)

**เข้า:** `รายรับ → ใบแจ้งหนี้ (ใบส่งของ, บันทึกลูกหนี้) → + สร้าง` (หรือปุ่ม `+ สร้างใบแจ้งหนี้` บนหน้ารายรับ = IV011 · ฟอร์มเดียวกัน).
**ฟอร์ม "สร้างใบแจ้งหนี้/ใบกำกับภาษี":** ① อ้างอิง (≤32) + เลขที่ (`IVT-20250800003` แก้ผ่าน ⚙) · ② ข้อมูลลูกค้า (ชื่อ*/วันที่ออก*/**วันที่ครบกำหนด*** [เครดิตเทอม]/ที่อยู่/เบอร์) · ③ กลุ่มจัดประเภท · ④ ราคาและภาษี (ประเภทราคา* · **toggle "ออกใบกำกับภาษี"** · THB) · ⑤ รายการ (สินค้า/บริการ·บัญชี·จำนวน·ราคา/หน่วย·ส่วนลด·ภาษี·มูลค่าก่อนภาษี·หัก ณ ที่จ่าย) · ⑥ **เงินมัดจำ** (`+ เลือกเงินมัดจำ`) · ⑦ สรุปข้อมูล · ⑧ หมายเหตุ/แนบไฟล์/แท็ก.
**ปุ่ม:** ยกเลิก · `บันทึกร่าง ▲` (ร่าง) · `อนุมัติใบแจ้งหนี้ ▲` (อนุมัติและพิมพ์/สร้างใหม่/ส่งอนุมัติ).
**หลังอนุมัติ:** "#IVT-…" สถานะ **รอรับชำระ** · แท็บ ข้อมูลเอกสาร/การชำระ/ประวัติ · ปุ่ม ส่งเอกสาร/พิมพ์ (ใบแจ้งหนี้/ใบส่งของ/ใบวางบิล/ใบเรียกเก็บ/ซอง)/ตัวเลือก · ลิงก์ `บันทึกบัญชี : SV-…`.
**Dr/Cr (SV):** บริการ (ไม่ออกใบกำกับ) → Dr 113101 3,210 / Cr 410201 3,000 / Cr **215102 ภาษีขายรอเรียกเก็บ** 210 · สินค้า (ออกใบกำกับ) → Dr 113101 4,280 / Cr 410101 4,000 / Cr **215101** 280 + Dr 510103 ต้นทุน / Cr 114102 สินค้า.
**Gotchas:** ส่วนลดรวมโผล่เมื่อทุกแถวภาษีเท่ากัน · วันที่ออกย้อนหลังล่วงหน้าไม่ได้ต้องกรอกเอง · 215102 vs 215101 ตาม toggle.

### สร้างใบแจ้งหนี้ ต่อจากใบเสนอราคา (IV002)

QO ต้อง **ยอมรับแล้ว**. `รายรับ → ใบเสนอราคา → ยอมรับ` → 2 วิธี: `+` ในกล่อง IV บน stepper · หรือปุ่มเขียว `สร้างใบแจ้งหนี้` (Sticky Bar). ถ้ายังไม่มีวันตอบรับ → dialog กรอก `วันที่ยอมรับ` → `ยืนยัน`. → dialog "สร้างเอกสารด้วยวิธี": **ตามใบเสนอราคา** / **แบ่งตามสินค้า/บริการแต่ละแถว** → ยืนยัน → ฟอร์มดึงข้อมูล QO มา → บันทึก/อนุมัติ.

### สร้างใบแจ้งหนี้ หลายใบจาก QO เดียว (IV003)

`ดูทั้งหมด` (tab ออกบางส่วน) → `สร้างใบแจ้งหนี้` → dialog 3 ตัวเลือก: **ตามใบเสนอราคา** / **แบ่งตามสินค้า/บริการ** (checkbox แต่ละแถว + แก้จำนวน/ราคา · ยอดคงเหลือ auto) / **แบ่งเปอร์เซ็นต์** (กรอก % เช่น 30/70 · สำหรับบริการ). ออกจนครบ → QO = "ออกครบแล้ว" (ระหว่างทาง "ออกบางส่วน").

### สร้างใบแจ้งหนี้แบบแบ่งชำระ (installment · RT010)

= รับเงินหลายครั้งในใบเดียว (ต่างจาก IV003 ที่ออกหลายใบ).
เปิด IV (รอรับชำระ) → แท็บ `ข้อมูลการชำระ` → `รับชำระ` → dialog (พื้นฐาน/ขั้นสูง) กรอก `จำนวนเงินที่รับชำระ` **บางส่วน** (เช่น 3,000/10,700) → แถบเขียว + "ต้องรับชำระอีก 7,700" → `รับชำระเงิน`. งวดถัดไปกด `รับชำระ` ซ้ำจนครบ → **รับชำระแล้ว**. แต่ละงวดมีการ์ด + `บันทึกบัญชี RV-…` + ปุ่ม `ออกใบเสร็จ` (ต่องวด) / `ออกใบเสร็จรวม`.
**Dr/Cr (ต่องวด · RV):** Dr เงินฝากธนาคาร / Cr 113101 ลูกหนี้ (ยอดรับจริง).

### สร้างใบแจ้งหนี้ รายเดือน / รายปี (Recurring · Pro Plus+)

`ดูทั้งหมด → + สร้างใบแจ้งหนี้ ▾ → + สร้างใบแจ้งหนี้อัตโนมัติ` (เลขที่รายการ `RCIV-…`). กรอก template + section **เอกสารรายเดือน:** รอบการสร้าง (ทุกวันที่ของเดือน / **ทุกปี** [เพิ่มช่อง "เดือน"]) · วันที่ · เริ่มตั้งแต่ (ย้อนหลังไม่ได้) · สิ้นสุด (จำนวนครั้ง/ไม่ระบุ) · **สร้างเอกสารเป็น** (ร่าง/รออนุมัติ/อนุมัติ → เลือก "อนุมัติ" มี toggle "ส่งอีเมลอัตโนมัติ") → `บันทึก`. ไปแท็บ **เอกสารอัตโนมัติ** (คอลัมน์ รอบการสร้าง/สร้างครั้งต่อไป/สถานะ) · toggle "เปิดใช้งาน" / `ตัวเลือก → ปิดการทำงาน`.
**Gotcha:** RCIV ≠ เลขเอกสารจริง · เลขเอกสารรันตามลำดับตอนออกจริง.

### พิมพ์ใบแจ้งหนี้ ใบส่งของ ใบวางบิล

เปิดเอกสาร → `พิมพ์ ▾` → เลือก: **ใบแจ้งหนี้ / ใบส่งของ-ส่งมอบงาน / ใบวางบิล**. หัวเอกสารกำหนดเอง: `แก้ไข` → ✏ ข้างชื่อหัว → "กำหนดเอง" (เฉพาะใบนี้/ตั้ง default). เอกสารพิมพ์มี QR + ช่องลายเซ็น. toggle "ออกใบกำกับภาษี" คุมว่าคำ "ใบกำกับภาษี" ขึ้นไหม + sync รายงานภาษี.

### เพิ่มรายการโปรด ใบแจ้งหนี้

หน้าสร้าง → `⭐ เพิ่มรายการโปรด` → popup (ชื่อ*≤92 · คำอธิบาย≤500 · บันทึกผู้ติดต่อ · บันทึกจำนวน/ราคา/ส่วนลด) → `เพิ่มรายการ` → การ์ดใต้ "รายการโปรดของใบแจ้งหนี้".

### แก้ไขใบแจ้งหนี้ (article 6018371 / 6043629 / 8570648)

**เคส 1 — รับชำระแล้ว ยังไม่ออกใบเสร็จ (6018371):** `ชำระแล้ว` tab → เปิดใบ (มีปุ่ม `สร้างใบเสร็จ` = ยังไม่ออก) → แท็บ `ข้อมูลการชำระ` (แม่กุญแจส้ม) → การ์ดชำระ `⋮ → ยกเลิก → ยืนยัน` → แท็บ `ข้อมูลเอกสาร → ตัวเลือก → แก้ไข` (สถานะกลับ รอรับชำระ) → แก้ → `บันทึก`. **การแก้ไม่ทำให้สินค้ากลับเข้าคลัง.**
**เคส 2 — ออกใบเสร็จแล้ว (6043629):** **ลำดับห้ามสลับ** → ปุ่ม `ดูใบเสร็จ` → หน้าใบเสร็จ `ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` (แก้ไข/คัดลอกเป็นเทา) → กลับ IV `ข้อมูลการชำระ` `⋮ → ยกเลิก` → `ตัวเลือก → แก้ไข → บันทึก`.
**เคส 3 — ถูกนำไปสร้างใบวางบิล (BN · 8570648):** (ก) ยกเลิก BN ทั้งใบ หรือ (ข) เปิด BN → `ตัวเลือก → แก้ไข` → `ย้อนกลับ` → หน้าเลือกเอกสารอ้างอิง → **uncheck** บรรทัด IV → `ถัดไป → บันทึก` → IV เป็นอิสระ → แก้ตามปกติ.

### ยกเลิกใบแจ้งหนี้ (article 6014203 / 5947365)

**6 กรณี (6014203):** (1) ยังไม่ชำระ → `ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` → **ยกเลิก** (dot เทา · ริบบิ้นเทา) · (2) ชำระแล้วยังไม่ออกใบเสร็จ → ยกเลิกการชำระ (`⋮`) → ยกเลิกเอกสาร · (3) ออกใบเสร็จแล้ว → ยกเลิกใบเสร็จ → ยกเลิกการชำระ → ยกเลิก IV · (4) อยู่ใน BN → ถอด/ยกเลิก BN ก่อน · (5) อยู่ในใบลดหนี้ → ยกเลิกใบลดหนี้ก่อน · (6) อยู่ในใบเพิ่มหนี้ (DBN) → ยกเลิก DBN ก่อน.
**ครบวงจร ใบเสร็จ+จับเงิน (5947365):** สถานการณ์ 1 (ไม่มีอ้างอิงต่อยอด): ยกเลิกใบเสร็จ → ยกเลิกการชำระ (`⋮`) → ยกเลิก IV · สถานการณ์ 2 (มีอ้างอิง): ยกเลิกใบเสร็จ → **ลบ/ยกเลิกเอกสารอ้างอิงทั้งหมด** → ยกเลิก IV.
**Gotchas สำคัญ:** เอกสารในแท็บยกเลิก **ลบไม่ได้** · **ยกเลิกแล้วนำเลขเดิมกลับมาใช้ (เอกสารเดิม) ไม่ได้** (คัดลอกเลขไปใช้ใบใหม่ได้) · ถ้า `ข้อมูลการชำระ` ไม่มี `⋮` = ถูกอ้างโดยเอกสารอื่น ต้องถอดก่อน · ระบบทำ reversing entry ให้อัตโนมัติ.

### ตรวจสอบคำสั่งซื้อจาก API (Shopee/Lazada/TikTok · article 7964849)

ตรวจว่าออเดอร์ถูกดึงมาสร้างใบเสร็จครบก่อนปิดงบ/ยื่นภาษี. `ตั้งค่า → ตั้งค่าเชื่อมต่อระบบภายนอก → ⋮ → ตั้งค่าการเชื่อมต่อ` → dropdown "คำสั่งซื้อที่นำมาสร้างเอกสาร ตั้งแต่สถานะ:" (แนะนำ **เฉพาะสถานะสำเร็จ** กันเอกสารผี). ตรวจ 2 วิธี: **XLOOKUP** (`=XLOOKUP(A2, รายงานใบเสร็จ!D:D, !B:B, "ไม่พบรายการ")`) · **มือ** (`ใบเสร็จรับเงิน → ค้นหาขั้นสูง → ช่องอ้างอิง` ใส่เลขออเดอร์). "ไม่พบ" + ไม่มีหมายเหตุคืน/ยกเลิก = ดึงเข้าเอง.

### บันทึกหนี้สงสัยจะสูญ / หนี้สูญ / หนี้สูญได้รับคืน (article 9710144)

Journal entry (สมุดรายวันทั่วไป). **ตั้งสำรอง:** Dr ค่าใช้จ่ายหนี้สงสัยจะสูญ / Cr ค่าเผื่อหนี้สงสัยจะสูญ (นำเป็นรายจ่ายภาษีไม่ได้ → บวกกลับ). **ตัดหนี้สูญ (กฎ 186):** มีค่าเผื่อ → Dr ค่าเผื่อ / Cr ลูกหนี้ + Dr ค่าใช้จ่ายหนี้สูญ / Cr ค่าเผื่อ · ไม่มีค่าเผื่อ → Dr ค่าใช้จ่ายหนี้สูญ / Cr ลูกหนี้. **ได้รับคืน:** Dr เงินสด/ธนาคาร / Cr รายได้อื่น (หนี้สูญได้รับคืน). ผลต่างบัญชี↔ภาษีขึ้นกับ (ก) ตั้งค่าเผื่อไหม (ข) ทำตามกฎ 186 ไหม.

### แก้หัก ณ ที่จ่าย บนใบแจ้งหนี้

**ออกใบเสร็จแล้ว (14178536):** `ชำระแล้ว` → `ดูใบเสร็จ` → `ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` → `ข้อมูลการชำระ ⋮ → ยกเลิก → ยืนยัน` → `ตัวเลือก → แก้ไข` → แก้คอลัมน์ หัก ณ ที่จ่าย (✏️) → `บันทึก` → ออกใบเสร็จ+ใบกำกับใหม่.
**ยังไม่ออกใบเสร็จ (14192051):** สั้นกว่า — ปุ่มคอลัมน์เป็น `สร้างใบเสร็จ` → `ข้อมูลการชำระ ⋮ → ยกเลิก → ยืนยัน` → `ตัวเลือก → แก้ไข` → แก้ WHT → `บันทึก`.
**Gotcha:** ไม่เห็นช่อง WHT → เปิดที่ `⚙ ตั้งค่าการแสดงผล` ก่อน.

### นำเข้าใบแจ้งหนี้ (Excel · IV007)

`รายรับ → นำเข้ารายการเอกสาร` → `+ สร้างเอกสารหลายรายการ` → **ใบแจ้งหนี้** → ตาราง spreadsheet (ไม่มีคอลัมน์รับชำระ) · หรือ `นำเข้าเอกสาร ▾ → รูปแบบ PEAK → ดาวน์โหลดฟอร์ม` (sheet `Import_Invoice` + `Description`). **คอลัมน์ A→S:** ลำดับที่* · วันที่* (YYYYMMDD) · เลขที่ (≤22, เว้น=รันเอง) · อ้างอิงถึง · ลูกค้า(E)/เลขทะเบียน 13(F)/เลขสาขา 5(G) · การออกใบกำกับ(H: 1=ออก/2=ไม่) · ประเภทราคา(I: 1แยก/2รวม/3ไม่มี) · สินค้า/บริการ(J) · บัญชี(K GL) · คำอธิบาย(L≤1000) · จำนวน(M) · ราคาต่อหน่วย(N) · ส่วนลดต่อหน่วย(O ≤ราคา) · อัตราภาษี(P default 7%) · ถูกหัก ณ ที่จ่าย(Q) · หมายเหตุ(R) · กลุ่มจัดประเภท(S). **ผล:** IVT อนุมัติแล้ว + รอรับชำระ. **Gotcha:** นำเข้าใบลดหนี้ผ่านวิธีนี้ไม่ได้ · ลำดับซ้ำ = รวมเป็นเอกสารเดียว.

## 4. ใบเสร็จรับเงิน (Receipt · RE/RT)

> **สรุปชนิดเอกสาร.** ใบเสร็จรับเงิน = ยืนยันว่า "ได้รับเงินแล้ว" — ออกได้เมื่อรับชำระจริง (ยกเว้นฉบับร่าง). รหัส **RE** (ล้วน) / **RT** (ใบเสร็จ/ใบกำกับภาษี · toggle "ออกใบกำกับภาษี").
> **สถานะ:** ร่าง → รออนุมัติ → **รับชำระแล้ว** (แก้ไม่ได้ · แม่กุญแจ 🔒) → ยกเลิก. + tab เอกสารอัตโนมัติ.
> **แก้ไข/ยกเลิก:** รับชำระแล้ว = แก้มูลค่า/ราคา/จำนวนไม่ได้ → **ยกเลิกแล้วสร้างใหม่** (นำเลขเดิมกลับมาใช้ได้). ยกเลิกอาจติดล็อกกุญแจ: น้ำตาล=ภ.พ.30 · น้ำเงิน=กระทบยอด · แดง=ปิดงวด (ปลดก่อน).
> **การลงบัญชี:** ต่อจาก IV → RV: Dr ธนาคาร (+WHT) / Cr ลูกหนี้ · ขายสด+ใบกำกับ → Dr ธนาคาร / Cr รายได้ + Cr 215101. บันทึกจริง = **RV-…** · **ฉบับร่าง = ยังไม่ลงบัญชี** (บันทึกบัญชี "-").

### สร้างใบเสร็จรับเงิน — สร้างสด / ต่อจาก IV / QO / BN

**สร้างสด (5999904/6022651):** `รายรับ → ใบเสร็จรับเงิน → + สร้าง` → ฟอร์ม (ข้อมูลลูกค้า/กลุ่มจัดประเภท/ราคาและภาษี+toggle ออกใบกำกับ/รายการ) + section **รับชำระเงิน** (พื้นฐาน/ขั้นสูง · วันที่ชำระ* · `ช่องทางการรับชำระเงิน → รับชำระโดย` [บัญชี] · จำนวนเงิน · `☑ ถูกหัก ณ ที่จ่าย` [ปกติ/e-Withholding · อัตรา · มูลค่า WHT]) → แถบเขียว + "ต้องรับชำระอีก 0.00" (บังคับ 0) → `อนุมัติใบเสร็จรับเงิน`.
**Dr/Cr (RT ขายสด+ใบกำกับ):** Dr 111301 ธนาคาร · Dr 115403 WHT / Cr 410xxx รายได้ · Cr 215101 ภาษีขาย.
**Gotchas:** ต้องระบุช่องทาง+จำนวนครบก่อนอนุมัติ · ใบเสร็จสร้างสด **แก้วันที่ตรงๆ ไม่ได้** ต้องยกเลิกแล้วออกใหม่ · ถ้าเคยบันทึกรับชำระบน IV แล้วยังไม่ออกใบเสร็จ → แท็บ `ข้อมูลการชำระ` ขึ้น "ยังไม่ออกใบเสร็จ" + ปุ่ม `ออกใบเสร็จ`.

**ต่อจาก IV:** `ใบแจ้งหนี้ → ดูทั้งหมด` → เปิด IV → (ยังไม่ชำระ) `ข้อมูลการชำระ → รับชำระ` → (ชำระแล้ว) `ออกใบเสร็จ` → ฟอร์มดึงข้อมูล IV (ล็อกเทา · แก้ได้แค่เลขเอกสาร/วันที่ออก/กลุ่มจัดประเภท/toggle ใบกำกับ) → `อนุมัติ`. **วันที่ออกใบเสร็จ ≥ วันที่ IV.** Dr/Cr: Dr ธนาคาร +WHT / Cr 113101 ลูกหนี้.

**ฉบับร่าง ต่อจาก IV:** stepper กด `+` กล่อง RE → กรอกรับชำระ → **`บันทึกร่าง`** (แทนอนุมัติ) → กล่อง RE บน Timeline **สีส้ม** = ร่าง · บันทึกบัญชี "-" · ไม่ขึ้นกระทบยอดธนาคาร. Gotcha: ปิดปุ่มช่องทางการรับชำระ (✏️ → เทา) เพื่อไม่โชว์บัญชีบนใบร่าง.

**ต่อจาก QO:** ตอบรับ QO ก่อน → (A) `ยอมรับแล้ว → คำสั่ง → สร้างใบแจ้งหนี้ ▲ → สร้างใบเสร็จรับเงิน` · (B) stepper `+` กล่อง RE → dialog "สร้างเอกสารด้วยวิธี" (ตามใบเสนอราคา/แบ่งตามสินค้า) → ฟอร์ม + รับชำระ → อนุมัติ (ข้าม IV).

**ต่อจาก BN (ใบวางบิล):** `ใบวางบิล → ดูทั้งหมด` → เปิด BN → `ข้อมูลการชำระ → รับชำระ` (แนะนำ **ขั้นสูง** เพราะรวมหลาย IV) → `ออกใบเสร็จ` → อนุมัติ. Dr/Cr: Dr ธนาคาร +WHT / Cr 113101 ลูกหนี้ (ตัดตาม IV ที่รวม).

### สร้างใบเสร็จรับเงิน/ใบกำกับภาษี (5999904/6022651) — เรื่อง "ใบกำกับภาษี ×2"

**toggle "การออกใบกำกับภาษี":** เปิด → **RT-** (มี chip TIV · "เป็นใบกำกับภาษี") · ปิด → **RE-** ("ยื่นยอดขาย ภ.พ.30 · ไม่ใช่ใบกำกับ"). ⚠️ ระบบยังไม่แยกเลขรัน "อย่างย่อ" กับ "เต็มรูป" อัตโนมัติ — จัดการลำดับเอง. **Dr/Cr:** Dr 111301 ธนาคาร (รวม VAT) / Cr 410101 รายได้ / Cr ภาษีขาย 7%.

### ยกเลิกใบเสร็จรับเงิน (6012814)

`ดูทั้งหมด` → เปิดใบ → `☰ ตัวเลือก → ▣ ยกเลิกเอกสาร` (แก้ไข/คัดลอกเป็นเทา) → **ยกเลิก** (dot เทา). สมุดรายวัน RV ที่ผูกก็เป็น "ยกเลิก" ตาม. **ล็อกกุญแจ** (น้ำตาล=ภ.พ.30/น้ำเงิน=กระทบยอด/แดง=ปิดงวด) ต้องปลดก่อน. **Gotcha:** ใบเสร็จ **ยกเลิกการรับชำระแยกไม่ได้** (สร้าง+รับชำระในขั้นเดียว → ยกเลิกทั้งใบ) · ยกเลิกใบที่อ้างจาก IV/QO ไม่กระทบต้นทาง.

### แก้ไขใบเสร็จรับเงินส่วนใดได้บ้าง (14073987)

กดดินสอ ✏️ ทีละฟิลด์. **สร้างเดี่ยว:** แก้ได้ เลขเอกสาร/อ้างอิง/กลุ่มจัดประเภท/**รายการ**/ข้อมูลการชำระ/หมายเหตุ/ไฟล์แนบ/แท็ก. **อ้างอิงจากเอกสารอื่น:** เหมือนกันแต่ **แก้ "รายการ" ไม่ได้** (ดึงจาก IV). **แก้ไม่ได้ทุกกรณี (ต้องยกเลิกแล้วสร้างใหม่):** ข้อมูลลูกค้า · ราคาและภาษี · จำนวน · ราคา · ส่วนลด · ภาษี · หัก ณ ที่จ่าย.

## 5. ใบกำกับภาษีขาย (Tax Invoice · TIV)

> **สรุปชนิดเอกสาร.** ใบกำกับภาษี = หลักฐาน VAT ขาย. PEAK รวมเป็นใบเดียวกับ IV/RE ผ่าน toggle "ออกใบกำกับภาษี" (→ IVT ม.78 / RT ม.78/1) หรือออก **แยกเดี่ยว (TIV · ม.86/4)** จากเอกสารที่มี VAT ล้วน. รหัส **TIV** · เลข `TIV-YYYYMM#####`.
> **สถานะ:** ร่าง → รออนุมัติ → **ออกแล้ว/ยอมรับแล้ว** → ยกเลิก. + tab e-Tax Invoice.
> **จุดรับผิด VAT (tax point):** ผู้ขายสินค้า = ส่งมอบ/รับเงินก่อน/โอนกรรมสิทธิ์ (เกิดก่อน) · ผู้ให้บริการ = รับเงิน.
> **การลงบัญชี (TIV แยก · JVTX):** Dr **215102 ภาษีขายรอเรียกเก็บ** / Cr **215101 ภาษีขาย ภ.พ.30** (ย้าย deferred → รับรู้). เก็บเอกสาร ≥ 5 ปี.

### สร้าง TIV แยก (จาก IV / RE / ใบรับเงินมัดจำ)

**เงื่อนไข:** เอกสารต้นทางต้องมี VAT + ยังไม่เคยดึงออกใบกำกับ.
1. `รายรับ → ใบกำกับภาษีขาย → + สร้าง` → stepper `● เลือกเอกสาร ⋯ ○ ใบกำกับภาษี (TIV)`.
2. **เลือกเอกสารอ้างอิง:** ตัวกรอง วันที่ออก / ประเภทเอกสาร (ใบแจ้งหนี้/ใบเสร็จ/ใบรับเงินมัดจำ) / ชื่อลูกค้า* / ค้นหา → **ติ๊ก radio** (1 ใบ) → `ถัดไป`.
3. ฟอร์ม (อ้างอิงจากระบบ IV-… / อ้างอิง / เลขที่ TIV-…) — inherit ลูกค้า+รายการ · แก้ได้แค่เลขเอกสาร+วันที่ออก.
4. `อนุมัติใบกำกับภาษีขาย ▾`.
**Dr/Cr (JVTX):** Dr 215102 ภาษีขายรอเรียกเก็บ / Cr 215101 ภาษีขาย ภ.พ.30 (คำอธิบาย "บันทึกใบกำกับภาษีขาย #TIV-…").

### สร้างใบกำกับภาษี 3 รูปแบบ (ตามมาตรา)

หัวใจ = **toggle "การออกใบกำกับภาษี"** ในฟอร์ม IV/RE. OFF → IV/RE ล้วน (VAT ลง **215102**) · ON → IVT (ม.78)/RT (ม.78/1) (VAT ลง **215101** ทันที). แบบที่ 3 = TIV แยก (ม.86/4) ตามข้อบน. เลือกให้ตรง tax point (สินค้า=ส่งมอบ/รับเงิน/โอนกรรมสิทธิ์ · บริการ=รับเงิน).

### ใบเสร็จ/ใบกำกับภาษีอย่างย่อ (TI004)

ต้องเป็น **RT-** (เปิดออกใบกำกับ). `พิมพ์ ▾` → **อย่างย่อ (A4)** หรือ **POS** (58mm). ฟอร์มสร้างเหมือนใบเสร็จปกติ (ข้อมูลลูกค้า/รายการ/เงินมัดจำ/รับชำระ พื้นฐาน+ขั้นสูง [ค่าธรรมเนียม/ตัดชำระ/WHT]). Dr/Cr: Dr ธนาคาร +WHT / Cr รายได้ + Cr 215101.

### ยกเลิกใบกำกับภาษี

`ใบกำกับภาษีขาย → ดูทั้งหมด` → เปิด TIV → `☰ ตัวเลือก ▾ → 🗎 ยกเลิกเอกสาร → ยืนยัน` → ย้ายแท็บ **ยกเลิก** (กลับรายการ JVTX). **Gotcha:** ยกเลิก/ลบ → เลขถัดไปรันต่ออัตโนมัติ (เลขเดิมหาย) · อยากใช้เลขเดิมต้อง "แก้ไขเลขที่เอกสาร" บนใบใหม่เอง.

### ออก TIV 1 ใบ จากหลาย IV

PEAK ทำตรงๆ ไม่ได้ ต้องผ่าน **ใบวางบิล (BN)**.
**วิธี 1 (RT รวมใบเดียว):** สร้าง BN จากหลาย IV → อนุมัติ → ต่อยอด `สร้างใบเสร็จรับเงิน` → **เปิด toggle ออกใบกำกับ** → RT รวมยอด (แจงทุก IV).
**วิธี 2 (TIV แยกรายการ):** สร้าง BN → ออก **RE ล้วน** (toggle OFF) → `ใบกำกับภาษีขาย → + สร้าง` → เลือกเอกสารอ้างอิง (ประเภท=ใบเสร็จ) → ติ๊ก RE → ถัดไป → อนุมัติ → **TIV 1 ใบ แจง line item แยกตาม IV ต้นทาง**.
**เงื่อนไข:** IV ต้นทางต้องเป็นแบบไม่ใช่ใบกำกับ (IV ล้วน) ก่อนรวม · วิธี 1 รวมยอดเดียว, วิธี 2 แสดง line item แยก.

## 6. ใบลดหนี้ (Credit Note) + 7. ใบเพิ่มหนี้ (Debit Note)

> **สรุปชนิดเอกสาร.** ใบลดหนี้ = ออกเพื่อ **ลดหนี้/ลดยอด** หลังขายไปแล้ว (สินค้าชำรุด/ไม่ครบ/ลดราคา/รับคืน). รหัส **CN** · เลข `CNT-YYYYMM#####`. **ใบเพิ่มหนี้ = เอกสารพี่น้อง (mirror)** ออกเพื่อ **เพิ่มยอด/เก็บเพิ่ม** (รหัส DBN — ดูหัวข้อ 7).
> **สถานะ (CN):** ร่าง → รออนุมัติ → รอชำระ → ชำระแล้ว → ยกเลิก.
> **การลงบัญชี (CN, กลับด้าน IV):** Dr 410xxx รายได้ + Dr 215101 ภาษีขาย / Cr 113101 ลูกหนี้ (ยังไม่จ่าย) หรือ Cr 111301 ธนาคาร (คืนเงิน) หรือ Cr 999999 บัญชีพัก (ไม่คืน หักครั้งหน้า).

**2 เส้นทางเข้า:** (1) `รายรับ → ใบลดหนี้ → + สร้าง` (เลือก อ้างอิง/ไม่อ้างอิง) · (2) เปิดเอกสารต้นทาง → `ตัวเลือก → สร้างใบลดหนี้`. **โครงฟอร์ม:** ข้อมูลลูกค้า → กลุ่มจัดประเภท → ราคาและภาษี → **ข้อมูลการลดหนี้** (สาเหตุ*/คำอธิบาย/ลดทุกรายการ?/รับคืนสินค้า? [รับคืน=สต็อกเพิ่ม]) → รายการ → **สรุปข้อมูล** (มูลค่าตามเอกสารเดิม/มูลค่าที่ถูกต้อง) → ชำระเงิน.

|---|---|---|---|---|
| **ไม่คืนเงิน หักครั้งหน้า** | ได้ทั้งคู่ | ไม่คืน | ชำระ "ขั้นสูง" → หักเงินที่ต้องจ่ายด้วย **999999 บัญชีพัก** → บิลถัดไปเลือกพักเดิม ระบบล้างเอง | 6363678 |

**ชำระเงินคืน (6360586):** เลือก `ช่องทางจ่ายชำระเงินคืน` → Cr เงินสด/ธนาคาร.
**ข้อควรระวัง:** IV ที่นำใบมัดจำมาตัดแล้ว ออกใบลดหนี้ไม่ได้ (ปุ่มเทา) · ใบรับเงินมัดจำล็อก "ไม่รับคืนสินค้า".

### ยกเลิกใบลดหนี้ (9345091)

`ดูทั้งหมด` → เปิด → `ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` → **ยกเลิก**. Gotcha: ยกเลิกแล้วได้เลขใหม่ตามลำดับล่าสุด (แก้กลับได้).

### Error Code 114 (10000420)

toast แดง **"ไม่สามารถออกใบลดหนี้วันที่ก่อนวันที่เอกสารที่อ้างอิง"** — สาเหตุ `วันที่ออก` CN เก่ากว่าวันที่เอกสารต้นทาง → แก้ `วันที่ออก` ให้ ≥ วันที่ต้นทาง.

## 7. ใบเพิ่มหนี้ (Debit Note · DBN/DBNT)

> **สรุปชนิดเอกสาร.** ใบเพิ่มหนี้ = mirror ของใบลดหนี้ — ออกเพื่อ **เพิ่มมูลค่า/เก็บเงินเพิ่ม** จากเอกสารเดิม (ออกบิลคิดขาด/ส่งของเพิ่ม/คำนวณราคาต่ำไป). รหัส **DBN** (ไม่อ้างอิง · `DBN-YYYYMM#####`) / **DBNT** (อ้างอิงเอกสารเดิม · `DBNT-YYYYMM#####`).
> **สถานะ:** ร่าง → รออนุมัติ → รอรับชำระ → รับชำระแล้ว → ยกเลิก.
> **สายเอกสาร:** `ใบแจ้งหนี้ (IV) ⟶ ใบเพิ่มหนี้ (DBN)`.
> **การลงบัญชี (ทิศทางกลับด้านกับใบลดหนี้ · เพิ่มรายได้/ลูกหนี้ + VAT ขายเพิ่ม):** อนุมัติ → SV + RV ตามความสัมพันธ์วันที่.

### ออกใบเพิ่มหนี้ แบบไม่อ้างอิงเอกสารเดิม (6324552)

`รายรับ → ใบเพิ่มหนี้ → + สร้าง` → รูปแบบ **"ไม่อ้างอิงเอกสารเดิม"** → `ถัดไป` → ฟอร์ม "สร้างใบเพิ่มหนี้/ใบกำกับภาษี":
- ข้อมูลลูกค้า (ชื่อ*/วันที่ออก*/วันที่ครบกำหนด*) · กลุ่มจัดประเภท · ราคาและภาษี (toggle ออกใบกำกับ)
- **ข้อมูลการเพิ่มหนี้:** สาเหตุ* · คำอธิบายอย่างย่อ* (≤98) · **มีการส่งสินค้าเพิ่มหรือไม่** (ส่งเพิ่ม=ตัดสต็อก / ไม่ส่ง=คงเดิม)
- รายการ (ระบุ "ยอดหนี้ที่เพิ่มขึ้น") · **สรุปข้อมูล:** ✏️ **มูลค่าตามเอกสารเดิม** (เช่น 6,000) → มูลค่าที่ถูกต้อง (=เดิม+เพิ่ม) → จำนวนเงินทั้งสิ้น (= ส่วนเพิ่ม+VAT)
- **รับชำระเงิน** (พื้นฐาน/ขั้นสูง [ค่าธรรมเนียม/ปรับปรุง]) หรือ **"ยังไม่ชำระเงิน (ตั้งหนี้ไว้ก่อน)"**
- `อนุมัติใบเพิ่มหนี้ ▾`.

**Dr/Cr (SV/RV):**
| กรณีวันที่ | SV | RV |
|---|---|---|
| ออก=ชำระ | Dr เงิน / Cr รายได้ | — |
| ออก<ชำระ | Dr ลูกหนี้ / Cr รายได้ | Dr เงิน / Cr ลูกหนี้ |
| ออก>ชำระ | Dr **212103** รายได้รับล่วงหน้า / Cr รายได้ | Dr เงิน / Cr 212103 |
| ตั้งหนี้ไว้ก่อน | Dr ลูกหนี้ / Cr รายได้ | Dr เงิน / Cr ลูกหนี้ |

> "ส่งสินค้าเพิ่ม" มีคู่บัญชีต้นทุน (Dr ต้นทุนขาย / Cr สินค้าคงเหลือ) + ตัดสต็อก.
**Gotcha:** ต้องกรอก "มูลค่าตามเอกสารเดิม" (✏️) เพื่อคำนวณ "มูลค่าที่ถูกต้อง" · จำนวนเงินคิดเฉพาะ **ส่วนที่เพิ่ม** + VAT.

### ออกใบเพิ่มหนี้ จากเอกสารเดิม (6324516)

`+ สร้าง` → **"อ้างอิงจากเอกสารเดิม"** → เลือกเอกสารอ้างอิง (ประเภท*/ชื่อลูกค้า*/ค้นหา) → ติ๊ก radio → `ถัดไป` → ฟอร์ม (เลข **DBNT-…**) ดึงลูกค้า+รายการมา → **ติ๊ก ☑ หน้าแถวที่เพิ่มหนี้** + แก้จำนวน/ราคา → สรุปดึง "มูลค่าตามเอกสารเดิม" ให้ → รับชำระ หรือ "ตั้งหนี้ไว้ก่อน" → `อนุมัติ`. รหัสมี **T** (DBNT) = ต่อยอด · ผูก "อ้างอิงจากระบบ" กับ IV ต้นทาง.

### ยกเลิกใบเพิ่มหนี้ (9345119)

`ดูทั้งหมด` → คลิกเลขเอกสาร → `☰ ตัวเลือก ▾ → ยกเลิกเอกสาร` (แก้ไข/คัดลอก/สร้างใบวางบิล เทา) → popup "ยืนยันการยกเลิก" → `ยืนยัน` → **ยกเลิก** (dot เทา · ริบบิ้นเทา, เก็บอ้างอิง IV-…/SV-…). Gotcha: ยกเลิก ≠ ลบ (ยกเลิกเก็บเลข+ประวัติ) · ลบแล้วเลขรันใหม่ (แก้เลขเดิมเองได้).

## 8. ใบวางบิล (Billing Note · BN)

> **สรุปชนิดเอกสาร.** ใบวางบิล = นำ **ใบแจ้งหนี้/ใบรับเงินมัดจำ/ใบลด-เพิ่มหนี้ (แบบอ้างอิง) หลายใบที่ "ยังไม่รับชำระ"** ของ **ลูกค้ารายเดียว + สกุลเงินเดียว** มารวมเป็นเอกสารเรียกเก็บใบเดียว → รับชำระ+ออกใบเสร็จรวมทีเดียว. รหัส **BN** · เลข `BN-YYYYMM#####`.
> **สถานะ:** ร่าง → รออนุมัติ → **รอรับชำระ** → เกินเวลารับชำระ → **รับชำระแล้ว** → ยกเลิก.
> **แก้ไข/ยกเลิก:** รอรับชำระ = แก้ได้เลย · รับชำระ+ออกใบเสร็จแล้ว = ถอย ใบเสร็จ → การรับชำระ → BN.
> **การลงบัญชี:** ออก BN = **ไม่ลงบัญชี** (หนี้ยังอยู่ที่ IV ต้นทาง) · **รับชำระ → RV** (Dr 111301 ธนาคาร + Dr 115403 WHT / Cr 113101 ลูกหนี้).

### สร้างใบวางบิล (5781303)

**เข้า:** `รายรับ → ใบวางบิล → + สร้าง` หรือ **ทางลัด** `ใบแจ้งหนี้ → รอรับชำระ` → ติ๊ก checkbox → `+ สร้างใบวางบิล`.
**Step 1 เลือกเอกสารอ้างอิง:** ตัวกรอง วันที่ออก / ประเภทเอกสาร* (ใบรับเงินมัดจำ/ใบแจ้งหนี้/ใบเพิ่มหนี้) / ชื่อลูกค้า* / ค้นหา → ตาราง + ช่อง **จำนวนเงินวางบิล** (แก้ได้/บรรทัด) → ติ๊ก → `ถัดไป`.
**Step 2 กรอกหัว:** อ้างอิง + เลขที่ `BN-…` (⚙) · ข้อมูลลูกค้า (วันที่ออก*/ครบกำหนด*) · **ราคานี้เป็นค่าเงิน*** · รายการ (แต่ละบรรทัด ✏️ แก้ยอดวางบิลรายสินค้าใน popup) · สรุป "เอกสารที่ออก" → `อนุมัติใบวางบิล ▾`.
**Step 3 รับชำระ + ออกใบเสร็จ:** แท็บ `ข้อมูลการชำระ → รับชำระ` → dialog (พื้นฐาน/ขั้นสูง · วันที่* · รับชำระโดย · จำนวน · WHT [ปกติ/e-Withholding · อัตรา]) → `รับชำระเงิน` → **รับชำระแล้ว** + `บันทึกบัญชี RV-…` + "ยังไม่ออกใบเสร็จ" + `ออกใบเสร็จ` → ฟอร์มใบเสร็จ (toggle ออกใบกำกับ) → `อนุมัติ`.
**Dr/Cr (RV · BN 11,665 WHT 3%):** Dr 111301 ธนาคาร 11,506.17 · Dr 115403 WHT 158.83 / Cr 113101 ลูกหนี้ 11,665.00.

### ใบวางบิลแบบแบ่งชำระ (6264575)

**จุดสำคัญ:** ในช่อง **จำนวนเงินวางบิล** กรอก **น้อยกว่ายอดเต็ม** (เช่น 6,955→1,500) → ระบบหักจากยอดคงเหลือ IV ต้นทาง โดยไม่ต้องสร้าง IV ใหม่. ตรวจยอดคงเหลือที่ `ใบแจ้งหนี้ → ดูทั้งหมด → คอลัมน์มูลค่าที่ต้องชำระ` → วนวางบิลรอบใหม่. **⚠️ 1 BN = 1 ใบเสร็จ** (แยกใบเสร็จหลายครั้งในใบเดียวไม่ได้).

### แก้ไขใบวางบิล — ยังไม่รับชำระ (6021413)

เปิด BN (รอรับชำระ) → `≡ ตัวเลือก → แก้ไข` → แก้ (ลูกค้า/วันที่/สกุล/จำนวนเงินวางบิล) · เพิ่ม/ลด IV: `ย้อนกลับ` → หน้าเลือกเอกสาร → ติ๊กเพิ่ม/เอาออก → `บันทึก`.

### แก้ไขใบวางบิล — รับเงิน+ออกใบเสร็จแล้ว (6054560)

**ถอยย้อนหลัง:** stepper กล่อง `ใบเสร็จรับเงิน (RE) ⌃` → เข้า RE → `≡ ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` (แก้ไข/คัดลอกเทา) → กลับ BN `ข้อมูลการชำระ → ⋮ → ยกเลิก → ยืนยัน` → สถานะกลับ รอรับชำระ → `ตัวเลือก → แก้ไข → บันทึก`. (การยกเลิกกลับรายการ RV.) **Gotcha:** ลำดับต้อง ใบเสร็จก่อน → การรับชำระทีหลัง.

### แก้ไขการแสดงข้อมูลสินค้า/บริการ บน BN (8974547)

เปิด BN → `พิมพ์` → preview → `⚙ แก้ไข` → ✏ หัวตาราง `รายการ` → dialog "แก้ไขการแสดงรายละเอียดเอกสาร": toggle **แสดงรายละเอียดสินค้า/บริการ** + ย่อย (ชื่อ/เลขที่/คำอธิบายสินค้า) → `บันทึก` (เฉพาะใบนี้ / ตั้ง default). ปิด = BN โชว์แค่ระดับ IV (เลข+ยอด).

### พิมพ์ใบวางบิล (6043628)

`ดูทั้งหมด` → เปิด BN → `พิมพ์` (หรือ `พิมพ์ ▾ → พิมพ์ PDF`) · `คัดลอกลิงก์` / `ส่งเอกสาร ▾` · หน้า list มี `พิมพ์รายงาน` (หลายเอกสาร).

### ยกเลิกใบวางบิล (6043684)

**กรณี A (ยังไม่รับชำระ):** เปิด BN → `≡ ตัวเลือก → ยกเลิกเอกสาร → ยืนยัน` → ย้ายแท็บยกเลิก. **กรณี B (รับชำระ+ใบเสร็จแล้ว):** ยกเลิกใบเสร็จ (RE ตัวเลือก→ยกเลิก) → ยกเลิกการรับชำระ (`⋮ → ยกเลิก`) → ยกเลิก BN. หนี้ต้นทางที่ IV กลับมาค้าง. **Gotcha:** ยกเลิก → เลขใบใหม่รันจากลำดับล่าสุด (ไม่ reuse อัตโนมัติ).

## 💡 สรุปสำหรับสร้างระบบ Pacred เอง

1. **Doc-code + running-series ต่อชนิด** — ทุกเอกสารมี prefix เฉพาะ + เลขรัน `PREFIX-YYYYMM#####` (ปีค.ศ.+เดือน+run 5 หลัก), แก้เลขได้, ยกเลิกแล้วรันเลขถัดไปอัตโนมัติ (เลขเดิมนำกลับมาใช้กับ *เอกสารเดิม* ไม่ได้). แยก IV/IVT · RE/RT (toggle "ออกใบกำกับภาษี" สลับ prefix). batch นำเข้าใช้ `IM+doc`.
2. **State-machine ต่อชนิด** — QO (ร่าง→รอตอบรับ→ยอมรับ/ปฏิเสธ→ออกบางส่วน/ครบ→ยกเลิก) · IV/BN (รอรับชำระ→เกินเวลา→รับชำระแล้ว) · DR (รอรับชำระ→รอหักมัดจำ→หักแล้ว) · RE/TIV (รับชำระ/ออกแล้ว→ยกเลิก). สถานะควบคุมว่าปุ่ม/การแก้ไขไหนเปิดใช้.
3. **Stepper convert-carry** — QO→IV→RE→TIV + BN + DR + CN/DBN. ปุ่ม `+` สร้างขั้นถัดไปโดย **inherit ข้อมูลลูกค้า/รายการ** (ล็อกฟิลด์ที่ดึงมา · แก้ได้แค่เลข/วันที่/toggle). รองรับแบ่ง (ตามใบ/แบ่งรายการ/แบ่ง %).
4. **GL-account per line + auto-posting** — ทุก line item ผูก **ผังบัญชี (410101 รายได้ขาย / 410201 รายได้บริการ / 212104 มัดจำ / 212103 รายได้รับล่วงหน้า ฯลฯ)**; ระบบ auto-สร้างสมุดรายวัน **SV (ขาย) / RV (รับ) / JV (ทั่วไป/ย้าย VAT)** ตอนอนุมัติ/รับชำระ พร้อม reversing entry อัตโนมัติเมื่อยกเลิก. QO+BN ไม่ post ตอนออก.
5. **VAT 7% + WHT per line + 2 บัญชี VAT** — คุมด้วยประเภทราคา (แยก/รวม/ไม่มีภาษี) + toggle ออกใบกำกับ. VAT ขายมี **215102 (รอเรียกเก็บ/deferred)** ↔ **215101 (ภ.พ.30/รับรู้)** — TIV = ย้าย 215102→215101. WHT ต่อ line → ตั้ง **115403 ภาษีถูกหัก ณ ที่จ่าย** (สินทรัพย์) รองรับ ปกติ/e-Withholding.
6. **เงินมัดจำ apply** — พักที่ **212104**, ตัดเข้าบิลผ่าน "เลือกเงินมัดจำ" (≤10 ใบ, วันที่มัดจำ ≤ วันบิล, **ประเภทภาษีต้องตรง**) หรือใส่ยอดติดลบ. ยอดสุทธิบิลติดลบได้ (มัดจำ>ยอด).
7. **ใบแจ้งหนี้ = ใบส่งของ = บันทึกลูกหนี้** — เอกสารเดียวทำ 3 หน้าที่ (ตั้งลูกหนี้ Dr 113101 + พิมพ์ได้หลายหัว: ใบแจ้งหนี้/ใบส่งของ/ใบวางบิล). ขายสินค้าแบบ perpetual auto-post ต้นทุน (Dr 510103 / Cr 114102) + ตัดสต็อก.
8. **ใบวางบิล = รวมหลาย IV = billing-run** — รวมเฉพาะ **ลูกค้า+สกุลเดียวกัน+ยังไม่ชำระ**, กรอก "จำนวนเงินวางบิล" ต่อบรรทัด (รองรับแบ่งชำระ = กรอกน้อยกว่ายอดเต็ม, หักจากยอดคงเหลือ IV โดยไม่สร้าง IV ใหม่). **1 BN = 1 ใบเสร็จ**. ออก BN ไม่ post; รับชำระจึง post RV.
9. **Edit-rules ขึ้นกับสถานะ (ถอยปลายน้ำก่อน)** — เอกสารที่ผูก (การชำระ/ใบเสร็จ/BN/CN/DBN) จะ **ล็อก (แม่กุญแจ)**; ต้องถอยลำดับ **ใบเสร็จ → การรับชำระ → เอกสารอ้างอิง → ต้นเอกสาร** ก่อนแก้/ยกเลิก. เอกสารในแท็บ "ยกเลิก" ลบไม่ได้ (เก็บ audit trail). แก้บางฟิลด์ผ่าน ✏️ ได้โดยไม่ยกเลิก; แก้มูลค่า/ราคา/จำนวน/ภาษี ต้องยกเลิกแล้วสร้างใหม่.
10. **ลดหนี้/เพิ่มหนี้ = การแก้ยอดหลังออกเอกสาร** — CN ลด/คืน (คืนสด / ตั้งหนี้ / หักบิลอื่น / พัก 999999 หักครั้งหน้า) · DBN เพิ่ม/เก็บเพิ่ม. ต้องกรอก "มูลค่าตามเอกสารเดิม" + "มูลค่าที่ถูกต้อง (≥0)"; วันที่ต้อง ≥ วันเอกสารต้นทาง (ไม่งั้น Error 114); ทิศทางบัญชีกลับด้านกัน + toggle รับคืนสินค้า (กระทบสต็อก/ต้นทุน).
11. **Draft vs Approve + recurring + import** — ทุกเอกสารมี "บันทึกร่าง" (ยังไม่ post, กล่อง stepper สีส้ม) กับ "อนุมัติ" (post). รองรับ **เอกสารอัตโนมัติ (RCIV/RCRT)** รายเดือน/รายปี (สร้างเป็น ร่าง/รออนุมัติ/อนุมัติ + ส่งอีเมล + สถานะ เปิด/ปิด/สิ้นสุด/ไม่สำเร็จ · ใช้ทำ deferred-revenue amortization ด้วย "ขั้นสูง → หัก 212103") และ **นำเข้า Excel/API** (batch IM… + template + preview ก่อนสร้างจริง).
12. **"ขั้นสูง" ในการรับชำระ = engine ของเคสพิเศษ** — checkbox "ค่าธรรมเนียม/รายการปรับปรุง" (เพิ่ม/หักเงินที่ต้องรับ + ผังบัญชี) ครอบเช็ค/เช็คล่วงหน้า (113102) / บัตร-e-Wallet (530501) / โอนเกิน (212306 คืน หรือ 420108 รายได้) / ตัดชำระกับเอกสาร / วันออก≠วันรับ (SV/JV/RV ตามความสัมพันธ์วันที่).
