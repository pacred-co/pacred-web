## 1. Current CSP state (next.config.ts:14-50)

```ts
// next.config.ts (current — Phase H2 deferred)
{
  key: "Content-Security-Policy",
  value: [
    "default-src 'self'",
    "script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.googletagmanager.com https://hcaptcha.com https://*.hcaptcha.com",
    "style-src 'self' 'unsafe-inline'",
    "img-src 'self' data: https:",
    "connect-src 'self' https://*.supabase.co https://*.hcaptcha.com https://www.google-analytics.com",
    "frame-src https://*.hcaptcha.com",
    "frame-ancestors 'none'",
    "base-uri 'self'",
    "form-action 'self'",
  ].join("; "),
}
```

**Comment in code:** *"CSP uses 'unsafe-inline' for styles/scripts because Next.js inlines critical CSS + Tailwind. Removing requires CSP nonce/hash strategy (nonces / hashes) is a Phase H2 task — not blocking launch."*

**Why it matters (OWASP Top 10):**
- `unsafe-inline` script-src = XSS protection significantly weakened. If an attacker injects `<script>` content (via user-generated content + a missed escape), it executes.
- `unsafe-eval` script-src = `eval()` / `Function()` constructors allowed. Pacred doesn't use these directly but some 3rd-party scripts might.
- `unsafe-inline` style-src = inline `style="..."` allowed. Style-based XSS is less severe but still a vector (data exfiltration via CSS).

**Pacred-specific risk profile:**
- Customer-side has minimal user-generated content (no rich-text editors, no public posts). XSS surface is low.
- Admin-side has `contact_messages` content rendered via `dangerouslySetInnerHTML` at `/admin/page.tsx:593` — small risk.
- BUT: defense-in-depth principle — fix anyway. Cost is moderate, benefit is real.

## 3. Migration approach (Next.js 16 nonce flow)

### 3.1 Generate nonce in middleware

`proxy.ts` — generate cryptographic nonce per-request:

```ts
import { createHash, randomBytes } from "crypto";

export async function middleware(request: NextRequest) {
  const nonce = randomBytes(16).toString("base64");
  const cspHeader = `
    default-src 'self';
    script-src 'self' 'nonce-${nonce}' 'strict-dynamic' https://www.googletagmanager.com https://hcaptcha.com https://*.hcaptcha.com;
    style-src 'self' 'nonce-${nonce}';
    img-src 'self' data: https:;
    connect-src 'self' https://*.supabase.co https://*.hcaptcha.com https://www.google-analytics.com;
    frame-src https://*.hcaptcha.com;
    frame-ancestors 'none';
    base-uri 'self';
    form-action 'self';
  `.replace(/\s{2,}/g, " ").trim();

  // Pass nonce to RSC via header (Next.js convention)
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("x-nonce", nonce);

  const response = NextResponse.next({ request: { headers: requestHeaders } });
  response.headers.set("Content-Security-Policy", cspHeader);
  return response;
}
```

### 3.2 Read nonce in server components

```tsx
// app/layout.tsx
import { headers } from "next/headers";

export default async function RootLayout({ children }) {
  const nonce = (await headers()).get("x-nonce") ?? "";

  return (
    <html>
      <head>
        <script
          nonce={nonce}
          dangerouslySetInnerHTML={{ __html: THEME_INIT_SCRIPT }}
        />
        {/* ... */}
      </head>
      <body>{children}</body>
    </html>
  );
}
```

### 3.3 Remove old CSP header from `next.config.ts`

Move CSP completely to `proxy.ts` (per-request) since it now depends on per-request nonce. Delete the `Content-Security-Policy` block from `next.config.ts:40-44`.

### 3.4 Tailwind v4 styles

`@tailwindcss/postcss` v4 outputs inline CSS at build time → currently relies on `'unsafe-inline'`. Two options:

**Option A — Hash-based (simpler):** compute SHA-256 hash of the inlined `<style>` blocks at build time → add to CSP `style-src 'self' 'sha256-...'`. Requires a build-time script that reads `.next/build-manifest.json` and emits hashes.

**Option B — Move CSS to external file:** force Tailwind to emit `.css` file (no inline). Requires `next.config.ts` flag `experimental.optimizeCss: false` + custom postcss config. Loses some perf wins.

**Recommend Option A** — small build-time script, no perf loss.
